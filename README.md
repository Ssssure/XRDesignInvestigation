# XRDesignInvestigation

## VAC (Vergence-Accommodation Conflict) 现象简介

### 什么是 VAC?

VAC（Vergence-Accommodation Conflict，辐辏-调节冲突）是当前 XR（扩展现实，包括 VR/AR/MR）头显设备中一个核心的视觉舒适性问题。它源于头显的光学显示方式与人眼自然观看方式之间的根本矛盾。

### 人眼的自然视觉机制

在现实世界中，人眼同时依赖两种协调机制来观看物体：

| 机制 | 说明 |
|------|------|
| **调节（Accommodation）** | 眼睛晶状体改变形状，使不同距离的物体在视网膜上清晰成像，类似相机对焦。 |
| **辐辏（Vergence）** | 双眼向内旋转（会聚）或向外旋转（发散），使双眼视线汇聚在同一目标物体上。 |

在自然视觉中，这两种机制是**耦合**的：当你注视一个近处物体时，晶状体会变厚以对焦近处，同时双眼会向内会聚；当你看远处时，晶状体变薄，双眼趋于平行。这种协调关系被称为**调节-辐辏联动（AC linkage）**。

### XR 头显中的冲突

在大多数当前的 XR 头显中，显示屏（或光学投影面）位于距眼睛**固定距离**处（通常约 1.5–2 米的光学焦平面）。然而，头显渲染的虚拟场景中的物体可能出现在任意深度——从近在咫尺到无限远处。

这就产生了冲突：

- **辐辏（Vergence）**：双眼根据虚拟物体的 3D 位置进行会聚，指向虚拟物体所在的深度。
- **调节（Accommodation）**：晶状体却需要对焦在显示屏的固定物理距离上，才能看清画面。

也就是说，眼睛的会聚深度和对焦深度被强制"解耦"了——你的眼睛会聚在 30cm 处的虚拟物体上，但晶状体却需要对焦在 2m 处的屏幕上。这种不一致就是 **VAC**。

```
现实世界（无冲突）：        XR 头显（存在 VAC）：

  物体 ●                      虚拟物体 ●（30cm）
      / \                          / \
     /   \  辐辏 → 30cm           /   \  辐辏 → 30cm
    👁   👁 调节 → 30cm          👁   👁 调节 → 2m（屏幕）
                                        ↑
    辐辏 = 调节 ✓               辐辏 ≠ 调节 ✗ （冲突！）
```

### VAC 带来的问题

VAC 会导致一系列不良的用户体验和健康问题：

1. **视觉疲劳（Visual Fatigue）**：长时间使用后眼睛酸痛、干涩。
2. **头痛与眩晕**：调节-辐辏不匹配加重前庭-视觉冲突。
3. **深度感知失真**：用户对虚拟物体的距离判断可能出现偏差。
4. **注视近处虚拟物体时不适感加剧**：冲突量与物体深度和屏幕焦平面之间的差距成正比，近距离交互时最为严重。

### 当前的缓解与解决方案

| 方案 | 原理 | 成熟度 |
|------|------|--------|
| **舒适区设计（Comfort Zone）** | 将关键内容放置在距焦平面较近的深度范围内（约 0.7–3m），减小冲突量 | 已广泛应用 |
| **景深模糊（Depth-of-Field Blur）** | 模拟真实光学散焦效果，为调节系统提供正确线索 | 部分应用 |
| **变焦显示（Varifocal Display）** | 机械或电子方式动态改变显示焦平面，使其匹配注视深度 | 研究/原型阶段（如 Meta Half Dome） |
| **多焦面显示（Multifocal Display）** | 同时提供多个焦平面，覆盖不同深度 | 研究阶段 |
| **光场显示（Light Field Display）** | 重建完整光场，从根本上解决 VAC | 早期研究 |
| **全息显示（Holographic Display）** | 利用衍射光学重建波前，提供正确的调节信息 | 早期研究 |

### 设计建议

在当前硬件条件下，XR 内容设计者可以通过以下方式减轻 VAC 的影响：

- 避免将关键交互元素放置在距用户过近（< 0.5m）或过远（> 10m）的位置
- 利用 UI 设计引导用户注视舒适深度范围内的内容
- 对超出舒适区的内容施加适当的景深模糊
- 减少需要频繁在不同深度之间切换注视的场景设计
- 在长时间使用场景中提供休息提示
