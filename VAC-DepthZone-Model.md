# VAC 深度分区模型 (VAC-DepthZone Model)

## 一、概述

### 1.1 目标

基于 VAC（Vergence-Accommodation Conflict，辐辏-调节冲突）的光学原理与人因文献，为 XR 环境中的虚拟内容建立一套**深度方向**的分区体系。对于每个分区，明确：

- **深度边界**（这个区域从多远到多远）
- **视觉感受**（用户在这个深度看东西是什么体验）
- **内容类型策略**（适合放什么类型的 UI）
- **视觉属性策略**（字号、对比度、色彩等怎么设计）
- **交互行为策略**（适合什么交互方式）
- **时间策略**（用户可以在这个深度持续注视多久）

### 1.2 与 ErgoSpatial 模型的关系

本模型是 ErgoSpatial 肩关节力矩模型的**独立平行维度**。

| 维度 | 模型 | 核心度量 | 回答的问题 |
| :--- | :--- | :--- | :--- |
| **空间位置（手臂可达性）** | ErgoSpatial Model | 归一化力矩 (0~1) | 手臂够到这个位置有多累？ |
| **深度距离（视觉舒适性）** | **VAC-DepthZone Model（本文档）** | 屈光度冲突量 ΔD | 眼睛看这个深度有多不舒服？ |

两套系统各自独立运行，最终通过交叉查询来获得对一个空间点的完整评估。例如：一个点可能在力矩模型中处于"舒适区"，但在 VAC 模型中处于"容忍区"——此时设计策略应取两者中更保守的建议。

### 1.3 参数化设计

本模型的核心输入参数是头显的**光学焦平面距离 F**（单位：米）。所有分区边界均由 F 自动推导，无需针对特定设备硬编码。更换设备时只需修改 F 值，整套分区体系自动重新计算。

## 二、光学原理

### 2.1 屈光度（Diopter）

人眼晶状体的调节量以**屈光度（Diopter, D）**为单位度量，定义为观看距离（米）的倒数：

```
D = 1 / d
```

其中 d 为物体到眼睛的距离（米）。

| 距离 d (m) | 屈光度 D | 说明 |
| :--- | :--- | :--- |
| 0.25 | 4.00 D | 近距阅读 |
| 0.50 | 2.00 D | 桌面工作 |
| 1.00 | 1.00 D | 手臂前方 |
| 1.30 | 0.77 D | AVP 焦平面（近似值） |
| 2.00 | 0.50 D | 室内中距 |
| 5.00 | 0.20 D | 室内远处 |
| ∞ | 0.00 D | 无穷远（平行光） |

**关键特征：** 屈光度与距离是非线性关系。近处的微小距离变化对应巨大的屈光度变化，远处的大幅距离变化仅对应微小的屈光度变化。这直接决定了分区在物理空间中的宽度分布。

### 2.2 VAC 冲突量

在固定焦平面头显中：

- **调节需求（Accommodation demand）**：始终锁定在焦平面距离 F，即 D_A = 1/F
- **辐辏需求（Vergence demand）**：随虚拟物体距离 d 变化，即 D_V = 1/d

VAC 冲突量定义为二者的屈光度差值：

```
ΔD = |D_V - D_A| = |1/d - 1/F|
```

当 d = F 时，ΔD = 0，冲突为零——这是唯一完全没有 VAC 的深度。

### 2.3 近侧冲突与远侧冲突

以焦平面 F 为分界点，冲突分为两类：

**近侧冲突（Crossed disparity）**：虚拟物体比焦平面更近（d < F）

```
ΔD_near = 1/d - 1/F    （d < F 时为正值）
```

此时双眼会聚度大于调节焦点，眼睛处于"过度会聚"状态。

**远侧冲突（Uncrossed disparity）**：虚拟物体比焦平面更远（d > F）

```
ΔD_far = 1/F - 1/d    （d > F 时为正值）
```

此时双眼会聚度小于调节焦点，眼睛处于"会聚不足"状态。

**远侧冲突存在天然上限：** 当 d → ∞ 时，ΔD_far → 1/F = D_A。这意味着远侧的最大可能冲突量等于焦平面的屈光度值。例如 F = 1.3m 时，远侧冲突永远不超过 0.77D。

近侧冲突无上限：当 d → 0 时，ΔD_near → ∞。

### 2.4 不对称性原理

VAC 的不适感在近侧和远侧并不对称。大量人因研究（Shibata et al., 2011; Hoffman et al., 2008）一致表明：

> **在相同的屈光度冲突量下，近侧冲突（crossed disparity）比远侧冲突（uncrossed disparity）引起更强的不适感。**

生理解释：

1. **调节-辐辏联动方向冲突**：近侧冲突要求辐辏向近处会聚，但调节系统被锁定在较远的焦平面，这迫使两个系统做相反方向的努力，耗费更多神经协调资源。
2. **调节静息位（Tonic accommodation）**：人眼的调节静息位约在 1.0–1.5m（约 0.67–1.0D），与多数头显焦平面接近。远侧物体要求调节放松（接近静息状态），代价小；近侧物体要求调节收紧（偏离静息状态），代价大。
3. **融合储备不对称**：人眼的正向融合储备（convergence reserve）通常小于负向融合储备（divergence reserve），近侧冲突更容易逼近融合极限。

基于 Shibata et al.（2011）的实验数据，远侧的舒适阈值约为近侧的 **1.5 倍**。本模型将此不对称比率定义为可配置参数：

```
k_asymmetry = 1.5   # 远侧阈值 = 近侧阈值 × k_asymmetry
```

## 三、分区阈值推导

### 3.1 文献依据

本模型的阈值来源于以下核心文献的交叉验证：

| 文献 | 核心发现 |
| :--- | :--- |
| **Hoffman et al. (2008)** | 屈光度冲突超过约 0.4D 时，立体视觉性能（融合速度、立体锐度）开始显著下降 |
| **Shibata et al. (2011)** | 系统测量了不同冲突量下的主观不适感。舒适区边界：crossed ~0.4D, uncrossed ~0.6D；可容忍边界更宽 |
| **Kramida (2016)** | 综述总结：<0.3D 几乎无感知，0.3–0.6D 可注意到但可接受，>0.6D 明显不适 |
| **Percival's Zone of Comfort** | 经典验光学准则，舒适辐辏范围约为融合储备的中间 1/3，换算约 ±0.3–0.5D |
| **Shibata et al. (2011) 不对称性数据** | crossed 方向的不适感评分显著高于 uncrossed 方向，比率约 1.5:1 |

### 3.2 阈值定义

综合上述文献，定义三个关键阈值（近侧基准值）：

| 阈值 | 近侧 (Crossed) | 远侧 (Uncrossed) | 感知意义 |
| :--- | :--- | :--- | :--- |
| **T₁ (感知阈)** | 0.2 D | 0.3 D | 低于此值，绝大多数用户无法感知到任何冲突 |
| **T₂ (舒适阈)** | 0.4 D | 0.6 D | 低于此值，长时间使用无明显不适 |
| **T₃ (容忍阈)** | 0.7 D | 1.0 D | 超过此值，短时间内即出现显著不适和视觉疲劳 |

远侧阈值 = 近侧阈值 × k_asymmetry（1.5），并取整到 0.1D 精度。

**重要说明：** 这些阈值是群体统计的中位数水平。个体差异可达 ±30%（年轻人耐受度更高，老年人调节能力衰退后冲突感知可能变化）。T₁、T₂、T₃ 和 k_asymmetry 均为可配置参数。

### 3.3 四分区体系

三个阈值自然划分出四个深度区域：

```
            近侧（d < F）                        远侧（d > F）
            ←——————————————— F ——————————————————→
 Zone D  |  Zone C  |  Zone B  | Zone A | Zone B |  Zone C  |  Zone D
  回避区    容忍区      舒适区    焦点区    舒适区     容忍区      回避区
 ΔD>0.7D  0.4~0.7D  0.2~0.4D  0~0.2D  0~0.3D   0.3~0.6D  0.6~1.0D  >1.0D
                                                              ↑
                                                    (远侧阈值更宽松)
```

| 分区 | 名称 | 近侧 ΔD 范围 | 远侧 ΔD 范围 |
| :--- | :--- | :--- | :--- |
| **Zone A** | 焦点区 (Focal Sweet Spot) | 0 — 0.2 D | 0 — 0.3 D |
| **Zone B** | 舒适区 (Comfort Zone) | 0.2 — 0.4 D | 0.3 — 0.6 D |
| **Zone C** | 容忍区 (Tolerance Zone) | 0.4 — 0.7 D | 0.6 — 1.0 D |
| **Zone D** | 回避区 (Avoidance Zone) | > 0.7 D | > 1.0 D |

## 四、分区边界计算

### 4.1 屈光度冲突量到物理距离的转换

给定焦平面距离 F 和屈光度冲突阈值 ΔD，可计算对应的物理距离边界：

**近侧边界**（物体比焦平面更近）：

```
d_near = F / (1 + ΔD × F)
```

**远侧边界**（物体比焦平面更远）：

```
d_far = F / (1 - ΔD × F)

# 注意：当 ΔD × F ≥ 1 时，d_far → ∞
# 即该冲突量在远侧不可能达到，区域延伸到无穷远
```

### 4.2 参数化边界公式（伪代码）

```python
# ===== 可配置参数 =====

F = 1.3                # 焦平面距离（米），设备相关
k_asymmetry = 1.5      # 远/近不对称比率

# 近侧阈值（基准）
T1_near = 0.2          # 感知阈 (D)
T2_near = 0.4          # 舒适阈 (D)
T3_near = 0.7          # 容忍阈 (D)

# 远侧阈值（= 近侧 × k_asymmetry）
T1_far = T1_near * k_asymmetry    # 0.3 D
T2_far = T2_near * k_asymmetry    # 0.6 D
T3_far = T3_near * k_asymmetry    # 1.05 D → 取 1.0 D


# ===== 边界计算函数 =====

def near_boundary(F, delta_D):
    """近侧：给定冲突量，返回物理距离"""
    return F / (1.0 + delta_D * F)

def far_boundary(F, delta_D):
    """远侧：给定冲突量，返回物理距离（可能为无穷大）"""
    denominator = 1.0 - delta_D * F
    if denominator <= 0:
        return float('inf')    # 该冲突量在远侧永远达不到
    return F / denominator


# ===== 计算所有边界 =====

# Zone A（焦点区）边界
zone_A_near = near_boundary(F, T1_near)    # 内边界（最近）
zone_A_far  = far_boundary(F, T1_far)      # 外边界（最远）

# Zone B（舒适区）边界
zone_B_near = near_boundary(F, T2_near)
zone_B_far  = far_boundary(F, T2_far)

# Zone C（容忍区）边界
zone_C_near = near_boundary(F, T3_near)
zone_C_far  = far_boundary(F, T3_far)

# Zone D（回避区）：Zone C 之外的所有深度
# zone_D_near: 0 — zone_C_near
# zone_D_far:  zone_C_far — ∞
```

### 4.3 示例计算：Apple Vision Pro（F ≈ 1.3m）

D_focal = 1/1.3 ≈ 0.769 D

| 边界 | 近侧距离 | 远侧距离 |
| :--- | :--- | :--- |
| T₁ 边界（焦点区↔舒适区） | **1.03 m** | **2.13 m** |
| T₂ 边界（舒适区↔容忍区） | **0.86 m** | **5.91 m** |
| T₃ 边界（容忍区↔回避区） | **0.68 m** | **∞**（注） |

> **注：** F = 1.3m 时，远侧最大冲突量为 1/F = 0.77D < T₃_far = 1.0D，因此远侧不存在回避区。即使看向无穷远处，冲突量也仅为 0.77D，仍在容忍范围内。

**物理深度范围：**

| Zone | 近侧范围 | 远侧范围 | 合计宽度 |
| :--- | :--- | :--- | :--- |
| **A 焦点区** | 1.03m — 1.30m（0.27m 厚） | 1.30m — 2.13m（0.83m 厚） | **1.10m** |
| **B 舒适区** | 0.86m — 1.03m（0.17m 厚） | 2.13m — 5.91m（3.78m 厚） | **3.95m** |
| **C 容忍区** | 0.68m — 0.86m（0.18m 厚） | 5.91m — ∞ | **∞** |
| **D 回避区** | 0m — 0.68m | 不存在 | — |

**关键洞察：**

1. **近侧分区极窄，远侧分区极宽。** 这是屈光度与距离的非线性关系决定的。近处 0.17m 的物理距离变化就跨越了一个分区，而远处跨越同一分区需要移动数米。
2. **焦点区以焦平面为中心，约 1.0~2.1m。** 这恰好覆盖了手臂操作范围和近距对话距离。
3. **远侧回避区可能不存在。** 取决于 F 和 T₃_far 的关系。当 F × T₃_far ≥ 1 时，远侧回避区消失。

### 4.4 示例计算：Meta Quest 3（F ≈ 1.5m）

D_focal = 1/1.5 ≈ 0.667 D

| 边界 | 近侧距离 | 远侧距离 |
| :--- | :--- | :--- |
| T₁ 边界 | **1.15 m** | **2.73 m** |
| T₂ 边界 | **0.94 m** | **15.0 m** |
| T₃ 边界 | **0.73 m** | **∞** |

Quest 3 焦平面更远，远侧分区更加宽广，远侧回避区同样不存在（最大远侧冲突仅 0.667D < 1.0D）。

### 4.5 示例计算：F = 2.0m

D_focal = 1/2.0 = 0.500 D

远侧最大冲突量（无穷远处）= 1/F = 0.500D。这个值不仅小于 T₃_far（1.0D），甚至小于 T₂_far（0.6D），意味着**远侧的容忍区和回避区都不存在**——无论多远的虚拟物体，其冲突量都不超过舒适区上限。

| 边界 | 近侧距离 | 远侧距离 |
| :--- | :--- | :--- |
| T₁ 边界（焦点区↔舒适区） | **1.43 m** | **5.00 m** |
| T₂ 边界（舒适区↔容忍区） | **1.11 m** | **∞**（注 1） |
| T₃ 边界（容忍区↔回避区） | **0.83 m** | **∞**（注 2） |

> **注 1：** T₂_far × F = 0.6 × 2.0 = 1.2 ≥ 1，分母为负，远侧舒适区延伸至无穷远。即无穷远处的冲突（0.5D）仍低于远侧舒适阈（0.6D）。
>
> **注 2：** T₃_far × F = 1.0 × 2.0 = 2.0 ≥ 1，远侧容忍区同样不存在。

**物理深度范围：**

| Zone | 近侧范围 | 远侧范围 | 合计宽度 |
| :--- | :--- | :--- | :--- |
| **A 焦点区** | 1.43m — 2.00m（0.57m 厚） | 2.00m — 5.00m（3.00m 厚） | **3.57m** |
| **B 舒适区** | 1.11m — 1.43m（0.32m 厚） | 5.00m — ∞ | **∞** |
| **C 容忍区** | 0.83m — 1.11m（0.28m 厚） | 不存在 | **0.28m** |
| **D 回避区** | 0m — 0.83m | 不存在 | — |

**关键洞察：**

1. **远侧极度友好。** F = 2.0m 时，整个远侧空间（2m 到无穷远）仅覆盖 Zone A 和 Zone B，远侧不适几乎不可能发生。这对需要展示大量远景内容的应用（如虚拟影院、数据可视化大屏）非常有利。
2. **焦点区显著扩大。** 对比 F = 1.3m 时焦点区仅 1.10m 宽，F = 2.0m 时焦点区扩展到 3.57m，核心工作区域的设计空间大幅增加。
3. **近侧分区变宽但仍有限。** 近侧各分区宽度约为 F = 1.3m 时的 1.5~2 倍，设计余裕增大，但 0.83m 以内仍为回避区。
4. **设计取舍明确。** 焦平面越远，远侧越宽松，但近距交互（如手持操作 <1m）面临更大的 VAC 压力。适合以观看和远场交互为主的场景，不适合需要大量近距直接触摸的应用。

**三款设备横向对比：**

| 指标 | AVP (F=1.3m) | Quest 3 (F=1.5m) | F=2.0m |
| :--- | :--- | :--- | :--- |
| D_focal | 0.769 D | 0.667 D | 0.500 D |
| 焦点区范围 | 1.03 — 2.13m | 1.15 — 2.73m | 1.43 — 5.00m |
| 焦点区宽度 | 1.10m | 1.58m | 3.57m |
| 远侧最大冲突 | 0.77D (Zone C) | 0.67D (Zone C) | 0.50D (Zone B) |
| 远侧最高分区 | 容忍区 | 容忍区 | **舒适区** |
| 近侧回避区起点 | < 0.68m | < 0.73m | < 0.83m |

### 4.6 焦平面距离对分区格局的影响

```
焦平面距离 F 越大：
  → D_focal = 1/F 越小
  → 远侧最大冲突越小（更不容易出现远侧不适）
  → 近侧分区稍微变宽（但仍然很窄）
  → 远侧分区大幅变宽

焦平面距离 F 越小：
  → D_focal = 1/F 越大
  → 近侧分区更窄（设计空间更小）
  → 远侧可能出现回避区（当 F 足够小时）
```

## 五、各分区视觉感受

### 5.1 Zone A — 焦点区 (Focal Sweet Spot)

**屈光度冲突：** 0 — 0.2D（近）/ 0 — 0.3D（远）

**主观视觉感受：**

- 视觉完全自然，与裸眼观看真实物体几乎无差异
- 双眼融合轻松、无意识完成
- 虚拟物体的深度感知准确，大小恒常性正常
- 焦点切换（在此区域内不同深度间转移视线）流畅无迟滞
- 文字阅读清晰度高，不会因调节误差导致边缘模糊
- 无任何视觉疲劳症状

**感知参考：** 类似于裸眼看 1~2 米外的实物。

### 5.2 Zone B — 舒适区 (Comfort Zone)

**屈光度冲突：** 0.2 — 0.4D（近）/ 0.3 — 0.6D（远）

**主观视觉感受：**

- 静态观看时几乎无不适，冲突可被感知但不构成干扰
- 长时间（>30 分钟）持续注视后，可能出现轻微的眼部紧张感
- 细小文字的阅读清晰度略有下降——晶状体在调节焦平面和辐辏深度之间微幅振荡，导致偶发的瞬间模糊（micro-fluctuation）
- 深度感知基本准确，但精密的深度判断（如两个物体的前后顺序）可能出现微小误差
- 从焦点区快速切换到此区域时，需要约 0.5~1 秒的再适应时间

**感知参考：** 类似于戴了一副度数略有偏差的眼镜看东西——可以正常使用，但细看时感觉"不是最舒服"。

### 5.3 Zone C — 容忍区 (Tolerance Zone)

**屈光度冲突：** 0.4 — 0.7D（近）/ 0.6 — 1.0D（远）

**主观视觉感受：**

- 不适感明确可感知，不再是"略有偏差"而是"明显不对劲"
- 持续注视 3~5 分钟后出现：眼部酸胀、干涩感增加、眨眼频率上升
- 文字阅读变得费力——晶状体调节振荡幅度增大，小字号文字持续处于轻微失焦状态
- 深度感知出现系统性偏差：近侧物体倾向于被知觉为比实际更近，远侧物体倾向于被知觉为比实际更远
- 在此深度停留过久后切回焦点区时，需要明显的再适应时间（1~3 秒），期间可能感到短暂的眩晕
- 调节-辐辏联动被强力解耦，部分用户（约 15~25%）可能触发轻度晕动症状

**感知参考：** 类似于长时间使用 3D 电影眼镜观看深度极端的场景——能看，但会累。

### 5.4 Zone D — 回避区 (Avoidance Zone)

**屈光度冲突：** >0.7D（近）/ >1.0D（远）

**主观视觉感受：**

- 调节-辐辏系统严重失调，部分用户可能无法完成双眼融合（出现复视或交替抑制）
- 注视数秒即出现：锐利的眼后疼痛、前额压迫感、恶心前驱症状
- 文字几乎不可阅读——调节系统在焦平面和辐辏深度之间剧烈振荡，画面持续模糊闪烁
- 深度感知严重失真：物体可能出现大小异常（micropsia/macropsia）和距离判断严重偏差
- 离开此深度后需要较长恢复时间（5~15 秒），期间其他深度的物体也会暂时变得不舒适
- 高敏感个体可能触发前庭-视觉冲突，出现明确的眩晕和恶心

**感知参考：** 类似于交叉眼看近处手指时的不适感——大脑在努力协调两个矛盾的信号，既累又不适。

## 六、各分区设计策略

### 6.1 策略总览

| 策略维度 | Zone A 焦点区 | Zone B 舒适区 | Zone C 容忍区 | Zone D 回避区 |
| :--- | :--- | :--- | :--- | :--- |
| **内容类型** | 核心工作内容 | 辅助参考内容 | 环境/通知信息 | 仅氛围装饰 |
| **文字密度** | 高密度可 | 中密度 | 低密度 | 避免文字 |
| **最小字号** | 常规（可小至等效 12pt） | 放大 1.2× | 放大 1.5× | 放大 2× 以上 |
| **对比度** | 标准（4.5:1 WCAG AA） | 增强（≥ 7:1） | 高对比（≥ 10:1） | 极高对比 |
| **色彩饱和度** | 全饱和度 | 全饱和度 | 降低 20~30% | 降低 50%+ |
| **交互方式** | 所有模态 | 所有模态 | 仅快速操作 | 避免交互 |
| **持续注视时长** | 无限制 | ≤ 30 分钟 | ≤ 3 分钟 | ≤ 10 秒 |
| **景深模糊** | 不需要 | 可选 | 推荐施加 | 强制施加 |

### 6.2 Zone A — 焦点区设计策略

#### 内容类型

焦点区是**核心工作区域**，应承载用户需要长时间注视和频繁交互的所有关键内容：

| 适合的内容 | 理由 |
| :--- | :--- |
| 主要文档/文本编辑器 | 需要长时间阅读和精细文字操作 |
| 核心工作面板（代码编辑、设计画布等） | 需要持续高精度视觉关注 |
| 虚拟键盘 | 需要高频、高精度连续注视 |
| 主要导航菜单和工具栏 | 需要频繁扫视和选择 |
| 对话界面（聊天、视频通话） | 需要持续注视对方面部/文字 |
| 精细 3D 模型操作（建模、雕刻） | 需要准确的深度感知和精密操作 |

#### 视觉属性

- **字号：** 无特殊限制，可使用等效 12pt 及以上的字号。密集文字排版可正常使用
- **对比度：** 遵循 WCAG AA 标准（正文 4.5:1，大字 3:1）即可满足
- **色彩：** 全色域可用，饱和色彩的深度感知在此区域最为准确
- **细节密度：** 可使用高信息密度的界面布局（如多栏排版、密集图表）
- **透明度/毛玻璃效果：** 可正常使用，调节准确性保证了通过半透明层仍能清晰辨认背后内容
- **动画：** 无特殊限制，帧率和运动幅度按常规设计即可

#### 交互行为

- 所有交互模态均可使用：直接触摸、眼动追踪、手势、语音
- 适合精细拖拽、双指缩放、连续书写等高精度操作
- 适合需要精确深度判断的交互（如在 3D 空间中拾取和放置物体）
- 焦点区内不同深度间的视线切换可快速完成，因此多层 UI（如弹出菜单、下拉列表）在此区域内安全

#### 时间策略

- **无时间限制**，用户可在此区域内持续工作数小时
- 仍建议每 45~60 分钟遵循通用用眼卫生准则（"20-20-20 规则"），但这不是 VAC 导致的需求

### 6.3 Zone B — 舒适区设计策略

#### 内容类型

舒适区是**辅助参考区域**，承载不需要持续注视但需要不时查看的内容：

| 适合的内容 | 理由 |
| :--- | :--- |
| 参考文档/数据面板 | 偶尔查看，不需要逐字阅读 |
| 媒体播放器（视频、图片浏览） | 中等注视时长，视觉负荷适中 |
| 辅助工具面板（属性检查器、图层面板） | 交互频率低于核心面板 |
| 地图/概览视图 | 提供空间感知，不需要精细阅读 |
| 日程/时钟/天气等信息卡片 | 扫一眼即获取信息 |
| 3D 场景中的中距物体 | 需要一定的深度感知准确性 |

#### 视觉属性

- **字号：** 建议将基准字号放大 **1.2 倍**（等效 14~15pt 起步）。此区域的微幅调节振荡会降低小字号的清晰度
- **对比度：** 建议提升至 **WCAG AAA 级（7:1）** 以上，补偿微弱的焦点模糊带来的辨认度下降
- **色彩：** 全饱和度仍可使用，但避免依赖细微的色差传递信息（如仅用相近色区分状态）
- **细节密度：** 降低为中等密度。避免密集排版，增加行间距和元素间距
- **透明度/毛玻璃效果：** 谨慎使用——调节振荡可能使半透明区域的背景内容出现闪烁感
- **动画：** 避免快速的前后（深度方向）运动，这会迫使调节系统频繁重新适应。横向/纵向运动无特殊限制
- **景深模糊：** 可选择性地为此区域内容施加轻微景深模糊，向调节系统暗示"这里不是焦点"，减少调节-辐辏的拉锯

#### 交互行为

- 所有交互模态可用，但应优先使用**低精度、快完成**的交互
- 适合：单次点按、开关切换、标签页切换、简单的列表滚动
- 需注意：精细拖拽操作的精度会略有下降（受微幅调节振荡影响），如果必须在此区域放置滑块等精细控件，应增大控件尺寸和可触摸范围
- 建议将需要超过 30 秒持续操作的交互控件移入 Zone A

#### 时间策略

- **建议持续注视不超过 30 分钟**
- 适合"扫一眼——操作——回到焦点区"的交互模式
- 如果用户在此区域停留过久（>20 分钟），可考虑显示柔和提示引导其将关注点移回焦点区

### 6.4 Zone C — 容忍区设计策略

#### 内容类型

容忍区是**外围信息区域**，仅承载扫一眼即可获取的低密度信息和极低频交互元素：

| 适合的内容 | 理由 |
| :--- | :--- |
| 系统通知/提示气泡 | 出现 → 瞥一眼 → 消失，注视时间极短 |
| 环境信息（时间、电量、网络状态） | 图标化呈现，无需阅读 |
| 3D 场景中的远景/背景物体 | 提供空间感，不需要精细辨认 |
| 音乐播放器状态（专辑封面、进度条） | 视觉信息以图形为主，文字极少 |
| 最小化的窗口/应用图标 | 仅需辨认轮廓，不需要阅读标题 |
| 空间锚点/路标 | 辅助空间导航，视觉简洁 |

#### 视觉属性

- **字号：** 建议将基准字号放大 **1.5 倍以上**（等效 18pt 起步）。此区域不适合呈现段落文字，仅使用关键词、数字或短标签
- **对比度：** 显著提升至 **10:1 以上**，确保在调节失焦状态下仍可辨认
- **色彩：** 建议降低饱和度 **20~30%**。在此深度，高饱和色彩可能因色差（chromatic aberration）加剧而产生色边（color fringing），降低饱和度可缓解
- **细节密度：** 极低密度。大图标、粗线条、高简洁度。使用图形符号替代文字
- **透明度/毛玻璃效果：** 避免使用。调节振荡会使半透明内容严重闪烁
- **动画：** 严格避免深度方向运动。横向运动应缓慢、平滑。突然出现（pop-in）的动画可以接受——短暂的注意力吸引后，用户视线应回到焦点区
- **景深模糊：** **推荐施加**。为容忍区内容添加与冲突量成正比的景深模糊，达到两个效果：(1) 减弱调节系统的无效努力（模糊的内容不会强烈驱动调节反射），(2) 向用户传达"这不是你应该持续注视的区域"的视觉暗示

#### 交互行为

- **优先使用远场交互模态**：眼动+捏合（Gaze & Pinch）、射线（Ray-casting）、语音
- 即使物理上可达（在 ErgoSpatial 力矩模型的可达范围内），也应避免要求用户在此深度进行直接触摸——VAC 导致的深度感知偏差会降低触摸准确性
- 仅限以下交互类型：
  - 单次点按确认/消除（如关闭通知）
  - 大幅挥动手势（如翻页、挥手呼出/关闭）
  - 语音命令
- **严禁：** 精细拖拽、文字输入、精密选择

#### 时间策略

- **持续注视建议不超过 3 分钟**
- 内容应设计为"一瞥即走"模式：用户瞥一眼（<5 秒）获取信息，视线立即返回焦点区
- 如果用户持续注视此区域超过 30 秒，系统应考虑将该内容自动移动或引导至焦点区
- 在此区域停留后切回焦点区时，留 1~3 秒的再适应缓冲（可通过渐变动画实现平滑过渡）

### 6.5 Zone D — 回避区设计策略

#### 内容类型

回避区是**纯氛围区域**，不应承载任何需要用户主动获取的信息：

| 适合的内容 | 理由 |
| :--- | :--- |
| 环境装饰（天空盒、远景建筑、粒子效果） | 纯视觉氛围，不含任何信息 |
| 空间音频源的视觉锚点 | 指示声音方向，不需要精细辨认 |
| 极远处的场景过渡元素 | 场景加载时的远景填充 |
| 近处的"禁入"视觉屏障 | 警告用户不要靠得更近（仅近侧回避区） |

> **注意：** 对于多数焦平面在 1.0m 以上的头显，远侧回避区不存在（见 4.3 节），因此 Zone D 策略主要针对**近侧回避区**（距用户过近的位置）。

#### 视觉属性

- **字号：** 如果必须出现文字（强烈不建议），字号放大至基准的 **2 倍以上**，仅限 1~2 个单词
- **对比度：** 对于装饰性内容，反而应**降低对比度**，使其自然融入背景，避免吸引用户注视。对于警告性内容（如近侧禁入屏障），使用极高对比度
- **色彩：** 大幅降低饱和度（**50% 以上**），以减弱视觉吸引力和色差效应
- **细节密度：** 最低。仅保留轮廓和大色块，不需要任何内部细节
- **动画：** 仅限缓慢、低频率的环境动画（如云层流动、光影变化）。绝对禁止突然的动画——可能触发前庭-视觉冲突
- **景深模糊：** **强制施加**。此区域的所有内容应进行明显的景深模糊处理，从光学和知觉两个层面同时遏制调节反射的误导

#### 交互行为

- **应避免在此区域放置任何交互元素**
- 如果存在极特殊的需求（如紧急警告需要用户确认），应使用语音交互或将确认按钮弹出至焦点区
- 用户不应被引导注视此区域——如果检测到用户视线在此区域停留，应将其注意力引导至焦点区（如通过焦点区内容的视觉提示）

#### 时间策略

- **持续注视建议不超过 10 秒**
- 不应存在任何让用户主动选择注视此区域的理由
- 如果检测到用户视线在回避区停留超过 5 秒，系统应触发柔和提示（如焦点区 UI 元素的微动画）引导视线回归

## 七、跨区域设计原则

### 7.1 深度过渡设计

当用户视线需要在不同分区间切换时，应遵循以下原则：

**7.1.1 相邻区跳转优先**

```
推荐：Zone A ↔ Zone B ↔ Zone C（逐级过渡）
避免：Zone A ↔ Zone C 或 Zone A ↔ Zone D（跨级跳转）
```

跨级深度跳转迫使调节系统在短时间内大幅调整，即使两个区域各自尚在可接受范围内，频繁的大幅跳转本身也会导致疲劳（类似于频繁对焦的相机镜头会加速马达磨损）。

**7.1.2 过渡动画策略**

| 过渡类型 | 推荐策略 |
| :--- | :--- |
| A ↔ B（小幅跳转） | 可瞬间切换，无需特殊处理 |
| A ↔ C 或 B ↔ C（中幅跳转） | 使用 200~400ms 的深度渐变动画，让内容沿深度方向平滑移入/移出 |
| 任何 → D 或 D → 任何（大幅跳转） | 避免。如果不可避免，使用交叉淡入淡出（cross-fade）而非深度运动 |

**7.1.3 深度切换频率限制**

- 同一 UI 流程中，不应要求用户在 10 秒内进行超过 3 次跨区深度切换
- 如果交互流程天然需要频繁切换深度（如 3D 场景编辑），应将所有交互控件聚合到 Zone A 内，仅让被编辑的场景内容分布在不同深度

### 7.2 景深模糊策略

景深模糊（Depth-of-Field blur）是缓解 VAC 的有效渲染手段。核心思路：为不在用户注视深度的内容施加与屈光度差异成正比的模糊：

```python
def compute_dof_blur(d_object, d_gaze, F):
    """
    计算某个深度 d_object 上的内容应该施加多少景深模糊。
    
    d_object: 该内容所在深度（米）
    d_gaze:   用户当前注视深度（米），由眼动追踪提供
    F:        焦平面距离（米）
    """
    # 物体与注视点的屈光度差异
    delta_D_gaze = abs(1.0/d_object - 1.0/d_gaze)
    
    # 物体与焦平面的屈光度差异（VAC 冲突量）
    delta_D_vac = abs(1.0/d_object - 1.0/F)
    
    # 模糊量 = 两者中较大的（模拟真实光学 + 缓解 VAC）
    blur_amount = max(delta_D_gaze, delta_D_vac)
    
    # 归一化到 0~1 范围（1D 屈光度差异对应全模糊）
    blur_normalized = min(1.0, blur_amount / 1.0)
    
    return blur_normalized
```

**分区默认模糊策略：**

| Zone | 用户注视此区域时的自身模糊 | 用户注视其他区域时的模糊 |
| :--- | :--- | :--- |
| A | 无 | 轻微（按与注视点的距离差计算） |
| B | 无 | 轻度至中度 |
| C | 可选施加轻微模糊（缓解 VAC） | 中度至强模糊 |
| D | 强制施加模糊 | 强模糊 |

### 7.3 与 ErgoSpatial 模型的联合评估

当一个空间点同时经过两个模型评估时，取**更保守的建议**：

```python
def joint_evaluation(P_target, user_model, F):
    """
    联合评估一个空间点的综合设计建议。
    """
    # ErgoSpatial 评估（手臂疲劳维度）
    ergo_result = evaluate_spatial_point(P_target, user_model)
    
    # VAC-DepthZone 评估（视觉舒适维度）
    d = distance_from_eye(P_target, user_model)
    vac_result = evaluate_vac_zone(d, F)
    
    # 取更保守（更差）的分区等级
    # Zone 等级映射：A=0, B=1, C=2, D=3
    ergo_level = zone_to_level(ergo_result["zone"])
    vac_level  = zone_to_level(vac_result["zone"])
    
    final_level = max(ergo_level, vac_level)
    
    return {
        "ergo_zone": ergo_result["zone"],
        "vac_zone": vac_result["zone"],
        "effective_zone": level_to_zone(final_level),
        "limiting_factor": "视觉舒适性" if vac_level > ergo_level 
                           else "手臂疲劳",
        "ergo_details": ergo_result,
        "vac_details": vac_result
    }
```

**常见的联合评估场景：**

| 场景 | ErgoSpatial 评估 | VAC 评估 | 有效分区 | 瓶颈因素 |
| :--- | :--- | :--- | :--- | :--- |
| 正前方 0.5m 处 | 过渡区（手臂需前伸） | 回避区（VAC 冲突极大） | **回避区** | VAC |
| 正前方 1.2m 处 | 高负荷区（接近臂长极限） | 焦点区（接近焦平面） | **高负荷区** | 手臂疲劳 |
| 正前方 3.0m 处 | 不可达 | 舒适区（远侧轻微冲突） | **不可达** | 物理距离 |
| 胸前 0.9m 处，偏下方 | 舒适区（自然手臂位置） | 舒适区（接近焦平面） | **舒适区** | 无瓶颈 ✓ |

## 八、分区评估伪代码

```python
# ===== 可配置参数 =====

F = 1.3               # 焦平面距离（米）
k_asymmetry = 1.5     # 远/近不对称比率

# 近侧阈值（屈光度）
T1_NEAR = 0.2         # 感知阈
T2_NEAR = 0.4         # 舒适阈
T3_NEAR = 0.7         # 容忍阈

# 远侧阈值 = 近侧 × k_asymmetry
T1_FAR = T1_NEAR * k_asymmetry    # 0.3
T2_FAR = T2_NEAR * k_asymmetry    # 0.6
T3_FAR = T3_NEAR * k_asymmetry    # 1.05 → 取 1.0


def evaluate_vac_zone(d, F):
    """
    评估给定深度 d 处的 VAC 分区。
    
    输入:
        d:  虚拟物体到用户眼睛的距离（米），d > 0
        F:  头显焦平面距离（米）
    输出:
        字典，包含分区名称、冲突量、设计建议等
    """
    
    # 计算屈光度
    D_object = 1.0 / d
    D_focal  = 1.0 / F
    
    # 判断方向并计算冲突量
    if d < F:
        side = "near"    # 近侧冲突 (crossed disparity)
        delta_D = D_object - D_focal
        T1, T2, T3 = T1_NEAR, T2_NEAR, T3_NEAR
    elif d > F:
        side = "far"     # 远侧冲突 (uncrossed disparity)
        delta_D = D_focal - D_object
        T1, T2, T3 = T1_FAR, T2_FAR, T3_FAR
    else:
        # 恰好在焦平面上
        return {
            "zone": "Zone A — 焦点区",
            "delta_D": 0.0,
            "side": "focal_plane",
            "max_viewing_time": "无限制"
        }
    
    # 分区判定
    if delta_D <= T1:
        zone = "Zone A — 焦点区 (Focal Sweet Spot)"
        content_strategy = "核心工作内容，高密度信息，精细交互"
        visual_strategy = "标准字号，标准对比度，全色域"
        interaction_strategy = "所有模态，精细操作可用"
        max_viewing_time = "无限制"
        dof_blur = "不需要"
        
    elif delta_D <= T2:
        zone = "Zone B — 舒适区 (Comfort Zone)"
        content_strategy = "辅助参考内容，中等密度信息"
        visual_strategy = "字号 ×1.2，对比度 ≥7:1，谨慎使用透明效果"
        interaction_strategy = "所有模态，优先低精度快完成操作"
        max_viewing_time = "≤ 30 分钟"
        dof_blur = "可选"
        
    elif delta_D <= T3:
        zone = "Zone C — 容忍区 (Tolerance Zone)"
        content_strategy = "环境/通知信息，极低密度，图形为主"
        visual_strategy = "字号 ×1.5，对比度 ≥10:1，降低饱和度 20-30%"
        interaction_strategy = "仅远场模态（眼动+捏合），仅快速操作"
        max_viewing_time = "≤ 3 分钟"
        dof_blur = "推荐施加"
        
    else:
        zone = "Zone D — 回避区 (Avoidance Zone)"
        content_strategy = "仅氛围装饰，不含信息"
        visual_strategy = "避免文字，降低饱和度 50%+，极低细节"
        interaction_strategy = "避免任何交互"
        max_viewing_time = "≤ 10 秒"
        dof_blur = "强制施加"
    
    return {
        "zone": zone,
        "delta_D": round(delta_D, 3),
        "side": side,
        "content_strategy": content_strategy,
        "visual_strategy": visual_strategy,
        "interaction_strategy": interaction_strategy,
        "max_viewing_time": max_viewing_time,
        "dof_blur": dof_blur,
        "distance_to_focal_plane": round(abs(d - F), 3)
    }


def compute_all_boundaries(F):
    """
    给定焦平面距离 F，计算所有分区的物理距离边界。
    """
    def near_bound(delta_D):
        return F / (1.0 + delta_D * F)
    
    def far_bound(delta_D):
        denom = 1.0 - delta_D * F
        return F / denom if denom > 0 else float('inf')
    
    return {
        "Zone A": {
            "near_inner": near_bound(T1_NEAR),
            "far_outer": far_bound(T1_FAR),
            "description": "焦点区"
        },
        "Zone B": {
            "near_inner": near_bound(T2_NEAR),
            "near_outer": near_bound(T1_NEAR),
            "far_inner": far_bound(T1_FAR),
            "far_outer": far_bound(T2_FAR),
            "description": "舒适区"
        },
        "Zone C": {
            "near_inner": near_bound(T3_NEAR),
            "near_outer": near_bound(T2_NEAR),
            "far_inner": far_bound(T2_FAR),
            "far_outer": far_bound(T3_FAR),
            "description": "容忍区"
        },
        "Zone D": {
            "near_outer": near_bound(T3_NEAR),
            "far_inner": far_bound(T3_FAR),
            "description": "回避区（近侧: 0~near_outer, 远侧: far_inner~∞）"
        },
        "focal_plane": F,
        "max_far_conflict": round(1.0/F, 3)
    }
```

## 十、设计检查清单

在完成 XR 界面设计后，可使用以下清单逐项检查 VAC 合规性：

### 10.1 内容放置检查

- [ ] 所有需要长时间阅读的文字内容是否位于 Zone A？
- [ ] 所有需要精细交互的控件是否位于 Zone A？
- [ ] 辅助参考面板是否位于 Zone A 或 Zone B？
- [ ] 通知和环境信息是否位于 Zone B 或 Zone C？
- [ ] Zone D 中是否不包含任何信息性内容？

### 10.2 视觉属性检查

- [ ] Zone B 中的文字是否比 Zone A 放大了至少 1.2 倍？
- [ ] Zone C 中的文字是否比 Zone A 放大了至少 1.5 倍？
- [ ] Zone B 的对比度是否达到 7:1？
- [ ] Zone C 的对比度是否达到 10:1？
- [ ] Zone C 和 Zone D 是否已施加景深模糊？

### 10.3 交互检查

- [ ] Zone C 中是否只存在"一击即走"的简单交互？
- [ ] Zone D 中是否不存在任何交互元素？
- [ ] 是否避免了需要用户在 Zone A 和 Zone C/D 之间频繁切换视线的交互流程？

### 10.4 时间检查

- [ ] 是否存在机制防止用户在 Zone C 持续注视超过 3 分钟？
- [ ] 是否存在机制防止用户在 Zone D 持续注视超过 10 秒？

## 十一、局限性与未来方向

### 11.1 当前局限

1. **阈值来源于群体统计中位数。** 个体差异（年龄、屈光不正、双眼视功能状况）可能导致个人的实际舒适阈与模型阈值偏差达 ±30%。
2. **假设焦平面固定。** 对于变焦显示（varifocal display）设备，焦平面实时变化，模型需要动态重新计算（但公式本身仍然适用——将 F 替换为实时焦平面距离即可）。
3. **未考虑内容因素对耐受度的影响。** 高沉浸感内容（如紧张的游戏场景）下用户对 VAC 的主观耐受度可能高于静态桌面工作场景。
4. **未考虑瞳距（IPD）的影响。** 瞳距影响辐辏角度计算，极端 IPD 的用户可能体验到略不同的冲突强度。

### 11.2 未来方向

1. **个性化校准：** 通过简短的视觉测试（如呈现不同深度的立体目标并收集主观评分），动态调整个人的 T₁、T₂、T₃ 阈值。
2. **变焦显示自适应：** 当设备支持变焦时，模型可反向使用——给定内容的目标深度，计算最优焦平面位置以最小化 VAC。
3. **累积疲劳建模：** 在时间维度上积分 VAC 冲突量，建立累积视觉疲劳指标，动态调整分区边界（随使用时间推移逐步收窄舒适区）。
4. **与 ErgoSpatial 模型的深度融合：** 构建统一的多维评估框架，输出最终的综合空间适用性评分。
