# XR 空间 UI 布局设计规范

## 基于人眼视野角与头部转动舒适度的多层决策框架

---

## 1. 概述

本规范为 XR 头显环境中虚拟交互内容的空间布局提供设计指导。基于**人眼视野角分区**、**头部转动舒适度分区**和**眼-头协同模型**三组独立的生理学基础数据，构建一个**多层决策框架**，指导从环境氛围设计到单个 UI 元素类型选择的全链路空间布局决策。

本规范采用**分层决策**而非**扁平融合**的方法论。三组基础数据并非在同一角度轴上取中间值合并为一张区域地图，而是在不同的设计决策层级上各自发挥主导作用——每种分区体系在其最相关的设计层使用原生阈值，避免跨体系强行对齐导致的远端偏差。

### 1.1 适用范围

| 属性 | 说明 |
|---|---|
| 平台 | XR 头显（通用，不绑定特定硬件） |
| 坐标参考 | Body-locked 和 World-locked UI（附带 Head-locked 建议） |
| 覆盖姿态 | 坐姿与站姿 |
| 维度 | 水平角度（Yaw）+ 垂直角度（Pitch），不含深度（Z 轴） |
| 内容类型 | 主交互面板、信息展示、通知提示、导航元素、媒体内容、环境氛围 |

### 1.2 坐标系定义

本规范采用以下坐标系约定：

- **原点**：用户头部中心位置
- **水平角度（Azimuth / Yaw）**：0° 为正前方，向左为负，向右为正，范围 -180° ~ +180°
- **垂直角度（Elevation / Pitch）**：以**自然静息视线**为 0° 基准，向上为正，向下为负
- **自然静息视线**：站姿约低于真实水平线 10°，坐姿约低于真实水平线 15°

> **为什么以自然静息视线为垂直基准？** 人在放松状态下的视线并非水平前视，而是自然下倾。以此为基准，能更准确地反映用户在 XR 环境中的真实感受，使得"0° 垂直"对应"毫无努力的自然注视方向"。

## 2. 基础数据：人眼视野角

人眼视野（Field of View）并非均匀的感知区域，而是从中心向外逐级退化的分层结构。

### 2.1 视野角功能分区

视网膜的中央区域——黄斑 (Macula)——覆盖注视点周围约 10° 半径，包含解剖学上的中央凹 (Fovea)、近中央凹 (Parafovea) 和周中央凹 (Perifovea) 三层同心结构，合称"中央视觉"。由于中央凹与近中央凹对 XR UI 设计的约束趋同（详见下方说明），本规范将两者合并为"中央凹区"，与周中央凹分别处理。超出黄斑区即进入周边视觉。

| 分区 | 角半径（从注视点） | 视觉能力 | XR 设计意义 |
|---|---|---|---|
| **中央凹区 (Foveal Zone)** | 0° ~ 5° | 最高视锐度，可分辨文字笔画与精细图形细节，颜色感知完整 | **精细阅读区**——适合正文文字、小型 UI 控件（按钮标签、输入光标、滑块刻度）。超出 ±5° 后无法逐字阅读 |
| **周中央凹 (Perifoveal)** | 5° ~ 10° | 视锐度降至中央凹区的约 30%；无法辨认正文字号的笔画，但可凭轮廓与对比度识别熟悉的大型元素 | 适合大号标题、熟悉图标、高对比度区块，无需眼跳即可判断元素类型。超出 ±10° 后无法识别具体内容（整个 0°–10° 构成**有效识别区**） |
| **近周边视觉 (Near Peripheral)** | 10° ~ 30° | 无法识别文字或图标的具体内容，仅可感知形状轮廓、色块与空间位置 | 扫视目标定位（"那边有个东西"）、UI 区块存在性感知。±30° 是不转头获取信息的实际极限（**功能视野边界**） |
| **中周边视觉 (Mid Peripheral)** | 30° ~ 60° | 静态细节几乎不可辨，以运动检测和明暗对比为主；颜色辨识退化 | 动态变化察觉（动画、闪烁、弹出）、空间方位感。±60° 以外仅对突发运动有反应（**运动感知边界**） |
| **远周边视觉 (Far Peripheral)** | 60° ~ 100°+ | 仅对突发运动和强烈明暗变化有反应；>60° 主要为单眼视野 | 威胁/事件警觉、空间存在感与包围感 |

> **关于分区的方向性**：以上分区为以注视点为圆心的**同心圆环结构**，在水平、垂直和对角方向上均成立——同一角半径处的视觉能力不随方向变化。方向性差异仅体现在视野的**物理外边界**上：水平方向单眼视野可达颞侧 ~100°、鼻侧 ~60°，而垂直方向仅为上方 ~60°、下方 ~70°–75°。这意味着远周边视觉区在垂直方向上更早到达物理极限。

> **关于中央凹区的合并**：解剖学上，0°–2° 的中央凹 (Fovea) 与 2°–5° 的近中央凹 (Parafovea) 在视网膜层结构上存在差异（神经节细胞层厚度不同），视锐度也呈梯度递减。但对 XR UI 设计而言，两者均处于高视锐度区域，都能支持正文文字的逐字阅读，设计约束一致（可使用的最小字号、控件尺寸等无需差异化），因此本规范将其合并为"中央凹区"。5° 是一条关键的设计分界线——超过此角度后用户**无法在不移动注视点的情况下阅读正文文字**，这一断崖式变化对 UI 元素的尺寸和信息密度有直接影响。

## 3. 基础数据：头部转动舒适度

头部转动涉及颈椎关节的运动，不同幅度的转动对应不同的肌肉负荷和主观感受。

### 3.1 水平转动（Yaw）舒适度分级

| 等级 | 转动幅度 | 肌肉参与 | 主观感受 | 可持续时间 |
|---|---|---|---|---|
| **轻松 (Easy)** | 0° ~ 20° | 无至轻微；颈部深层肌群低水平激活 | 从无意识的微调到自然的随意转头，类似围坐聊天时目光在几位朋友之间扫视；用户不会将转头本身感知为刻意动作 | 数分钟以上 |
| **舒适 (Comfortable)** | 20° ~ 35° | 中等；胸锁乳突肌与头夹肌主动参与 | 有转头意识但不费力，类似在多屏工作台前转头看旁边的显示器；颈部运动流畅，无肌肉紧张感 | 数十秒 ~ 1分钟 |
| **适度 (Moderate)** | 35° ~ 55° | 较明显；颈部旋转肌群全面参与，上斜方肌开始协同 | 需要刻意转头，类似开车变道时转头查看盲区；颈部有明确的肌肉工作感，保持数秒后会想回正 | 数秒 ~ 十余秒 |
| **费力 (Effortful)** | 55° ~ 75° | 显著；颈部肌群高负荷，上斜方肌与肩胛提肌明显紧张 | 明显的颈部紧张，类似课堂上回头看后排同学；肩部可能跟随转动，持续数秒即产生不适 | 数秒 |
| **极限 (Extreme)** | >75° | 高强度；颈部肌群接近极限，肩部与躯干旋转肌群介入 | 接近生理极限，类似试图不转身地回头看正后方；需要肩部和躯干辅助旋转，仅能瞬间维持 | 瞬间 |

### 3.2 垂直转动（Pitch）舒适度分级

垂直方向存在**显著的上下不对称性**——向下看比向上看更舒适，因为颈椎的屈曲（低头）比伸展（仰头）的肌肉负荷更低，且符合日常习惯（阅读、走路时自然下视）。

#### 向上（仰头）

| 等级 | 转动幅度（从自然视线起） | 肌肉参与 | 主观感受 | 可持续时间 |
|---|---|---|---|---|
| **轻松** | 0° ~ 10° | 无至轻微；枕下肌群微调 | 自然上瞥，类似从桌面抬眼看对面人的脸；无颈部感知 | 数分钟以上 |
| **舒适** | 10° ~ 20° | 轻度；头夹肌与头半棘肌主动收缩 | 轻微仰头，类似抬头看墙上挂钟；颈后有轻微拉伸感但不产生负担 | 数十秒 |
| **适度** | 20° ~ 30° | 中等；颈后伸肌群全面参与，上斜方肌协同 | 明显仰头，类似站在展厅抬头看高处画作；颈后肌群持续紧张，会主动想回正 | 数秒 ~ 十余秒 |
| **费力** | >30° | 显著；颈后肌群高负荷，接近关节活动度极限 | 大幅仰头，类似仰望天花板；颈后疲劳且颈椎伸展接近极限，可能伴随轻微头晕 | 数秒 |

#### 向下（低头）

| 等级 | 转动幅度（从自然视线起） | 肌肉参与 | 主观感受 | 可持续时间 |
|---|---|---|---|---|
| **轻松** | 0° ~ 15° | 无至轻微；重力辅助，颈深层屈肌（颈长肌、头长肌）轻度控制 | 自然下瞥，类似低头看手中的书；完全符合日常阅读习惯，比平视更放松 | 数分钟以上 |
| **舒适** | 15° ~ 25° | 轻度；颈深层屈肌持续参与，后侧肌群提供离心控制 | 轻微低头，类似坐着看桌面上的手机；有轻微负荷但仍感舒适 | 数十秒 ~ 1分钟 |
| **适度** | 25° ~ 40° | 中等；胸锁乳突肌双侧参与，颈后伸肌群承受离心负荷 | 有意识低头，类似站着查看腰间工牌；颈前侧和肩部有持续负荷感 | 数秒 ~ 十余秒 |
| **费力** | >40° | 显著；颈部屈伸肌群均高负荷，椎间盘承压增大 | 过度低头，类似长时间弯腰系鞋带的姿势；颈椎承压显著，XR 头显重心前移会加剧不适 | 数秒 |

### 3.3 坐姿与站姿差异

| 参数 | 站姿 | 坐姿 | 影响 |
|---|---|---|---|
| 自然静息视线（相对真实水平线） | 约 -10° | 约 -15° | 坐姿时内容整体应略低放置 |
| 水平舒适转动范围 | ±35° | ±30° | 坐姿因躯干受限，水平舒适范围略小 |
| 向下舒适范围 | -25° | -30°（含靠背支撑优势） | 坐姿有靠背支撑，低头反而略轻松 |
| 向上舒适范围 | +20° | +15° | 坐姿仰头受限，上方区域更应谨慎 |

> **设计启示**：为兼容两种姿态，应以更保守的值为准（即取两者的交集），或在应用中检测用户姿态并动态调整 UI 位置。

## 4. 基础数据：眼-头协同模型

当用户注视偏离正前方某个角度的内容时，视觉系统会自动将角度分配给眼球转动和头部转动。该协同模型是理解空间布局中"运动代价"的关键基础。

### 4.1 水平眼-头协同

当用户需要观看偏离正前方某个水平角度 θ 的内容时，视觉系统会自动将这个角度分配给眼球转动和头部转动：

| 水平偏移角 θ | 眼-头分配 | 协同特征 |
|---|---|---|
| θ < ~15° | 纯眼球运动 | 头部保持不动，眼球在近周边范围内自然扫视 |
| 15° < θ < ~30° | 眼球为主，头部微调 | 头部开始轻微偏转辅助，用户几乎无感 |
| 30° < θ < ~55° | 眼球与头部共同分担 | 头部成为主要贡献者，颈部肌群明确参与 |
| θ > ~55° | 头部主导，接近极限 | 头部达到舒适极限，需要更大幅度转动或肩/躯干参与 |


### 4.2 垂直眼-头协同

垂直方向的眼-头协同与水平方向遵循相同的渐进分担原则，但存在两个关键差异：**上下不对称性**和**姿态依赖性**。

**上下不对称性**

向下看比向上看更轻松，原因包括：

- **重力辅助**：低头时重力帮助头部下倾，颈部肌肉负荷更低
- **眼球运动不对称**：眼球向下旋转的舒适范围（~30°）大于向上（~25°）
- **习惯性偏好**：人类阅读、书写、手工操作等日常活动建立了根深蒂固的下视习惯

这使得向下方向的纯眼球运动范围更大、头部介入阈值更晚、各等级舒适区更宽。

**向上看（仰视）**

| 角度（从自然视线起） | 眼-头分配 | 协同特征 |
|---|---|---|
| θ < ~8° | 纯眼球运动 | 头部不动，眼球轻微上转即可 |
| 8° < θ < ~15° | 眼球为主，头部微调 | 眼球接近舒适上限，头部开始轻微仰起辅助 |
| 15° < θ < ~25° | 头部为主，眼球辅助 | 颈后伸肌群全面参与，头部承担大部分角度 |
| θ > ~25° | 头部接近极限 | 颈部后伸接近关节活动度上限，可能伴随不适 |

**向下看（俯视）**

| 角度（从自然视线起） | 眼-头分配 | 协同特征 |
|---|---|---|
| θ < ~12° | 纯眼球运动 | 头部不动，眼球自然下转（比上转范围更大） |
| 12° < θ < ~22° | 眼球为主，头部微调 | 重力辅助头部轻微下倾，几乎无额外肌肉负荷 |
| 22° < θ < ~35° | 头部为主，眼球辅助 | 颈部屈肌持续参与，但比同等仰头角度更轻松 |
| θ > ~35° | 头部接近极限 | 颈椎承压增大，XR 头显重心前移加剧负荷 |

**姿态依赖性**

与水平方向不同，垂直方向的眼-头协同**受身体姿态（坐姿/站姿）显著影响**。坐姿使向上区域收窄、向下区域扩展，具体表现为：

| 协同阶段 | 站姿 | 坐姿 | 差异原因 |
|---|---|---|---|
| 纯眼球范围（上/下） | 上 <8°；下 <12° | 同站姿 | 眼球运动不受姿态影响 |
| 头部舒适极限（上） | ~30° | ~25° | 坐姿时躯干固定，颈椎后伸空间受限 |
| 头部舒适极限（下） | ~40° | ~45° | 坐姿有靠背支撑，颈部屈曲负荷分散 |

> **关键结论**：坐姿使向上舒适范围缩减约 5°，向下舒适范围扩展约 5°。这一差异在长时间使用场景下对 UI 布局有实质影响。

## 5. 多模型联合决策框架

### 5.1 三问判定法

对任意一个待放置的 UI 元素，依次回答以下三个问题，取**最严格的约束**作为最终空间位置。

```
问题 1 ─ 该元素的交互频率是多少？（→ 眼-头协同模型）
│
├─ 高频（>5 次/分钟）
│     纯眼球范围：水平 ±15°｜上 +8°｜下 -12°
│     （头部实际转动 ≈ 0°）
├─ 中频（1–5 次/分钟）
│     眼主头辅范围：水平 ±30°｜上 +15°｜下 -22°
│     （头部实际转动：水平 ≈ 9°｜上 ≈ 5°｜下 ≈ 7°）
├─ 低频（每小时数次）
│     眼头共担范围：水平 ±55°｜上 +25°｜下 -35°
│     （头部实际转动：水平 ≈ 33°｜上 ≈ 17°｜下 ≈ 24°）
└─ 仅偶发
      头部主导范围：水平 >55°｜上 >25°｜下 >35°
                │
问题 2 ─ 该元素需要什么视觉精度？（→ 视野角分区）
│
├─ 逐字阅读      → 必须在中央凹区（0°–5°）
├─ 识别图标/标题  → 必须在有效识别区（0°–10°）
├─ 感知色块/轮廓  → 可在近周边区（0°–30°）
├─ 检测运动      → 可在中周边区（0°–60°）
└─ 仅需明暗感知   → 可在远周边区（60°+）
                │
问题 3 ─ 该元素需要用户注视多长时间？（→ 头部转动舒适分区）
│
├─ 持续性（>1 分钟）    → 必须在轻松区
│     水平 ±20°｜上 +10°｜下 -15°
├─ 短时（10 秒–1 分钟） → 必须在舒适区以内
│     水平 ±35°｜上 +20°｜下 -25°
├─ 一瞥（3–10 秒）      → 可在适度区以内
│     水平 ±55°｜上 +30°｜下 -40°
└─ 瞬间（<3 秒）        → 可在费力区以内
      水平 ±75°｜上 >30°｜下 >40°
                │
最终 ─ 取三问中最严格的约束作为该元素的允许位置
```

### 5.2 眼-头协同角与头部实际转动角折算表

眼-头协同模型中的角度是**总注视偏移角**（眼球 + 头部之和），并非头部实际转动角。下表给出折算关系，供与头部转动舒适分区（第 3 章）交叉验证。

#### 水平方向

| 总注视偏移角 θ | 协同阶段 | 头部贡献比例 | 头部实际转动（估算） | 对应头部舒适等级 |
|---|---|---|---|---|
| 0°–15° | 纯眼球运动 | 0% | **0°** | — （头部不动） |
| 15°–30° | 眼主头辅 | ~30%（头部约 9°） | **5°–9°** | 轻松（0°–20°）✅ |
| 30°–55° | 眼头共担 | ~60% | **18°–33°** | 轻松→舒适（20°–35°）⚠️ 上沿 |
| >55° | 头部主导 | ~80%+ | **>44°** | 适度→费力（35°–75°）❌ 不可持续 |

#### 垂直向上（仰视）

| 总注视偏移角 θ | 协同阶段 | 头部贡献比例 | 头部实际转动（估算） | 对应头部舒适等级 |
|---|---|---|---|---|
| 0°–8° | 纯眼球运动 | 0% | **0°** | — |
| 8°–15° | 眼主头辅 | ~30%–40% | **2°–6°** | 轻松（0°–10°）✅ |
| 15°–25° | 头部为主 | ~60%–70% | **9°–18°** | 轻松→舒适（10°–20°）⚠️ |

| 总注视偏移角 θ | 协同阶段 | 头部贡献比例 | 头部实际转动（估算） | 对应头部舒适等级 |

## 7. 独立维度速查表

以下维度与具体 UI 组件绑定关系不强，按角度带或功能逻辑独立整理。

### 7.1 渲染策略分级（Foveated Rendering）

| 角度带（从注视点） | 建议渲染分辨率 | 来源与原因 |
|---|---|---|
| 0°–5°（中央凹区） | **100%** 全分辨率、全细节 | [视] 最高视锐度区，用户可辨认笔画级细节 |
| 5°–10°（周中央凹） | **≥ 70%** | [视] 视锐度降至 30%，但用户仍可识别大型元素轮廓 |
| 10°–30°（近周边） | **30%–50%** | [视] 仅感知形状轮廓与色块，无法识别细节 |
| 30°–60°（中周边） | **15%–25%**，重点保留运动流畅度 | [视] 以运动检测为主，静态细节无意义 |
| 60°+（远周边） | **5%–10%**，保留明暗变化与运动帧率 | [视] 仅对突发运动和强烈明暗变化有反应；>60° 为单眼视野，无需立体深度渲染 |

### 7.2 注意力引导链（从外围到中心的标准通知流程）

当需要将用户注意力从外围引导至中心时，应遵循以下由外向内的四阶段流程：

| 阶段 | 角度带 | 行为 | 用户感知 | 时间窗口 |
|---|---|---|---|---|
| **阶段 1：本能警觉** | 60°+ | 强烈闪烁 / 大面积明暗变化 / 空间音频 | 本能警觉"某个方向有事" | <1 秒 |
| **阶段 2：方向感知** | 30°–60° | 动画滑入概览卡片（标题+摘要） | "有事发生，在那个方向" | 2–3 秒 |
| **阶段 3：扫视定位** | 10°–30° | 高对比度锚点出现 | 用户眼跳定位目标 | 2–3 秒 |
| **阶段 4：详细阅读** | 0°–10° | 展开完整内容面板 | 用户注视阅读/交互 | 按需（可持续） |

> **来源**：[视] 渐进式信息披露原则 + 注意力引导链；[眼头H] 由外向内的运动代价递减逻辑。

**关键约束**：每阶段在该角度带的可持续时间内完成——阶段 1 在费力区（数秒），阶段 2 在适度区（数秒），阶段 3 在舒适区（数十秒），阶段 4 在轻松区（可持续）。

### 7.3 动态重定位策略（UI Recentering）

当用户身体转动或注意力转移后，原本处于舒适区的 UI 面板可能漂移到更大角度。以下策略的触发条件通过 5.1 三问判定法的逆向应用推导：确定 UI 当前所处的眼-头协同区间 → 经 5.2 折算为头部实际转动角 → 对照第 3 章头部转动舒适分区得出可持续时间 → 在可持续时间窗口内设置触发点。

| 策略 | 触发条件 | 系统行为 | 关键约束 |
|---|---|---|---|
| **惰性跟随** | UI 落入总注视偏移角 >30° 且持续 >2–3 秒 | UI 面板缓慢漂移向用户当前朝向，重新居中 | 动画 300–500ms ease-out，缓慢可预测；避免突兀跳跃引发眩晕 |
| **手动重定位** | 用户执行特定手势（双击、长按） | 整个 UI 布局瞬间以当前头部朝向为新中心 | 无角度约束，用户主动触发；任何角度下均可用的兜底手段 |
| **场景感知重定位** | 用户注视点连续 >15 秒偏离 UI 面板 >15° | 主面板自动吸附到注视区域附近 | 需确认注意力已转移（非短暂分神），避免误触发 |
| **超时内容迁移** | UI 落入总注视偏移角 20°–35° 且用户持续注视 >40 秒 | 面板边缘出现"拖到中心"柔和视觉提示 | 仅提示不强制；用户可忽略或手动迁移 |
| **自动缩略回中** | UI 落入总注视偏移角 35°–55° 且用户持续注视 >8 秒 | 内容自动缩略并向中心方向滑动 | 缩略保留关键信息摘要，可一键恢复完整视图 |
| **转身引导** | UI 落入总注视偏移角 55°–75° 且持续 >3 秒 | 地面箭头 + 空间音频引导用户转身 | 多模态引导（视觉+音频），不强制转动 |

> **触发条件推导（5.1 逆向应用 + 5.2 折算验证）**：
>
> | 策略 | UI 所处角度带 | 5.2 折算：头部实际转动 | 对应舒适等级（§3.1） | 该等级可持续时间 | 触发时间点及推导 |
> |---|---|---|---|---|---|
> | **惰性跟随** | >30°（眼头共担区） | ≈18°+（at 30°: 轻松区上沿） | 轻松→舒适 | 数分钟→数十秒 | **2–3 秒**：此时头部尚在轻松区可持续范围内，等待 2–3 秒用于确认用户是意图性的朝向改变而非瞬态扫视，随后主动跟随，避免 UI 继续漂移进入舒适区 |
> | **超时内容迁移** | 20°–35°（眼主头辅→眼头共担） | 20° 时 ≈6°（轻松区）；35° 时 ≈21°（舒适区下沿） | 轻松→舒适 | 数分钟→数十秒–1 分钟 | **40 秒**：该角度带横跨轻松与舒适两区；在舒适区部分（30°–35°），可持续上限为 数十秒–1 分钟，40 秒为该上限前的预留余量 |
> | **自动缩略回中** | 35°–55°（眼头共担区） | ≈21°–33°（舒适区） | 舒适 | 数十秒–1 分钟 | **8 秒**：舒适区虽可持续数十秒，但 35°–55° 已处于眼头共担的深处，角度越大头部负荷增长越快（33° 逼近舒适区上限 35°），在可持续窗口早期即介入 |
> | **转身引导** | 55°–75°（头部主导区） | ≈44°–60°（适度→费力区） | 适度→费力 | 数秒–十余秒→数秒 | **3 秒**：适度区仅可维持数秒–十余秒，费力区仅数秒；在可持续窗口最早期触发，引导用户转身替代颈部极限转动 |
>
> **核心原则**：突兀的 UI 跳跃会引发眩晕和空间迷失。重定位动画必须缓慢且可预测。策略的激进程度随角度带递增——轻松区仅柔和提示，舒适区主动迁移，适度/费力区立即引导转身。

|---|---|---|
| 0°–15° | 注视交互（dwell-to-select、gaze-and-pinch）精度最高；全类型手势、语音 | [眼头H] 纯眼球区无头部抖动干扰 |
| 15°–30° | 手势（点击、滚动）为主；注视交互精度略降 | [眼头H] 头部微调引入轻微抖动 |
| 30°–55° | 头部转动本身可作为交互信号（head-gaze reveal）；大面积触发区 | [眼头H] 头部大幅偏转时手部追踪精度下降 |
| >55° | 仅大面积触发区或方向性提示 | [眼头H] 头部接近极限，手部姿态别扭 |

### 7.6 时间约束总表（水平 + 垂直联合）

| 舒适等级 | 水平 | 向上 | 向下 | 可持续时间 | 允许的内容复杂度 |
|---|---|---|---|---|---|
| **轻松** | ±20° | +10° | -15° | **数分钟以上** | 长文本阅读、表单填写、视频观看、代码编写 |
| **舒适** | 20°–35° | 10°–20° | 15°–25° | **数十秒–1 分钟** | 参考文档查阅（10–30 秒）、面板操作（3–10 秒） |
| **适度** | 35°–55° | 20°–30° | 25°–40° | **数秒–十余秒** | 通知预览（2–5 秒）、方向确认（1–2 秒） |
| **费力** | 55°–75° | >30° | >40° | **数秒** | 存在性感知（<1 秒） |
| **极限** | >75° | — | — | **瞬间** | 无交互内容 |

> **联合规则**：对同时存在水平和垂直偏移的位置，取两个方向中**更严格的时间约束**。

### 7.7 运动代价与信息回报匹配

用户付出的运动代价必须获得相应的信息回报，否则产生"不值得"感：

| 转头幅度（总注视角） | 头部实际转动 | 用户期望获得的回报 | 来源 |
|---|---|---|---|
| 0°–15° | 0°（纯眼球） | 即时微反馈（按钮高亮、数值变化） | [眼头H] |
| 15°–30° | 5°–9° | 一组相关信息（展开面板、新视图） | [眼头H] |
| 30°–55° | 18°–33° | 显著的上下文切换（新工作区、新场景） | [眼头H] |
| >55° | >44° | 值得转身的重要事件 | [眼头H] |

> **常见败笔 [眼头H]**：用户转头 45° 只看到一个小按钮——运动代价与信息密度的错配。

### 7.8 垂直布局核心原则：重心下沉

整个 UI 的垂直重心应位于自然视线**下方约 5°–8°**，而非正中央。

**四维度验证**：

| 维度 | 上方  | 下方  | 结论 |
|---|---|---|---|
| 舒适等级 | 轻松区内（勉强） | 轻松区最佳位置 | 下方更优 [舒V] |
| 主观描述 | "自然上瞥" | "比平视更放松" | 下方更自然 [舒V] |
| 肌肉参与 | 枕下肌群微调 | 重力辅助，仅轻度控制 | 下方负担更小 [眼头V] |
| 可用空间 | 上方轻松区仅 10° | 下方轻松区 15° | 下方多 50% [舒V] |


上下空间预算比约为 **+8°：-12° （1 : 1.5）**。

### 7.9 零成本核心区形状

将水平与垂直纯眼球运动范围叠加，**零成本核心区**为一个**上窄下宽的不对称椭圆**：

```
               上 8°（纯眼球）
            ┌─────────────────┐
            │                 │
 左 15° ── │    ★ 注视中心   │ ── 右 15°
（纯眼球）  │     零成本区     │  （纯眼球）
            │                 │
            │                 │
            └─────────────────┘
              下 12°（纯眼球）

水平 ±15°（总 30°）× 上 8° + 下 12°（总 20°）
@0.7m 交互距离 ≈ 38cm 宽 × 25cm 高
```

> **所有高频精细交互必须纳入这个不对称椭圆之内。**

### 7.10 头显重量力矩效应

（该模块可暂时忽略）

头显自重（300–500g）在非正中姿势下通过力矩放大颈部负担。垂直方向影响尤为显著：

| 姿势 | 力矩效应 | 压力集中点 | 舒适度折减（相对裸头） | 设计对策 |
|---|---|---|---|---|
| 仰头 10°–20° | 重心后移，上缘压额头 | 前额上缘、后部绑带 | 约减 3°–5° | 更保守的仰视限制 |
| 仰头 >20° | 头显下滑趋势，需收紧绑带 | 额头全面受压 | 约减 5°–8° | 几乎不应要求此角度 |
| 低头 15°–25° | 重心前坠，鼻梁受压 | 鼻梁、面颊 | 约减 3°–5° | 持续俯视 >20° 超 30 秒应提示 |
| 低头 >25° | 前倾力矩显著 | 鼻梁集中受力 | 约减 5°–10° | 此区域头显影响最严重 |
| **正中偏下 5°** | **力矩最小，头显最稳定** | — | **0°** | 再次印证内容重心下沉的合理性 |

**佩戴头显后建议轻松区修正**：

| 方向 | 裸头轻松区 | 佩戴头显后建议 |
|---|---|---|
| 向上 | 0°–10° | 0°–7° |
| 向下 | 0°–15° | 0°–12° |

### 7.11 累积疲劳模型

单次转头代价是瞬时的，但重复转头的代价是累积的：

| 场景 | 单次代价 | 累积问题 | 设计对策 |
|---|---|---|---|
| 反复在 0° 和 30° 之间切换 | 舒适 | 30 分钟内累计转头上百次，颈部累积疲劳 | 双面板总跨度压缩到 20° 以内，或快捷键在同一位置切换内容 |
| 反复在 0° 和 50° 之间切换 | 适度 | 10 分钟内即可感到颈部酸痛 | 此工作流不可接受，必须重新布局 |
| 持续头部在 25°–35° 小幅摆动 | 舒适 | 旋转肌群长时间中等负荷重复工作 | 定期休息提醒，或自动调整使摆动降至 15° 以内 |

**简化疲劳公式**：

```
疲劳指数 ∝ Σ(角度 × 频次 × 持续时间) / 休息间隔
```

XR 系统应实时追踪此指数，累积超阈值时主动干预（建议休息、重组布局）。

### 7.12 空间音频配合建议

| 角度带 | 音频策略 | 来源 |
|---|---|---|
| 0°–10° | 无需音频辅助，视觉信息充分 | [视] 有效识别区 |
| 10°–30° | 建议配合空间音频提示引导注意力转向 | [视] 超出 10° 的视觉信息单独使用效果下降 |
| 30°–60° | 空间音频作为动态通知的必要伴随 | [视] 仅运动检测有效，需多模态增强 |
| 60°+ | 空间音频 + 触觉反馈作为紧急警报的核心通道 | [视] 视觉几乎无效，需替代通道 |

## 8. 常见设计错误汇总

（以下内容内刊，如发现异常及时修改）

以下错误来自五个模型的独立分析，已合并重复项。每条标注来源模型。

### 8.1 空间位置错误

| 错误做法 | 生理原因 | 修正方案 | 来源 |
|---|---|---|---|
| 在 >10° 处放小字正文 | 超出有效识别区，用户无法阅读 | 移入 5° 以内，或放大至标题级（≥ 2× 正文字号） | [视] |
| 主工具栏放在 20°–30° | 高频操作触发反复头部微调，累积颈部疲劳 | 移入 ±15° 以内（纯眼球运动范围） | [眼头H] |
| 对话窗口跨度 >30° | 阅读需持续眼头共担运动，体验割裂 | 窗口宽度限制 ±15°，纵向延伸替代横向扩展 | [眼头H] |
| 双屏并排跨度达 60° | 频繁切换需大幅转头，极易颈部疲劳 | 总跨度控制在 ±25° 以内，或用切换替代并排 | [眼头H] |
| 在 30° 处放置需 2 分钟填写的表单 | 舒适区可维持时间仅约 1 分钟 | 表单弹出到 0°–20° 轻松区，填完后自动回收 | [舒H] |
| 在 45° 处放需精细阅读的通知详情 | 适度区仅维持数秒，且视锐度不足以阅读正文 | 45° 仅显示通知摘要（大字），详情在注视后展开到中心 | [舒H] |
| 环形排列 6 个等距面板（每个间隔 60°） | 最远面板在 ±90°，进入极限区 | 扇形排列在 ±35° 以内，用切换而非环绕 | [舒H] |
| 所有 UI 挤在 0°–5° | 中央凹区面积极小，信息过载导致频繁微扫视 | 按层级分布到 0°–30° 各环 | [视] |

### 8.2 垂直布局错误

| 错误做法 | 生理原因 | 修正方案 | 来源 |
|---|---|---|---|
| UI 上下对称分布（上方 20° / 下方 20°） | 上方 20° 已进入不适区，下方 20° 仍在舒适区 | 整体下移，上方不超 +10°，下方可延伸到 -20° | [眼头V] |
| 顶部导航栏放在 +15° 以上 | 每次访问需明确仰头，高频使用时颈后肌持续紧张 | 导航栏放在 +5°~+8°，或改为下方/侧方导航 | [眼头V] |
| 虚拟键盘放在视线正中（0°） | 打字时视线在键盘和内容间反复切换 | 键盘下移至 -15°~-22°，输入内容在 -5°~-10° | [眼头V][舒V] |
| 上下对称环形菜单（上下各 30°） | 上方 30° 已入适度区（数秒即想回正），下方 30° 仍在舒适区 | 菜单下移，上缘不超 +10°，下缘可达 -25° | [舒V] |
| VR 影院屏幕中心放在正前方（0°） | 忽视"微微低头比平视更放松" | 屏幕中心下移至 -8°~-12° | [舒V] |
| 长时间要求仰视 >20° | 颈后伸肌持续对抗重力+头显重量 | 避免关键交互高于 +15°，或自动旋转将高处内容带到视线前方 | [眼头V] |
| AR 导航箭头固定在地面（下 35°+） | 持续低头颈部不适，头显前坠加剧鼻梁压力 | 导航路径投影提升到下 15°–20° 半透明 HUD | [舒V] |

### 8.3 内容形态错误

| 错误做法 | 生理原因 | 修正方案 | 来源 |
|---|---|---|---|
| 在 30°–60° 放静态图标 | 中周边视觉对静态细节几乎失明 | 改用动态提示（动画图标），或移入 30° 以内 | [视] |
| 30°–55° 区域放静态文字信息 | 既看不清文字，又需有意识转头才到达 | 改用动态入口，点击后拉回中心显示 | [眼头H] |
| 紧急通知仅用静态提示 | 30°+ 区域只对运动敏感 | 使用动画、闪烁或脉冲效果 | [视] |
| 在 >60° 依赖立体深度传递信息 | 已进入单眼视野，无双目视差 | 使用运动视差或明暗线索替代 | [视] |

### 8.4 交互方式错误

| 错误做法 | 生理原因 | 修正方案 | 来源 |
|---|---|---|---|
| >55° 放置需精确点选的按钮 | 头部达极限，手部追踪精度也因姿态别扭而下降 | 仅放大面积触发区或瞬态提示 | [眼头H] |
| 在适度区（35°–55°）放精细操作控件 | 颈部有明确肌肉工作感，数秒后想回正；手部精度下降 | 此区域只能放"扫一眼"内容，5 秒内完成 | [舒H] |

### 8.5 系统行为错误

| 错误做法 | 生理原因 | 修正方案 | 来源 |
|---|---|---|---|
| UI 面板固定不动，用户转身后"丢失"界面 | 转身后 UI 落入 >90°，完全不可见 | 惰性跟随或一键重定位手势 | [眼头H] |
| 长时间会议中 UI 固定不动 | 用户自然缓慢转体，原先 20° 面板逐渐偏移到 40°+ | 轻松区 UI 缓慢跟随用户朝向漂移 | [舒H] |
| 持续 AR 桌面操作（低头 30°+）无休息提示 | 头显重心前移致鼻梁持续受压，20 分钟后出现压痕 | 每 5 分钟提示抬头休息 10 秒；提供"桌面内容抬升到眼前"快捷手势 | [舒V] |
| 仰头观星 App 未提供姿势管理 | 仰视 >30° 超数十秒即颈部疲劳，伴随轻微头晕 | 提供"天空下拉"手势将天球拉到平视范围；每 2 分钟仰视后建议休息 | [舒V] |
| 多人协作中协作者虚拟形象放在 70° | 转头看人颈部紧张，社交体验受损 | 虚拟形象在 ±30° 以内弧形排列，最多 3–4 人 | [舒H] |
