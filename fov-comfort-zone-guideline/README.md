# XR 空间 UI 布局设计规范

## 基于人眼视野角与头部转动舒适度的多层决策框架

---

## 1. 概述

本规范为 XR 头显环境中虚拟交互内容的空间布局提供设计指导。基于**人眼视野角分区**、**头部转动舒适度分区**和**眼-头协同模型**三组独立的生理学基础数据，构建一个**多层决策框架**，指导从环境氛围设计到单个 UI 元素类型选择的全链路空间布局决策。

本规范采用**分层决策**而非**扁平融合**的方法论。三组基础数据并非在同一角度轴上取中间值合并为一张区域地图，而是在不同的设计决策层级上各自发挥主导作用——每种分区体系在其最相关的设计层使用原生阈值，避免跨体系强行对齐导致的远端偏差。

### 1.1 适用范围

| 属性 | 说明 |
|---|---|
| 平台 | XR 头显（通用，不绑定特定硬件） |
| 坐标参考 | Body-locked 和 World-locked UI（附带 Head-locked 建议） |
| 覆盖姿态 | 坐姿与站姿 |
| 维度 | 水平角度（Yaw）+ 垂直角度（Pitch），不含深度（Z 轴） |
| 内容类型 | 主交互面板、信息展示、通知提示、导航元素、媒体内容、环境氛围 |

### 1.2 坐标系定义

本规范采用以下坐标系约定：

- **原点**：用户头部中心位置
- **水平角度（Azimuth / Yaw）**：0° 为正前方，向左为负，向右为正，范围 -180° ~ +180°
- **垂直角度（Elevation / Pitch）**：以**自然静息视线**为 0° 基准，向上为正，向下为负
- **自然静息视线**：站姿约低于真实水平线 10°，坐姿约低于真实水平线 15°

> **为什么以自然静息视线为垂直基准？** 人在放松状态下的视线并非水平前视，而是自然下倾。以此为基准，能更准确地反映用户在 XR 环境中的真实感受，使得"0° 垂直"对应"毫无努力的自然注视方向"。

## 2. 基础数据：人眼视野角

人眼视野（Field of View）并非均匀的感知区域，而是从中心向外逐级退化的分层结构。

### 2.1 视野角功能分区

视网膜的中央区域——黄斑 (Macula)——覆盖注视点周围约 10° 半径，包含解剖学上的中央凹 (Fovea)、近中央凹 (Parafovea) 和周中央凹 (Perifovea) 三层同心结构，合称"中央视觉"。由于中央凹与近中央凹对 XR UI 设计的约束趋同（详见下方说明），本规范将两者合并为"中央凹区"，与周中央凹分别处理。超出黄斑区即进入周边视觉。

| 分区 | 角半径（从注视点） | 视觉能力 | XR 设计意义 |
|---|---|---|---|
| **中央凹区 (Foveal Zone)** | 0° ~ 5° | 最高视锐度，可分辨文字笔画与精细图形细节，颜色感知完整 | **精细阅读区**——适合正文文字、小型 UI 控件（按钮标签、输入光标、滑块刻度）。超出 ±5° 后无法逐字阅读 |
| **周中央凹 (Perifoveal)** | 5° ~ 10° | 视锐度降至中央凹区的约 30%；无法辨认正文字号的笔画，但可凭轮廓与对比度识别熟悉的大型元素 | 适合大号标题、熟悉图标、高对比度区块，无需眼跳即可判断元素类型。超出 ±10° 后无法识别具体内容（整个 0°–10° 构成**有效识别区**） |
| **近周边视觉 (Near Peripheral)** | 10° ~ 30° | 无法识别文字或图标的具体内容，仅可感知形状轮廓、色块与空间位置 | 扫视目标定位（"那边有个东西"）、UI 区块存在性感知。±30° 是不转头获取信息的实际极限（**功能视野边界**） |
| **中周边视觉 (Mid Peripheral)** | 30° ~ 60° | 静态细节几乎不可辨，以运动检测和明暗对比为主；颜色辨识退化 | 动态变化察觉（动画、闪烁、弹出）、空间方位感。±60° 以外仅对突发运动有反应（**运动感知边界**） |
| **远周边视觉 (Far Peripheral)** | 60° ~ 100°+ | 仅对突发运动和强烈明暗变化有反应；>60° 主要为单眼视野 | 威胁/事件警觉、空间存在感与包围感 |

> **关于分区的方向性**：以上分区为以注视点为圆心的**同心圆环结构**，在水平、垂直和对角方向上均成立——同一角半径处的视觉能力不随方向变化。方向性差异仅体现在视野的**物理外边界**上：水平方向单眼视野可达颞侧 ~100°、鼻侧 ~60°，而垂直方向仅为上方 ~60°、下方 ~70°–75°。这意味着远周边视觉区在垂直方向上更早到达物理极限。

> **关于中央凹区的合并**：解剖学上，0°–2° 的中央凹 (Fovea) 与 2°–5° 的近中央凹 (Parafovea) 在视网膜层结构上存在差异（神经节细胞层厚度不同），视锐度也呈梯度递减。但对 XR UI 设计而言，两者均处于高视锐度区域，都能支持正文文字的逐字阅读，设计约束一致（可使用的最小字号、控件尺寸等无需差异化），因此本规范将其合并为"中央凹区"。5° 是一条关键的设计分界线——超过此角度后用户**无法在不移动注视点的情况下阅读正文文字**，这一断崖式变化对 UI 元素的尺寸和信息密度有直接影响。

## 3. 基础数据：头部转动舒适度

头部转动涉及颈椎关节的运动，不同幅度的转动对应不同的肌肉负荷和主观感受。

### 3.1 水平转动（Yaw）舒适度分级

| 等级 | 转动幅度 | 肌肉参与 | 主观感受 | 可持续时间 |
|---|---|---|---|---|
| **轻松 (Easy)** | 0° ~ 20° | 无至轻微；颈部深层肌群低水平激活 | 从无意识的微调到自然的随意转头，类似围坐聊天时目光在几位朋友之间扫视；用户不会将转头本身感知为刻意动作 | 数分钟以上 |
| **舒适 (Comfortable)** | 20° ~ 35° | 中等；胸锁乳突肌与头夹肌主动参与 | 有转头意识但不费力，类似在多屏工作台前转头看旁边的显示器；颈部运动流畅，无肌肉紧张感 | 数十秒 ~ 1分钟 |
| **适度 (Moderate)** | 35° ~ 55° | 较明显；颈部旋转肌群全面参与，上斜方肌开始协同 | 需要刻意转头，类似开车变道时转头查看盲区；颈部有明确的肌肉工作感，保持数秒后会想回正 | 数秒 ~ 十余秒 |
| **费力 (Effortful)** | 55° ~ 75° | 显著；颈部肌群高负荷，上斜方肌与肩胛提肌明显紧张 | 明显的颈部紧张，类似课堂上回头看后排同学；肩部可能跟随转动，持续数秒即产生不适 | 数秒 |
| **极限 (Extreme)** | >75° | 高强度；颈部肌群接近极限，肩部与躯干旋转肌群介入 | 接近生理极限，类似试图不转身地回头看正后方；需要肩部和躯干辅助旋转，仅能瞬间维持 | 瞬间 |

### 3.2 垂直转动（Pitch）舒适度分级

垂直方向存在**显著的上下不对称性**——向下看比向上看更舒适，因为颈椎的屈曲（低头）比伸展（仰头）的肌肉负荷更低，且符合日常习惯（阅读、走路时自然下视）。

#### 向上（仰头）

| 等级 | 转动幅度（从自然视线起） | 肌肉参与 | 主观感受 | 可持续时间 |
|---|---|---|---|---|
| **轻松** | 0° ~ 10° | 无至轻微；枕下肌群微调 | 自然上瞥，类似从桌面抬眼看对面人的脸；无颈部感知 | 数分钟以上 |
| **舒适** | 10° ~ 20° | 轻度；头夹肌与头半棘肌主动收缩 | 轻微仰头，类似抬头看墙上挂钟；颈后有轻微拉伸感但不产生负担 | 数十秒 |
| **适度** | 20° ~ 30° | 中等；颈后伸肌群全面参与，上斜方肌协同 | 明显仰头，类似站在展厅抬头看高处画作；颈后肌群持续紧张，会主动想回正 | 数秒 ~ 十余秒 |
| **费力** | >30° | 显著；颈后肌群高负荷，接近关节活动度极限 | 大幅仰头，类似仰望天花板；颈后疲劳且颈椎伸展接近极限，可能伴随轻微头晕 | 数秒 |

#### 向下（低头）

| 等级 | 转动幅度（从自然视线起） | 肌肉参与 | 主观感受 | 可持续时间 |
|---|---|---|---|---|
| **轻松** | 0° ~ 15° | 无至轻微；重力辅助，颈深层屈肌（颈长肌、头长肌）轻度控制 | 自然下瞥，类似低头看手中的书；完全符合日常阅读习惯，比平视更放松 | 数分钟以上 |
| **舒适** | 15° ~ 25° | 轻度；颈深层屈肌持续参与，后侧肌群提供离心控制 | 轻微低头，类似坐着看桌面上的手机；有轻微负荷但仍感舒适 | 数十秒 ~ 1分钟 |
| **适度** | 25° ~ 40° | 中等；胸锁乳突肌双侧参与，颈后伸肌群承受离心负荷 | 有意识低头，类似站着查看腰间工牌；颈前侧和肩部有持续负荷感 | 数秒 ~ 十余秒 |
| **费力** | >40° | 显著；颈部屈伸肌群均高负荷，椎间盘承压增大 | 过度低头，类似长时间弯腰系鞋带的姿势；颈椎承压显著，XR 头显重心前移会加剧不适 | 数秒 |

### 3.3 坐姿与站姿差异

| 参数 | 站姿 | 坐姿 | 影响 |
|---|---|---|---|
| 自然静息视线（相对真实水平线） | 约 -10° | 约 -15° | 坐姿时内容整体应略低放置 |
| 水平舒适转动范围 | ±35° | ±30° | 坐姿因躯干受限，水平舒适范围略小 |
| 向下舒适范围 | -25° | -30°（含靠背支撑优势） | 坐姿有靠背支撑，低头反而略轻松 |
| 向上舒适范围 | +20° | +15° | 坐姿仰头受限，上方区域更应谨慎 |

> **设计启示**：为兼容两种姿态，应以更保守的值为准（即取两者的交集），或在应用中检测用户姿态并动态调整 UI 位置。

## 4. 基础数据：眼-头协同模型

当用户注视偏离正前方某个角度的内容时，视觉系统会自动将角度分配给眼球转动和头部转动。该协同模型是理解空间布局中"运动代价"的关键基础。

### 4.1 水平眼-头协同

当用户需要观看偏离正前方某个水平角度 θ 的内容时，视觉系统会自动将这个角度分配给眼球转动和头部转动：

| 水平偏移角 θ | 眼-头分配 | 协同特征 |
|---|---|---|
| θ < ~15° | 纯眼球运动 | 头部保持不动，眼球在近周边范围内自然扫视 |
| 15° < θ < ~30° | 眼球为主，头部微调 | 头部开始轻微偏转辅助，用户几乎无感 |
| 30° < θ < ~55° | 眼球与头部共同分担 | 头部成为主要贡献者，颈部肌群明确参与 |
| θ > ~55° | 头部主导，接近极限 | 头部达到舒适极限，需要更大幅度转动或肩/躯干参与 |

**协同阈值的推导**

上表中的三个阈值（~15°、~30°、~55°）由两个生理学输入交叉推导而来：

**输入 A：眼球舒适偏转范围**

眼球在眼眶中的旋转由六条外眼肌控制。在约 ±15° 以内，外眼肌处于低张力状态，可长时间维持任意方向的偏转而不产生疲劳。超出 ±15° 后肌肉张力急增，至 ±20°~25° 达到舒适极限——眼球仍可偏转至此角度，但无法长时间停留。这个 **~15° 的零代价偏转范围**是整个协同模型的基础输入。

**输入 B：VOR 增益曲线**

当所需注视角 θ 超过眼球舒适范围时，前庭-眼反射（Vestibulo-Ocular Reflex, VOR）自动启动头部运动来分担角度。VOR 增益 = 头部贡献角 / 总注视角，该值随 θ 呈 S 形曲线上升：

| 总注视角 θ | VOR 增益（近似） | 眼球贡献（近似） | 头部贡献（近似） | 头部贡献所对应的舒适度（第 3.1 节） |
|---|---|---|---|---|
| 10° | ~0 | ~10° | ~0° | —（无头部运动） |
| 15° | ~0.1 | ~14° | ~1° | 轻松（0°~20°） |
| 30° | ~0.4~0.5 | ~17° | ~13° | 轻松（0°~20°） |
| 45° | ~0.6 | ~18° | ~27° | 舒适→适度（20°~35° → 35°~55°） |
| 55° | ~0.7 | ~17° | ~38° | 适度（35°~55°） |
| 70° | ~0.8 | ~14° | ~56° | 费力（55°~75°） |

> 眼球贡献在 30°~55° 区间达到峰值（~18°~20°）后开始回落——大角度注视时眼球倾向回归中心位置（**向心回弹**），这是前庭-眼反射的固有特性。

**三个阈值的推导**

| 协同阈值 | 推导依据 | 与第 2 节的交叉验证 | 与第 3.1 节的交叉验证 |
|---|---|---|---|
| **~15°** | 眼球舒适偏转上限（输入 A）。低于此值时 VOR 增益接近 0，头部不参与——纯眼球即可完成注视转移 | 15° 仍在近周边视觉前段（10°~30°），未超出功能视野 | 头部贡献 <2°，远在轻松范围内（0°~20°） |
| **~30°** | VOR 增益达 ~0.4~0.5（输入 B），头部开始承担接近一半角度。眼球接近舒适极限（~17°），标志着"眼球独立工作"转为"双方协同" | 恰好对应近周边→中周边视觉过渡点（30°） | 头部贡献 ~13° 仍在轻松范围，但已接近轻松→舒适阈值（~20°） |
| **~55°** | VOR 增益达 ~0.7（输入 B），头部承担绝大部分角度（~38°）。同时眼球贡献因向心回弹开始从峰值回落，系统进入"头部主导"模式 | 55° 处于中周边视觉中段（30°~60°），转头前几乎无静态辨识能力 | 头部贡献 ~38° 处于适度范围（35°~55°），接近适度→费力阈值 |

> **总结**：眼-头协同阈值由一个生理常数（眼球舒适偏转 ~15°）和一条行为曲线（VOR 增益的 S 形上升）共同决定。第 2 节的视觉分区和第 3.1 节的头部舒适度等级并非这些阈值的输入，而是**独立的交叉验证**——三组独立数据在相近角度处发生质变，表明人体视觉-运动系统在这些角度存在真实的行为模式转折。

水平方向的眼-头协同具有良好的**左右对称性**——向左和向右的运动代价基本一致，且**不受身体姿态（坐/站）显著影响**（坐姿仅因躯干受限使舒适转动范围从 ±35° 略缩至 ±30°，协同模式不变）。

### 4.2 垂直眼-头协同

垂直方向的眼-头协同与水平方向遵循相同的渐进分担原则，但存在两个关键差异：**上下不对称性**和**姿态依赖性**。

**上下不对称性**

向下看比向上看更轻松，原因包括：

- **重力辅助**：低头时重力帮助头部下倾，颈部肌肉负荷更低
- **眼球运动不对称**：眼球向下旋转的舒适范围（~30°）大于向上（~25°）
- **习惯性偏好**：人类阅读、书写、手工操作等日常活动建立了根深蒂固的下视习惯

这使得向下方向的纯眼球运动范围更大、头部介入阈值更晚、各等级舒适区更宽。

**向上看（仰视）**

| 角度（从自然视线起） | 眼-头分配 | 协同特征 |
|---|---|---|
| θ < ~8° | 纯眼球运动 | 头部不动，眼球轻微上转即可 |
| 8° < θ < ~15° | 眼球为主，头部微调 | 眼球接近舒适上限，头部开始轻微仰起辅助 |
| 15° < θ < ~25° | 头部为主，眼球辅助 | 颈后伸肌群全面参与，头部承担大部分角度 |
| θ > ~25° | 头部接近极限 | 颈部后伸接近关节活动度上限，可能伴随不适 |

**向下看（俯视）**

| 角度（从自然视线起） | 眼-头分配 | 协同特征 |
|---|---|---|
| θ < ~12° | 纯眼球运动 | 头部不动，眼球自然下转（比上转范围更大） |
| 12° < θ < ~22° | 眼球为主，头部微调 | 重力辅助头部轻微下倾，几乎无额外肌肉负荷 |
| 22° < θ < ~35° | 头部为主，眼球辅助 | 颈部屈肌持续参与，但比同等仰头角度更轻松 |
| θ > ~35° | 头部接近极限 | 颈椎承压增大，XR 头显重心前移加剧负荷 |

**姿态依赖性**

与水平方向不同，垂直方向的眼-头协同**受身体姿态（坐姿/站姿）显著影响**。坐姿使向上区域收窄、向下区域扩展，具体表现为：

| 协同阶段 | 站姿 | 坐姿 | 差异原因 |
|---|---|---|---|
| 纯眼球范围（上/下） | 上 <8°；下 <12° | 同站姿 | 眼球运动不受姿态影响 |
| 头部舒适极限（上） | ~30° | ~25° | 坐姿时躯干固定，颈椎后伸空间受限 |
| 头部舒适极限（下） | ~40° | ~45° | 坐姿有靠背支撑，颈部屈曲负荷分散 |

> **关键结论**：坐姿使向上舒适范围缩减约 5°，向下舒适范围扩展约 5°。这一差异在长时间使用场景下对 UI 布局有实质影响。

## 5. 多层决策框架

### 5.1 框架概述：从扁平融合到分层决策

**旧方法的问题**

此前的方法将第 2、3、4 节的三组基础数据在同一角度轴上融合，取各体系质变区间的中间值作为复合区域边界。这种方法在近端（如 ±10°、±25°）尚可接受——因为三组数据的质变点彼此接近，中间值偏差不大。但到了远端（如 ±70°），问题变得显著：视野角质变在 ~60°，头部舒适度质变在 ~55°，眼-头协同转折也在 ~55°，强行求中间值得到 70°，与三个原始阈值的偏差达 10°~15°，失去了生理学意义上的解释力。

**新方法的核心洞察**

三组基础数据并非描述同一个问题的三个角度，而是描述不同层级设计问题的各自答案：

- **人眼视野角分区**回答的是"在某个离心度上能感知到什么"——这是一个纯视觉问题，与运动无关
- **头部转动舒适度**回答的是"保持某个头部角度能持续多久"——这是一个纯运动持续性问题
- **眼-头协同模型**回答的是"到达某个角度的运动模式是什么"——这是一个运动策略问题

将它们分配到不同的设计决策层级后，每种数据在其主导层使用原生阈值，不需要与其他体系的阈值对齐或求平均。

**四层决策结构**

| 层级 | 设计问题 | 主导基础数据 | 坐标参考系 |
|---|---|---|---|
| **第一层：环境设计** | 环境氛围元素如何在用户周围分布？ | 人眼视野角（第 2 节） | 以当前注视方向为中心 |
| **第二层：多交互对象的空间分布** | 多个共存窗口/面板如何在空间中排布？ | 眼-头协同模型（第 4 节）+ 头部舒适度（第 3 节） | 以身体正前方为中心 |
| **第三层：单个交互对象的内部布局** | 一个窗口应该多大？内部内容区块如何划分？ | 眼-头协同模型（第 4 节），缩小尺度应用 | 以窗口中心为原点（假设头部已面向窗口） |
| **第四层：UI 元素类型** | 在窗口内某个位置应放什么类型的 UI 元素？ | 人眼视野角（第 2 节） | 以当前注视点为中心 |

> **层间独立假设**：本框架当前假设各层独立决策。第三层和第四层的约束相对于"用户已转头面向该窗口"的状态生效，不考虑转头本身带来的累积颈部负荷。此局限性在第 5.7 节详细讨论。

---

### 5.2 第一层：环境设计

**设计问题**：环境氛围元素（背景装饰、空间分隔、光影氛围）应该在用户周围如何分布？

**主导基础数据**：人眼视野角分区（第 2 节）

**为什么由视野角主导**：环境元素不需要用户主动注视，它们通过周边视觉被"感知到存在"。决定环境元素能以什么形式被感知的，是视网膜在各离心度上的感知能力——这正是视野角分区直接描述的内容。头部转动舒适度和眼-头协同在这一层不起作用，因为用户不需要转头去"看"环境，环境是被动感知的。

#### 5.2.1 水平方向

视野角分区的原生边界（10°、30°、60°、100°）直接定义环境元素的设计约束：

| 角度范围 | 视野角分区 | 视觉感知能力 | 环境元素设计约束 |
|---|---|---|---|
| 0° ~ ±10° | 中央凹区 + 周中央凹 | 可辨识细节 | **交互对象的领地**——环境应退避，不在此范围内放置装饰元素，以免与交互内容争夺注意力 |
| ±10° ~ ±30° | 近周边视觉 | 可感知形状轮廓、色块、空间位置 | 可放置有辨识度的空间地标（如有清晰轮廓的建筑结构、标志性物体）、空间分区的显性边界（如墙面、门框） |
| ±30° ~ ±60° | 中周边视觉 | 仅感知运动和明暗对比，颜色辨识退化 | 适合色调区域、缓慢的光影变化、渐变色背景、空间分区的隐性暗示（如用不同色调区分功能区域） |
| ±60° ~ ±100°+ | 远周边视觉 | 仅对突发运动和强烈明暗变化有反应 | 纯氛围存在感——极缓慢的环境呼吸动画、大面积色彩渐变、包围感营造。避免快速运动或高对比度突变 |
| >±100° | 超出单眼视野 | 不可见（未转头时） | 可放置环境元素以保证空间完整感，但用户转身前不会感知到 |

#### 5.2.2 垂直方向

视野角分区以注视点为中心呈同心圆环分布，在垂直方向上同样适用，但受限于**垂直方向的物理视野边界更窄**（上方 ~60°，下方 ~70°–75°）：

| 角度范围 | 视觉感知能力 | 环境元素设计约束 |
|---|---|---|
| 0° ~ ±10°（上下对称） | 中央凹区 + 周中央凹 | 交互对象领地，环境退避 |
| ±10° ~ ±30° | 近周边视觉 | 可辨识的空间结构（如天花板纹理、地面材质边界） |
| 向上 +30° ~ +60° / 向下 -30° ~ -70° | 中周边 ~ 远周边 | 氛围性元素（天空渐变、地面光影）。向上方向在 +60° 即达物理极限，远周边区域在垂直方向上比水平方向更窄 |

> **垂直方向的特殊性**：由于垂直物理视野比水平方向窄（上方仅 ~60°，水平可达 ~100°），环境元素在垂直方向的可用空间更有限。上方环境元素主要通过"天空感"和"空间高度感"发挥作用；下方通过"地面纹理"和"空间锚定感"发挥作用。

---

### 5.3 第二层：多交互对象的空间分布

**设计问题**：多个共存的交互窗口/面板在空间中如何排布？主窗口、参考窗口、工具窗口、独立任务窗口分别放在哪里？

**主导基础数据**：眼-头协同模型（第 4 节）定义**窗口角色**，头部转动舒适度（第 3 节）定义**可持续时长**

**为什么由这两组数据共同主导**：多窗口布局的核心问题是"切换到那个窗口要付出多大运动代价"和"到了之后颈部能支撑多久"。前者由眼-头协同模型描述（到达某角度的运动模式），后者由头部舒适度描述（在某角度停留的可持续性）。这两个维度在同一角度位置提供独立的设计约束（角色 + 时长），设计师同时参考两者。

**为什么视野角分区不在这一层起作用**：多窗口布局关注的是"切换到那个窗口要付出什么代价"和"到了能停留多久"。视觉分辨率不是约束因素——一旦转头面向窗口，视野角分区会以新的注视点为中心重新生效（视网膜结构不随头部位置改变），窗口内的视觉能力与它在空间中的位置无关。

#### 5.3.1 水平方向

**维度 A：交互模式→窗口角色（来自眼-头协同模型，第 4.1 节）**

| 协同阶段 | 原生阈值 | 运动代价 | → 适合的窗口角色 |
|---|---|---|---|
| 纯眼动 | <~15° | 零切换成本，瞬间可达 | **主工作窗口**——用户注意力的大本营，承载当前核心任务 |
| 眼动 + 轻微头动 | ~15° ~ 30° | 极低切换成本，几乎无感 | **参考窗口**——频繁瞥视的辅助信息（如聊天记录、参数监视面板、文档参考） |
| 眼-头共同分担 | ~30° ~ 55° | 中等切换成本，需刻意发起 | **工具窗口**——偶尔调用的功能面板（如设置面板、文件浏览器、属性检查器） |
| 头部主导 | >~55° | 高切换成本，打断当前任务流 | **独立任务窗口**——与主任务平行但不频繁切换的内容（如后台监控、次要应用） |

**维度 B：可持续时长→交互密度上限（来自头部舒适度，第 3.1 节）**

| 舒适等级 | 原生阈值 | 可持续时间 | → 该位置窗口的交互密度上限 |
|---|---|---|---|
| 轻松 (Easy) | 0° ~ 20° | 数分钟以上 | 可承载高密度、长时间连续交互（文字编辑、数据录入） |
| 舒适 (Comfortable) | 20° ~ 35° | 数十秒 ~ 1 分钟 | 可承载中密度交互（查看信息、偶尔点击操作） |
| 适度 (Moderate) | 35° ~ 55° | 数秒 ~ 十余秒 | 仅适合短暂交互（查看-操作-回正的快速循环） |
| 费力 (Effortful) | 55° ~ 75° | 数秒 | 仅适合瞬间瞥视或单次点击 |
| 极限 (Extreme) | >75° | 瞬间 | 不适合放置需要交互的内容 |

**两个维度的协同使用**

两组阈值各自保持原生值，不求平均。设计师在决定某个角度位置适合放什么窗口时，同时查阅两个维度：

| 角度区间 | 交互模式（维度 A） | 持续性约束（维度 B） | → 综合布局建议 |
|---|---|---|---|
| 0° ~ ~15° | 纯眼动，零成本 | 轻松，可持续数分钟 | 核心工作区：主任务面板，支持长时间高密度交互 |
| ~15° ~ ~20° | 眼动 + 轻微头动 | 仍在轻松范围内 | 近侧参考区：频繁参考的辅助面板，头部微调无感，可长时间停留 |
| ~20° ~ ~30° | 眼动 + 轻微头动 | 跨入舒适，有意识参与 | 参考区：辅助面板，切换成本低但停留时长开始受限（数十秒级） |
| ~30° ~ ~35° | 眼-头共同分担 | 舒适末段 | 近侧工具区：偶尔调用的面板，需刻意转头但尚可舒适停留 |
| ~35° ~ ~55° | 眼-头共同分担 | 适度，颈部有工作感 | 工具区：短暂调用的功能面板，查看-操作-回正模式 |
| ~55° ~ ~75° | 头部主导 | 费力，数秒即不适 | 边缘区：独立任务窗口或极低频操作，仅瞬间访问 |
| >~75° | 超出颈部单独转动能力 | 极限 | 不建议放置交互窗口 |

> **坐姿调整**：坐姿因躯干受限，头部舒适转动范围从 ±35° 略缩至 ±30°，维度 B 的各阈值相应收窄约 5°，但维度 A 的眼-头协同模式不受姿态显著影响。

#### 5.3.2 垂直方向

垂直方向需分别处理向上和向下，且受姿态影响。

**向上（仰视）——窗口角色（来自眼-头协同模型，第 4.2 节）**

| 协同阶段 | 原生阈值 | → 适合的窗口角色 |
|---|---|---|
| 纯眼动 | <~8° | 主工作区的上方延伸，可无感扫视到达 |
| 眼动 + 轻微头动 | ~8° ~ 15° | 信息展示区——状态栏、通知区等不频繁但需要瞥视的内容 |
| 头部为主 | ~15° ~ 25° | 偶尔访问区——低频查看的信息或环境元素 |
| 头部接近极限 | >~25° | 不建议放置需要交互的窗口 |

**向上——可持续时长（来自仰头舒适度，第 3.2 节）**

| 舒适等级 | 站姿阈值 | 坐姿阈值 | 可持续时间 |
|---|---|---|---|
| 轻松 | 0° ~ 10° | 0° ~ 10° | 数分钟以上 |
| 舒适 | 10° ~ 20° | 10° ~ 15° | 数十秒 |
| 适度 | 20° ~ 30° | 15° ~ 25° | 数秒 ~ 十余秒 |
| 费力 | >30° | >25° | 数秒 |

**向下（俯视）——窗口角色（来自眼-头协同模型，第 4.2 节）**

| 协同阶段 | 原生阈值 | → 适合的窗口角色 |
|---|---|---|
| 纯眼动 | <~12° | 主工作区的下方延伸，手部操作自然区域 |
| 眼动 + 轻微头动 | ~12° ~ 22° | 操作面板区——手部交互内容的自然位置，重力辅助使头部微调几乎无代价 |
| 头部为主 | ~22° ~ 35° | 辅助操作区——低频交互的次要面板 |
| 头部接近极限 | >~35° | 不建议放置需要持续交互的窗口 |

**向下——可持续时长（来自低头舒适度，第 3.2 节）**

| 舒适等级 | 站姿阈值 | 坐姿阈值 | 可持续时间 |
|---|---|---|---|
| 轻松 | 0° ~ 15° | 0° ~ 15° | 数分钟以上 |
| 舒适 | 15° ~ 25° | 15° ~ 30° | 数十秒 ~ 1 分钟 |
| 适度 | 25° ~ 40° | 30° ~ 45° | 数秒 ~ 十余秒 |
| 费力 | >40° | >45° | 数秒 |

> **垂直方向的设计原则**：向下方向的纯眼动范围（~12°）大于向上（~8°），轻松维持的头部范围也更宽（向下 15° vs 向上 10°）。因此，在垂直方向上排布多窗口时，**向下延伸优于向上**——下方空间可以承载更多交互内容且舒适度更高。坐姿下这一优势更为明显（靠背支撑进一步延后向下的舒适阈值）。

### 5.4 第三层：单个交互对象的内部布局

**设计问题**：一个窗口/面板应该多大？内部内容区块如何划分——核心操作区、辅助信息区、边缘功能区各占多少角范围？

**主导基础数据**：眼-头协同模型（第 4 节），以窗口中心为原点，缩小尺度应用

**层间独立前提**：假设用户已转头面向该窗口。此时眼-头协同模型从窗口中心开始重新生效——窗口内部的扫视行为取决于从窗口中心到各个内容区域的角距离所对应的眼-头协同阶段。

**为什么不需要视野角分区**：在窗口内部扫视时，注视点在移动，中央凹始终跟随注视点。窗口内任何位置被注视时的视觉能力完全相同——决定内部布局差异的不是"看不看得清"，而是"扫视到那里要不要动头"。

**为什么不需要头部舒适度分区**：在层间独立假设下，窗口内的头部微调幅度很小（通常 <30°），处于头部舒适度的"轻松"和"舒适"范围内，不构成实质约束。

#### 5.4.1 水平方向

| 距窗口中心的角距离 | 眼-头协同模式 | → 窗口内的布局角色 |
|---|---|---|
| <~15° | 纯眼动 | **核心交互区**——主要操作入口、关键信息、高频控件。用户不需要任何头部运动即可自由扫视此区域 |
| ~15° ~ 30° | 眼动 + 轻微头动 | **扩展区**——辅助工具栏、次要信息、侧边栏。用户需要极轻微的头部微调来扫视到此区域，但几乎无感 |
| >~30° | 需要明显头部运动 | **窗口不应延伸到此范围**——超出此距离意味着窗口过大，应考虑拆分为多个独立窗口（回到第二层决策） |

**窗口角尺寸推荐**：

| 类型 | 推荐角宽度 | 最大角宽度 | 依据 |
|---|---|---|---|
| 主任务面板 | ~30°（±15°） | ~50°（±25°） | 核心操作在纯眼动范围内，扩展功能允许轻微头动 |
| 参考信息面板 | ~20°（±10°） | ~30°（±15°） | 参考面板应紧凑，避免在面板内还要大幅扫视 |
| 工具/设置面板 | ~20°（±10°） | ~30°（±15°） | 短暂调用的面板应尽可能紧凑以减少停留时间 |
| 媒体面板 | ~30°（±15°） | ~40°（±20°） | 参考 30° 的影院最佳观影角；3D 模型展示可略宽 |

#### 5.4.2 垂直方向

垂直方向需分别考虑窗口中心的上方和下方，利用眼-头协同的上下不对称性：

**窗口中心以上**

| 距窗口中心的向上角距离 | 眼-头协同模式 | → 布局角色 |
|---|---|---|
| <~8° | 纯眼动 | 核心区上半部分——标题区、关键信息（自然上瞥即可到达） |
| ~8° ~ 15° | 眼动 + 轻微仰头 | 扩展区——状态指示器、非关键信息头部 |
| >~15° | 明显仰头 | 窗口不应向上延伸至此 |

**窗口中心以下**

| 距窗口中心的向下角距离 | 眼-头协同模式 | → 布局角色 |
|---|---|---|
| <~12° | 纯眼动 | 核心区下半部分——操作按钮、输入区域（符合手部自然位置） |
| ~12° ~ 22° | 眼动 + 轻微低头 | 扩展区——补充操作、详细选项。重力辅助使此范围代价极低 |
| >~22° | 明显低头 | 窗口不应向下延伸至此 |

**窗口角高度推荐**：

| 类型 | 推荐角高度 | 最大角高度 | 依据 |
|---|---|---|---|
| 主任务面板 | ~20°（上 8° + 下 12°） | ~35°（上 13° + 下 22°） | 利用上下不对称性，下方分配更多空间 |
| 参考/工具面板 | ~15° | ~25° | 紧凑为宜 |
| 媒体面板 | ~20° | ~30° | 影院观感的最佳角高度 |

> **内部布局的非对称原则**：窗口内部在垂直方向上应"信息偏上、操作偏下"——标题/状态信息放在窗口上部（利用 8° 的向上纯眼动区），操作按钮/输入区放在窗口下部（利用 12° 的更宽向下纯眼动区，且符合手部自然操作位置）。

### 5.5 第四层：UI 元素类型

**设计问题**：在窗口内某个位置，应该放什么类型的 UI 元素——正文文字、大标题、图标、色块、动画指示？

**主导基础数据**：人眼视野角分区（第 2 节），以当前注视点为中心

**核心逻辑**：当用户注视窗口内某个锚点（如正在阅读的一行文字）时，周围不同离心度的视觉能力决定了临近区域能放置什么类型的 UI 元素。这一层的约束完全由视网膜解剖结构决定，**不受头部位置、窗口位置或姿态的任何影响**——无论窗口在正前方还是侧方 40°，无论用户站着还是坐着，只要注视点确定了，周围的视觉能力分布完全相同。

#### 5.5.1 以注视点为中心的 UI 元素分布

| 视野角分区 | 原生阈值（以注视点为中心） | 视觉能力 | → 可放置的 UI 元素类型 |
|---|---|---|---|
| **中央凹区** | 0° ~ 5° | 最高视锐度，可逐字阅读 | 正文文字、小号标签、精细控件（滑块刻度、输入光标、单选框）、需要辨认笔画的任何元素 |
| **周中央凹** | 5° ~ 10° | 视锐度约 30%，可识别熟悉的大型元素 | 大号标题、熟悉的图标（如关闭/最小化按钮）、高对比度区块标识、Tab 标签 |
| **近周边视觉** | 10° ~ 30° | 仅感知轮廓、色块、空间位置 | 存在性指示器（"这里有个按钮"但看不清标签）、区块轮廓与分隔线、色彩编码的状态指示、引导下一次扫视的视觉锚点 |

> 此分布以注视点为中心呈同心圆环结构，在水平、垂直和对角方向上均成立，**不需要按方向分别定义**。

#### 5.5.2 设计含义

**文字排版**：
- 需要连续阅读的正文文字块，其行宽应控制在注视点两侧 ±5° 以内。超出此范围后用户必须移动注视点（眼跳）才能继续阅读——这意味着行宽不是越宽越好
- 标题可以更宽（±10° 内），因为标题字号大、笔画粗，在周中央凹仍可识别

**控件布局**：
- 高频操作的控件群（如工具栏按钮）应紧凑排列在 ±10° 以内，使用户在注视其中一个控件时可以通过周中央凹识别相邻控件的类型
- 超出 10° 的控件应使用色彩编码或高对比度轮廓来增强存在性感知，因为用户在注视其他位置时只能看到模糊的形状

**信息密度梯度**：
- 窗口的信息密度应从注视锚点向外递减：中心 5° 内放最高密度信息，5°-10° 放中等密度的引导信息，10° 以外仅保留空间结构和存在性线索

### 5.6 四层决策流程总结

设计师使用本框架时，按以下顺序逐层决策：

```
┌─────────────────────────────────────────────────────────┐
│ 第一层：环境设计                                           │
│ 输入 → 人眼视野角分区（原生阈值：10°, 30°, 60°, 100°）       │
│ 决策 → 环境氛围元素的空间分布                                │
│        近周边（10°-30°）：可辨识的空间地标                    │
│        中周边（30°-60°）：色调与光影                         │
│        远周边（60°-100°+）：纯氛围存在感                     │
├─────────────────────────────────────────────────────────┤
│ 第二层：多窗口空间分布                                      │
│ 输入A → 眼-头协同模型（原生阈值：~15°, ~30°, ~55°）→ 窗口角色 │
│ 输入B → 头部舒适度（原生阈值：~20°, ~35°, ~55°, ~75°）→ 时长  │
│ 决策 → 不同角色窗口的空间位置分配                             │
│        垂直方向：向上阈值更紧、向下更宽；受姿态影响              │
├─────────────────────────────────────────────────────────┤
│ 第三层：单窗口内部布局                                      │
│ 输入 → 眼-头协同模型（缩小尺度，以窗口中心为原点）              │
│ 决策 → 窗口角尺寸上限 + 核心区/扩展区划分                     │
│        水平：核心 ±15°，最大 ±30°                           │
│        垂直：上方核心 8°/下方核心 12°（利用不对称性）            │
├─────────────────────────────────────────────────────────┤
│ 第四层：UI 元素类型                                        │
│ 输入 → 人眼视野角分区（以注视点为中心，原生阈值：5°, 10°）      │
│ 决策 → 各位置的 UI 元素类型                                 │
│        中央凹（0°-5°）：正文文字、精细控件                     │
│        周中央凹（5°-10°）：大标题、图标                       │
│        近周边（10°-30°）：存在性指示器                        │
└─────────────────────────────────────────────────────────┘
```

**为什么这种分层避免了旧方法的偏差问题**：

旧方法在远端（如 ±70°）产生大偏差，是因为要求三个系统在同一角度轴上"对齐"——视野角质变在 ~60°、头部舒适度质变在 ~55°、眼-头协同转折在 ~55°，强行求中间值得到 70°，与三个原始阈值都不接近。

在新框架中：
- 60° 就是 60°——它在第一层环境设计中作为"中周边→远周边"的原生边界使用
- 55° 就是 55°——它在第二层多窗口布局中分别作为眼-头协同"共同分担→头部主导"的转折点和头部舒适度"适度→费力"的阈值使用
- 每个数字在其所属层中保持原始生理学含义，不被其他系统的数字"拉扯"

### 5.7 已知局限

#### 局限一：层间耦合缺失（累积负荷传递）

当前框架假设各层独立决策。但实际上存在累积效应：

一个放在身体正前方偏右 40° 的工具窗口（第二层决定其位置），当用户转头 40° 面向它时，颈部已处于"适度"负荷状态。此时窗口内部的第三层约束应更严格——虽然眼球的舒适偏转范围不受影响（眼球不关心颈部状态），但窗口内的头部微调会叠加到已有的 40° 颈部负荷上，使"轻微头动"变得不那么轻微。

**潜在改进方向**：引入"累积负荷修正系数"——根据窗口在第二层的空间位置，对第三层的窗口角尺寸上限和核心区范围进行折减。例如，位于 40° 处的窗口，其内部核心区半径可能需要从 ±15° 收缩到 ±12°。具体修正系数需要实验验证。

#### 局限二：垂直方向的姿态动态性

第二层垂直方向的约束受姿态（坐/站）显著影响，但实际使用中用户可能在坐姿和站姿之间切换。当前框架提供了两种姿态的独立参数，但未定义姿态切换时的动态调整策略。

**建议**：姿态未知或混合场景下，取坐/站姿的保守交集（取各阈值中更严格的一方）。若应用可检测用户姿态（通过头显 IMU 或座椅传感器），可动态切换参数。

#### 局限三：第一层与第二层的空间重叠

环境元素（第一层）与最外围的交互窗口（第二层）可能在 ±30° ~ ±55° 的角度区间内空间重叠。第一层认为 ±30° 以外可以放置色调类环境元素，第二层认为 ±30° ~ ±55° 适合放置工具窗口。

**建议**：在重叠区间内，交互窗口优先级高于环境元素。环境元素应在窗口周围留出视觉缓冲区，避免与窗口边界直接相邻。可以利用环境的色调/光影来暗示窗口的功能分区（如工具区用偏冷色调环境）。

### 6.2 内容类型详细策略

#### 主交互面板（菜单、工具栏、输入表单）

- **第二层**：放置在身体正前方的纯眼动区（<15°），垂直偏下方——符合手部自然操作位置和桌面交互心智模型
- **第三层**：核心操作控件集中在窗口中心 ±15°，辅助功能向两侧扩展区展开，最大宽度 ±25°。垂直方向利用不对称性，将操作区放在窗口下部（下方纯眼动范围更大）
- **第四层**：高频操作按钮的标签文字应在用户注视该按钮时处于中央凹区（±5°），相邻按钮应在周中央凹（±10°）内以便识别

#### 参考信息面板（文档、聊天、数据看板）

- **第二层**：放在眼动+轻微头动区（15°~30°），使用户从主面板瞥视过来几乎无感。垂直保持在自然视线附近（V0），信息展示可略偏上（+5°~+8°）
- **第三层**：控制宽度在 ±10°~±15°，确保核心内容在纯眼动范围内可扫视完毕
- **第四层**：需持续阅读的文字必须在中央凹区（±5°）——这限制了单行文字的有效宽度。数据看板的核心指标在中央凹区，次要指标在周中央凹

#### 工具/设置面板（设置、文件浏览器、属性检查器）

- **第二层**：放在眼-头共同分担区（30°~55°），头部舒适度在"适度"范围内（35°~55°），仅支持数秒至十余秒的停留——适合"打开-操作-关闭"的短暂交互模式
- **第三层**：极紧凑设计，宽度 ±10°，减少用户在面板内的扫视需求，缩短停留时间
- **第四层**：由于停留时间短，应使用更大号的图标和标签以加速识别。控件间距适当增大，降低精确点击难度

#### 通知/提示（系统消息、警告、状态更新）

- **第二层**：放在核心区的上方边缘（水平 10°~30°，垂直 +5°~+10°），利用近周边视觉的"存在性感知"和上方出现新物体的"跳出感"
- **第三层**：极小型，角尺寸尽量控制在 5°~10° 以内
- **第四层**：使用高对比度色块（近周边可感知）+ 大号精简文字（注视后在中央凹区可阅读）

#### 媒体内容（视频播放、3D 模型展示）

- **第二层**：放在身体正前方纯眼动区的中心（水平 <10°，垂直 ±5°），持续观看需最大舒适度
- **第三层**：视频面板宽度 ±15°（参考 30° 的影院最佳观影角），3D 模型展示可略宽至 ±20°
- **第四层**：媒体内容本身为连续视觉流，UI 控件（播放/暂停）在注视时位于中央凹区下方

#### 导航元素（空间锚点、路径指引、方向标识）

导航元素横跨第一层和第二层，是两者的交叉地带：
- **第一层（作为环境元素）**：空间锚点在近周边视觉区（10°~30°）以有辨识度的轮廓和色块存在，帮助用户建立空间方位感
- **第二层（作为交互目标）**：方向性指引可分布在眼-头共同分担区（30°~55°）甚至头部主导区，因为其目的是引导用户转向——转头本身就是预期行为
- **第四层**：配合运动线索（动画轨迹、脉冲闪烁）利用周边视觉的运动敏感性引导注意

## 7. 设计规则与策略

### 7.1 通用规则

| # | 规则 | 对应层级 | 理由 |
|---|---|---|---|
| R1 | **主工作窗口限制在纯眼动区（水平 ±15°）** | 第二层 | 主任务应保持零头部运动代价 |
| R2 | **高频交互窗口限制在水平 ±30° 以内** | 第二层 | 超出此范围的频繁切换需刻意转头，导致颈部疲劳 |
| R3 | **持续注视内容（阅读、视频）居中放置** | 第二层 | 持续注视需最低运动代价的位置 |
| R4 | **垂直布局遵循"信息上、操作下"原则** | 第三层 | 符合重力方向的认知直觉和手部自然操作位置 |
| R5 | **利用垂直不对称性：向下延伸优于向上** | 第二层 + 第三层 | 向下纯眼动范围更大（12° vs 8°），头部舒适度更高 |
| R6 | **正文文字行宽控制在注视点 ±5° 以内** | 第四层 | 超出中央凹区后无法逐字阅读 |
| R7 | **通知出现在主任务上方边缘（近周边区）** | 第二层 | 上方出现的变化更易引起注意，近周边可感知存在 |
| R8 | **环境元素避免使用快速运动或高对比度突变** | 第一层 | 周边视觉对运动和明暗变化极敏感，会持续干扰注意力 |
| R9 | **窗口最大角宽度不超 ±30°（水平）、上 15° + 下 22°（垂直）** | 第三层 | 超出此范围窗口内扫视需明显头部运动，应拆分窗口 |
| R10 | **兼顾两种姿态时，取更保守的阈值** | 所有层级 | 确保同一 UI 在坐/站姿下都保持舒适 |

### 7.2 层内过渡策略

**第一层（环境）**：从交互区向外围，环境元素的信息量应逐渐降低——近周边区可有辨识度的结构，中周边区渐变为色调与光影，远周边区仅保留极简氛围。避免在区域边界产生突兀的视觉断层。

**第二层（多窗口）**：从主工作区向外围，窗口的交互密度和信息量应逐渐降低。主工作窗口→参考窗口→工具窗口→独立任务窗口，形成清晰的空间层次。窗口之间应留出视觉间隔，利用环境元素（第一层）作为自然分隔。

**第三层（窗口内部）**：从窗口中心向边缘，信息密度和交互频率应递减——核心操作居中，辅助功能向边缘展开。窗口边缘不放置关键内容。

**第四层（UI 类型）**：以用户常驻注视锚点为中心，最高密度信息在中央凹区，向外逐层简化。避免在距注视锚点 10° 以外放置需要阅读的文字。

### 7.3 Head-locked UI 附加建议

虽然 Head-locked UI 始终在视野内（随头部转动），但第四层的视野角分区仍然约束其放置位置：

| 策略 | 依据层级 | 说明 |
|---|---|---|
| 避开中央凹区（±5°） | 第四层 | Head-locked 内容不应遮挡正中心的注视区，否则持续干扰精细阅读 |
| 放在周中央凹边缘（约 8°~10°） | 第四层 | 在周中央凹可被识别但不遮挡核心内容 |
| 透明度与按需清晰化 | 第一层逻辑 | 类似环境元素的处理——非注视时保持低透明度，注视时逐渐变清晰 |
| 极简信息量 | 通用 | 永远在视野中的 UI 必须极度精简以避免视觉疲劳 |

| 0° ~ +8° | 纯眼动 | 轻松 | 轻松 | 信息展示、通知区 |
| +8° ~ +10° | 眼动+轻微仰头 | 轻松 | 轻松 | 状态指示器 |
| +10° ~ +15° | 眼动+轻微仰头 | 舒适 | 舒适→适度 | 低频信息（站姿可，坐姿慎用） |
| +15° ~ +20° | 头部为主 | 舒适 | 适度 | 偶尔访问（坐姿下已勉强） |
| +20° ~ +25° | 头部为主 | 适度 | 费力 | 站姿偶尔访问，坐姿避免 |
| >+25° | 头部接近极限 | 适度→费力 | 费力 | ⛔ 避免放置交互内容 |

#### 垂直方向——向下（俯视）

| 角度 | 交互模式 | 站姿舒适度 | 坐姿舒适度 | 适合内容 |
|---|---|---|---|---|
| 0° ~ -12° | 纯眼动 | 轻松 | 轻松 | 操作面板、手部交互 |
| -12° ~ -15° | 眼动+轻微低头 | 轻松 | 轻松 | 操作扩展区 |
| -15° ~ -22° | 眼动+轻微低头 | 舒适 | 轻松→舒适 | 辅助面板（坐姿更舒适） |
| -22° ~ -25° | 头部为主 | 舒适 | 舒适 | 次要面板 |
| -25° ~ -35° | 头部为主 | 适度 | 舒适→适度 | 低频操作（坐姿更宽裕） |
| -35° ~ -40° | 头部接近极限 | 适度→费力 | 适度 | 坐姿可偶尔访问，站姿慎用 |
| >-40° | 头部接近极限 | 费力 | 适度→费力 | ⛔ 避免放置交互内容 |

### 8.3 第三层速查：窗口内部布局（眼-头协同原生阈值）

| 方向 | 纯眼动核心区 | 眼动+轻微头动扩展区 | 窗口角尺寸上限 |
|---|---|---|---|
| 水平（左右对称） | ±15° | ±15° ~ ±30° | 最大 ~60° 宽 |
| 向上（距窗口中心） | 0° ~ 8° | 8° ~ 15° | 最大 ~15° 上延 |
| 向下（距窗口中心） | 0° ~ 12° | 12° ~ 22° | 最大 ~22° 下延 |

推荐窗口角高度：上 8°~13° + 下 12°~22° ≈ **总高 20°~35°**

### 8.4 第四层速查：UI 元素类型（视野角原生阈值，以注视点为中心）

| 视野角分区 | 角半径 | 可放置的 UI 元素 |
|---|---|---|
| 中央凹区 | 0° ~ 5° | 正文文字、精细控件、需要辨认笔画的任何元素 |
| 周中央凹 | 5° ~ 10° | 大号标题、熟悉图标、高对比度区块标识 |
| 近周边 | 10° ~ 30° | 存在性指示器、区块轮廓、色彩编码状态 |

### 8.5 姿态偏移修正值

在真实水平线坐标系中，需加上以下偏移量：

| 姿态 | 自然视线偏移 | 应用方式 |
|---|---|---|
| 站姿 | 真实水平线 -10° | 所有垂直区域角度 + (-10°) 得到相对水平线的绝对角度 |
| 坐姿 | 真实水平线 -15° | 所有垂直区域角度 + (-15°) 得到相对水平线的绝对角度 |

### 9.1 术语表

| 术语 | 定义 |
|---|---|
| FOV (Field of View) | 视野范围，从注视点到视觉感知边界的角度 |
| Foveal Zone | 中央凹区，本规范指注视点周围 0°–5° 的高视锐度区域（含解剖学上的中央凹 Fovea 与近中央凹 Parafovea） |
| Perifoveal Vision | 周中央凹视觉，中央凹区外围 5°–10° 的过渡区域 |
| Peripheral Vision | 周边视觉，黄斑区以外的视觉区域（>10°） |
| Yaw | 偏航角，头部水平转动 |
| Pitch | 俯仰角，头部垂直转动 |
| Natural Resting Gaze | 自然静息视线，放松状态下的默认视线方向 |
| Body-locked | 相对用户身体/躯干锚定的 UI |
| World-locked | 相对世界坐标系固定的 UI |
| Head-locked | 相对用户头部/视线锚定的 UI |
| VAC | 辐辏-调节冲突（Vergence-Accommodation Conflict），本规范不涉及 |
