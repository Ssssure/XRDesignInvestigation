# XR 空间 UI 布局设计规范

## 基于人眼视野角与头部转动舒适度的复合区域模型

---

## 1. 概述

本规范为 XR 头显环境中虚拟交互内容的空间布局提供设计指导。通过将**人眼视野角分区**与**头部转动舒适度分区**融合为一个复合区域模型，定义不同角度区域内用户的身体感受，并为每个区域提供明确的内容放置策略。

### 1.1 适用范围

| 属性 | 说明 |
|---|---|
| 平台 | XR 头显（通用，不绑定特定硬件） |
| 坐标参考 | Body-locked 和 World-locked UI（附带 Head-locked 建议） |
| 覆盖姿态 | 坐姿与站姿 |
| 维度 | 水平角度（Yaw）+ 垂直角度（Pitch），不含深度（Z 轴） |
| 内容类型 | 主交互面板、信息展示、通知提示、导航元素、媒体内容、环境氛围 |

### 1.2 坐标系定义

本规范采用以下坐标系约定：

- **原点**：用户头部中心位置
- **水平角度（Azimuth / Yaw）**：0° 为正前方，向左为负，向右为正，范围 -180° ~ +180°
- **垂直角度（Elevation / Pitch）**：以**自然静息视线**为 0° 基准，向上为正，向下为负
- **自然静息视线**：站姿约低于真实水平线 10°，坐姿约低于真实水平线 15°

> **为什么以自然静息视线为垂直基准？** 人在放松状态下的视线并非水平前视，而是自然下倾。以此为基准，能更准确地反映用户在 XR 环境中的真实感受，使得"0° 垂直"对应"毫无努力的自然注视方向"。

## 2. 基础分区：人眼视野角

人眼视野（Field of View）并非均匀的感知区域，而是从中心向外逐级退化的分层结构。

### 2.1 视野角功能分区

视网膜的中央区域——黄斑 (Macula)——覆盖注视点周围约 10° 半径，包含解剖学上的中央凹 (Fovea)、近中央凹 (Parafovea) 和周中央凹 (Perifovea) 三层同心结构，合称"中央视觉"。由于中央凹与近中央凹对 XR UI 设计的约束趋同（详见下方说明），本规范将两者合并为"中央凹区"，与周中央凹分别处理。超出黄斑区即进入周边视觉。

| 分区 | 角半径（从注视点） | 视觉能力 | XR 设计意义 |
|---|---|---|---|
| **中央凹区 (Foveal Zone)** | 0° ~ 5° | 最高视锐度，可分辨文字笔画与精细图形细节，颜色感知完整 | **精细阅读区**——适合正文文字、小型 UI 控件（按钮标签、输入光标、滑块刻度）。超出 ±5° 后无法逐字阅读 |
| **周中央凹 (Perifoveal)** | 5° ~ 10° | 视锐度降至中央凹区的约 30%；无法辨认正文字号的笔画，但可凭轮廓与对比度识别熟悉的大型元素 | 适合大号标题、熟悉图标、高对比度区块，无需眼跳即可判断元素类型。超出 ±10° 后无法识别具体内容（整个 0°–10° 构成**有效识别区**） |
| **近周边视觉 (Near Peripheral)** | 10° ~ 30° | 无法识别文字或图标的具体内容，仅可感知形状轮廓、色块与空间位置 | 扫视目标定位（"那边有个东西"）、UI 区块存在性感知。±30° 是不转头获取信息的实际极限（**功能视野边界**） |
| **中周边视觉 (Mid Peripheral)** | 30° ~ 60° | 静态细节几乎不可辨，以运动检测和明暗对比为主；颜色辨识退化 | 动态变化察觉（动画、闪烁、弹出）、空间方位感。±60° 以外仅对突发运动有反应（**运动感知边界**） |
| **远周边视觉 (Far Peripheral)** | 60° ~ 100°+ | 仅对突发运动和强烈明暗变化有反应；>60° 主要为单眼视野 | 威胁/事件警觉、空间存在感与包围感 |

> **关于分区的方向性**：以上分区为以注视点为圆心的**同心圆环结构**，在水平、垂直和对角方向上均成立——同一角半径处的视觉能力不随方向变化。方向性差异仅体现在视野的**物理外边界**上：水平方向单眼视野可达颞侧 ~100°、鼻侧 ~60°，而垂直方向仅为上方 ~60°、下方 ~70°–75°。这意味着远周边视觉区在垂直方向上更早到达物理极限。

> **关于中央凹区的合并**：解剖学上，0°–2° 的中央凹 (Fovea) 与 2°–5° 的近中央凹 (Parafovea) 在视网膜层结构上存在差异（神经节细胞层厚度不同），视锐度也呈梯度递减。但对 XR UI 设计而言，两者均处于高视锐度区域，都能支持正文文字的逐字阅读，设计约束一致（可使用的最小字号、控件尺寸等无需差异化），因此本规范将其合并为"中央凹区"。5° 是一条关键的设计分界线——超过此角度后用户**无法在不移动注视点的情况下阅读正文文字**，这一断崖式变化对 UI 元素的尺寸和信息密度有直接影响。

## 3. 基础分区：头部转动舒适度

头部转动涉及颈椎关节的运动，不同幅度的转动对应不同的肌肉负荷和主观感受。

### 3.1 水平转动（Yaw）舒适度分级

| 等级 | 转动幅度 | 肌肉参与 | 主观感受 | 可持续时间 |
|---|---|---|---|---|
| **轻松 (Easy)** | 0° ~ 20° | 无至轻微；颈部深层肌群低水平激活 | 从无意识的微调到自然的随意转头，类似围坐聊天时目光在几位朋友之间扫视；用户不会将转头本身感知为刻意动作 | 数分钟以上 |
| **舒适 (Comfortable)** | 20° ~ 35° | 中等；胸锁乳突肌与头夹肌主动参与 | 有转头意识但不费力，类似在多屏工作台前转头看旁边的显示器；颈部运动流畅，无肌肉紧张感 | 数十秒 ~ 1分钟 |
| **适度 (Moderate)** | 35° ~ 55° | 较明显；颈部旋转肌群全面参与，上斜方肌开始协同 | 需要刻意转头，类似开车变道时转头查看盲区；颈部有明确的肌肉工作感，保持数秒后会想回正 | 数秒 ~ 十余秒 |
| **费力 (Effortful)** | 55° ~ 75° | 显著；颈部肌群高负荷，上斜方肌与肩胛提肌明显紧张 | 明显的颈部紧张，类似课堂上回头看后排同学；肩部可能跟随转动，持续数秒即产生不适 | 数秒 |
| **极限 (Extreme)** | >75° | 高强度；颈部肌群接近极限，肩部与躯干旋转肌群介入 | 接近生理极限，类似试图不转身地回头看正后方；需要肩部和躯干辅助旋转，仅能瞬间维持 | 瞬间 |

### 3.2 垂直转动（Pitch）舒适度分级

垂直方向存在**显著的上下不对称性**——向下看比向上看更舒适，因为颈椎的屈曲（低头）比伸展（仰头）的肌肉负荷更低，且符合日常习惯（阅读、走路时自然下视）。

#### 向上（仰头）

| 等级 | 转动幅度（从自然视线起） | 肌肉参与 | 主观感受 | 可持续时间 |
|---|---|---|---|---|
| **轻松** | 0° ~ 10° | 无至轻微；枕下肌群微调 | 自然上瞥，类似从桌面抬眼看对面人的脸；无颈部感知 | 数分钟以上 |
| **舒适** | 10° ~ 20° | 轻度；头夹肌与头半棘肌主动收缩 | 轻微仰头，类似抬头看墙上挂钟；颈后有轻微拉伸感但不产生负担 | 数十秒 |
| **适度** | 20° ~ 30° | 中等；颈后伸肌群全面参与，上斜方肌协同 | 明显仰头，类似站在展厅抬头看高处画作；颈后肌群持续紧张，会主动想回正 | 数秒 ~ 十余秒 |
| **费力** | >30° | 显著；颈后肌群高负荷，接近关节活动度极限 | 大幅仰头，类似仰望天花板；颈后疲劳且颈椎伸展接近极限，可能伴随轻微头晕 | 数秒 |

#### 向下（低头）

| 等级 | 转动幅度（从自然视线起） | 肌肉参与 | 主观感受 | 可持续时间 |
|---|---|---|---|---|
| **轻松** | 0° ~ 15° | 无至轻微；重力辅助，颈深层屈肌（颈长肌、头长肌）轻度控制 | 自然下瞥，类似低头看手中的书；完全符合日常阅读习惯，比平视更放松 | 数分钟以上 |
| **舒适** | 15° ~ 25° | 轻度；颈深层屈肌持续参与，后侧肌群提供离心控制 | 轻微低头，类似坐着看桌面上的手机；有轻微负荷但仍感舒适 | 数十秒 ~ 1分钟 |
| **适度** | 25° ~ 40° | 中等；胸锁乳突肌双侧参与，颈后伸肌群承受离心负荷 | 有意识低头，类似站着查看腰间工牌；颈前侧和肩部有持续负荷感 | 数秒 ~ 十余秒 |
| **费力** | >40° | 显著；颈部屈伸肌群均高负荷，椎间盘承压增大 | 过度低头，类似长时间弯腰系鞋带的姿势；颈椎承压显著，XR 头显重心前移会加剧不适 | 数秒 |

### 3.3 坐姿与站姿差异

| 参数 | 站姿 | 坐姿 | 影响 |
|---|---|---|---|
| 自然静息视线（相对真实水平线） | 约 -10° | 约 -15° | 坐姿时内容整体应略低放置 |
| 水平舒适转动范围 | ±35° | ±30° | 坐姿因躯干受限，水平舒适范围略小 |
| 向下舒适范围 | -25° | -30°（含靠背支撑优势） | 坐姿有靠背支撑，低头反而略轻松 |
| 向上舒适范围 | +20° | +15° | 坐姿仰头受限，上方区域更应谨慎 |

> **设计启示**：为兼容两种姿态，应以更保守的值为准（即取两者的交集），或在应用中检测用户姿态并动态调整 UI 位置。

## 4. 复合区域模型

将视野角分区与头部转动舒适度分区叠加，形成以下复合区域模型。该模型描述的是用户注视某个角度方向上的 UI 元素时，**眼球运动与头部转动的协同代价总和**。

### 4.1 水平方向

#### 4.1.1 水平眼-头协同

当用户需要观看偏离正前方某个水平角度 θ 的内容时，视觉系统会自动将这个角度分配给眼球转动和头部转动：

| 水平偏移角 θ | 眼-头分配 | 协同特征 |
|---|---|---|
| θ < ~15° | 纯眼球运动 | 头部保持不动，眼球在近周边范围内自然扫视 |
| 15° < θ < ~30° | 眼球为主，头部微调 | 头部开始轻微偏转辅助，用户几乎无感 |
| 30° < θ < ~55° | 眼球与头部共同分担 | 头部成为主要贡献者，颈部肌群明确参与 |
| θ > ~55° | 头部主导，接近极限 | 头部达到舒适极限，需要更大幅度转动或肩/躯干参与 |

水平方向的眼-头协同具有良好的**左右对称性**——向左和向右的运动代价基本一致，且**不受身体姿态（坐/站）显著影响**（坐姿仅因躯干受限使舒适转动范围从 ±35° 略缩至 ±30°，协同模式不变）。

这种协同机制意味着：复合区域的边界不是视野角和头部转动的简单相加，而是反映了两者协同工作时的总体舒适感受。

**从三要素到区域边界的推导**

水平负荷区域的边界由**视觉能力分区**（第 2 节）、**头部转动舒适度**（第 3.1 节）和**眼-头协同模式**（上表）三个维度共同决定。每条边界设在三个维度的质变点尽可能对齐的角度上：

| 区域边界 | 视觉能力（第 2 节） | 头部舒适度（第 3.1 节） | 眼-头协同（上表） | → 综合体验转折 |
|---|---|---|---|---|
| **±10°（H0→H1）** | 超出周中央凹——有效识别区上限；10° 以外无法在不移动注视点的情况下识别具体内容 | 无头部运动 | 纯眼球范围内（<15°） | 视觉从"可辨识细节"骤降为"仅感知存在"，尽管头部无需运动，信息获取能力已发生质变 |
| **±25°（H1→H2）** | 近周边视觉中段——仅能感知形状轮廓与色块 | 跨越轻松→舒适阈值（~20°）：头部开始有意识参与 | 纯眼球（<15°）→眼球为主+头部微调（15°~30°）的过渡带中段 | 头部从"无感"进入"有意识参与"，同时眼球已需头部辅助——零运动代价区终止 |
| **±45°（H2→H3）** | 近周边→中周边过渡（~30° 后静态辨识消失，仅检测运动） | 跨越舒适→适度阈值（~35°），进入适度中段：颈部有明确肌肉工作感 | 共同分担（30°~55°）阶段中段，头部已成为主要贡献者 | 视觉退至仅能检测运动 + 头部运动需刻意发起——"随意一瞥"变为"刻意转头" |
| **±70°（H3→H4）** | 中周边→远周边过渡（~60° 后仅对突发运动有反应） | 跨越适度→费力阈值（~55°），进入费力上段：颈部肌群高负荷 | 头部主导（>55°），接近舒适极限 | 转头前几乎无有效视觉 + 转头本身产生明显肌肉负荷——超出多数用户愿意主动执行的范围 |
| **±100°（H4→H5）** | 接近单眼视野物理极限（颞侧 ~100°） | 超越极限阈值（>75°），需肩部/躯干介入 | 超出颈部单独转动能力 | 视觉和颈部运动均到达物理极限——只能通过身体转向到达 |

> **推导逻辑**：区域边界并非精确对应某个单一生理阈值，而是设在三个维度的"质变区间"重叠的角度上。例如 ±25° 不等于眼-头协同阈值（~15°）也不等于头部舒适阈值（~20°），而是取两者转折带的重叠中段——此角度下头部刚跨入"有意识运动"且眼球运动开始需要头部辅助，用户整体感受从"毫不费力"过渡为"略有感知"。这种多维度对齐使每条边界对应一个清晰的**体验质变**。

#### 4.1.2 水平负荷区域定义

以下定义以正前方（0°）为中心，左右对称。

---

#### H0 — 静默焦点区 (Silent Focus Zone)

| 属性 | 值 |
|---|---|
| **角度范围** | 0° ~ ±10° |
| **眼球运动** | 中央凹区 + 周中央凹范围内，微小扫视 |
| **头部运动** | 无 |
| **身体感受** | 完全无感，纯认知沉浸状态。用户甚至不会意识到自己在"看"某个位置，注意力完全投入内容本身。相当于将书本举在眼前正中的体验。 |
| **视觉能力** | 最高分辨率，可阅读小号文字、辨识精细图标 |

---

#### H1 — 核心舒适区 (Core Comfort Zone)

| 属性 | 值 |
|---|---|
| **角度范围** | ±10° ~ ±25° |
| **眼球运动** | 周中央凹至近周边，自然扫视 |
| **头部运动** | 无至极微（<10°） |
| **身体感受** | 轻松自在，类似于坐在书桌前目光自然扫视桌面的感觉。无任何肌肉紧张感，可以长时间保持。用户能在此区域内自由浏览而不产生疲劳。 |
| **视觉能力** | 高分辨率，可阅读正常字号文字，识别标准 UI 控件 |

---

#### H2 — 自然转视区 (Natural Glance Zone)

| 属性 | 值 |
|---|---|
| **角度范围** | ±25° ~ ±45° |
| **眼球运动** | 近周边视觉 + 头部协同 |
| **头部运动** | 轻度至中度（15° ~ 30°） |
| **身体感受** | 类似于坐在会议桌旁偏头看身边同事的感觉。头部转动是自然的、不费力的，但用户能清晰感知到"我转了头"。可以在此方向停留数十秒而不感到不适，但不适合长时间凝视。 |
| **视觉能力** | 可识别大型文字、图标和色块，精细阅读需要将头部转向以使内容进入 H0/H1 |

---

#### H3 — 刻意转头区 (Deliberate Turn Zone)

| 属性 | 值 |
|---|---|
| **角度范围** | ±45° ~ ±70° |
| **眼球运动** | 中周边视觉，主要依赖头部对准 |
| **头部运动** | 中度至大幅（35° ~ 55°） |
| **身体感受** | 类似于驾车时偏头看侧后视镜的感觉。需要有意识地决定"我要看那边"，颈部肌肉有明确的工作感。保持数秒可接受，但超过十几秒开始产生轻微疲劳感。用户的主要任务焦点会因转头而中断。 |
| **视觉能力** | 未转头时仅能检测运动/变化；转头对准后恢复正常视觉能力 |

---

#### H4 — 极限边缘区 (Edge Reach Zone)

| 属性 | 值 |
|---|---|
| **角度范围** | ±70° ~ ±100° |
| **眼球运动** | 远周边，转头前几乎无有效视觉 |
| **头部运动** | 大幅至接近极限（55° ~ 75°+），肩部可能参与 |
| **身体感受** | 类似于听到身后有人叫你名字时的大幅度回头。颈部肌肉明显紧张，可能伴随轻微的肩部转动。这是一个"为了看到内容不得不付出显著努力"的区域。仅适合瞬间瞥视，长时间注视会产生颈部酸痛。 |
| **视觉能力** | 转头前仅限运动检测；需要大幅转头才能有效获取信息 |

---

#### H5 — 身体转向区 (Body Turn Zone)

| 属性 | 值 |
|---|---|
| **角度范围** | >±100° |
| **眼球运动** | 完全超出有效视野 |
| **头部运动** | 超出颈部单独转动能力，需躯干旋转 |
| **身体感受** | 相当于有人在你身后说话，你需要整个人转过去面对他。这不再是"视觉行为"而是"身体行为"，会完全打断当前任务、改变身体朝向、重置空间定位。在 XR 中尤其需要避免，因为频繁的身体旋转会增加线缆缠绕风险（有线头显）、加重眩晕感、并破坏空间认知模型。 |
| **视觉能力** | 转身前完全不可见 |

---

### 4.2 垂直方向

#### 4.2.1 垂直眼-头协同

垂直方向的眼-头协同与水平方向遵循相同的渐进分担原则，但存在两个关键差异：**上下不对称性**和**姿态依赖性**。

**上下不对称性**

向下看比向上看更轻松，原因包括：

- **重力辅助**：低头时重力帮助头部下倾，颈部肌肉负荷更低
- **眼球运动不对称**：眼球向下旋转的舒适范围（~30°）大于向上（~25°）
- **习惯性偏好**：人类阅读、书写、手工操作等日常活动建立了根深蒂固的下视习惯

这使得向下方向的纯眼球运动范围更大、头部介入阈值更晚、各等级舒适区更宽。

**向上看（仰视）**

| 角度（从自然视线起） | 眼-头分配 | 协同特征 |
|---|---|---|
| θ < ~8° | 纯眼球运动 | 头部不动，眼球轻微上转即可 |
| 8° < θ < ~15° | 眼球为主，头部微调 | 眼球接近舒适上限，头部开始轻微仰起辅助 |
| 15° < θ < ~25° | 头部为主，眼球辅助 | 颈后伸肌群全面参与，头部承担大部分角度 |
| θ > ~25° | 头部接近极限 | 颈部后伸接近关节活动度上限，可能伴随不适 |

**向下看（俯视）**

| 角度（从自然视线起） | 眼-头分配 | 协同特征 |
|---|---|---|
| θ < ~12° | 纯眼球运动 | 头部不动，眼球自然下转（比上转范围更大） |
| 12° < θ < ~22° | 眼球为主，头部微调 | 重力辅助头部轻微下倾，几乎无额外肌肉负荷 |
| 22° < θ < ~35° | 头部为主，眼球辅助 | 颈部屈肌持续参与，但比同等仰头角度更轻松 |
| θ > ~35° | 头部接近极限 | 颈椎承压增大，XR 头显重心前移加剧负荷 |

**姿态依赖性**

与水平方向不同，垂直方向的眼-头协同**受身体姿态（坐姿/站姿）显著影响**。坐姿使向上区域收窄、向下区域扩展，具体表现为：

| 协同阶段 | 站姿 | 坐姿 | 差异原因 |
|---|---|---|---|
| 纯眼球范围（上/下） | 上 <8°；下 <12° | 同站姿 | 眼球运动不受姿态影响 |
| 头部舒适极限（上） | ~30° | ~25° | 坐姿时躯干固定，颈椎后伸空间受限 |
| 头部舒适极限（下） | ~40° | ~45° | 坐姿有靠背支撑，颈部屈曲负荷分散 |

> **关键结论**：坐姿使向上舒适范围缩减约 5°，向下舒适范围扩展约 5°。这一差异在长时间使用场景下对 UI 布局有实质影响，因此垂直负荷区域需按姿态分别定义。

**从三要素到区域边界的推导**

与水平方向相同，垂直负荷区域的边界由**视觉能力分区**（第 2 节）、**头部垂直转动舒适度**（第 3.2 节）和**垂直眼-头协同模式**（上表）三个维度共同决定。但垂直方向需额外处理**上下不对称**和**姿态差异**，因此分为向上和向下两组推导。

**向上（仰视）区域边界推导**

| 区域边界 | 视觉能力（第 2 节） | 仰头舒适度（第 3.2 节） | 眼-头协同（上表） | → 综合体验转折 |
|---|---|---|---|---|
| **+5°（V0→V+1）** | 超出中央凹区（±5°）——精细阅读能力开始下降 | 纯眼球范围内（<8°），无头部运动 | 纯眼球运动 | 视觉从最高锐度区退出；虽仍在纯眼球范围，但信息获取精度已质变——与水平方向的 ±10° 对称逻辑一致（视觉能力主导此边界） |
| **+15° 站 / +10° 坐（V+1→V+2）** | 近周边视觉——无法辨识文字 | 站：跨越轻松→舒适阈值（~10°）；坐：已进入舒适上段（~8°起） | 眼球为主→头部开始介入（~8°~15°） | 头部从"无感"进入"有意识仰头"，视觉已退出可识别范围——坐姿更早到达此转折，因颈椎后伸受限 |
| **+25° 站 / +20° 坐（V+2→V+3）** | 近→中周边过渡 | 站：跨越舒适→适度阈值（~20°）；坐：跨越舒适→适度阈值（~15°） | 头部为主（>15°），接近极限 | 颈后伸肌群持续紧张，不适感明确积累——坐姿下 +20° 即到达站姿 +25° 的等效负荷 |

**向下（俯视）区域边界推导**

| 区域边界 | 视觉能力（第 2 节） | 低头舒适度（第 3.2 节） | 眼-头协同（上表） | → 综合体验转折 |
|---|---|---|---|---|
| **-5°（V0→V-1）** | 超出中央凹区——与向上对称 | 纯眼球范围内（<12°），无头部运动 | 纯眼球运动 | 与 +5° 对称；视觉精度下降，但下视方向因习惯优势，实际舒适感更好 |
| **-20° 站 / -25° 坐（V-1→V-2）** | 近周边视觉中段 | 站：跨越轻松→舒适阈值（~15°）；坐：跨越轻松→舒适阈值（~20°，靠背支撑延后） | 眼球为主→头部微调（~12°~22°） | 头部从"无感低倾"进入"有意识低头"——坐姿因靠背支撑延后 5° 到达此转折 |
| **-35° 站 / -40° 坐（V-2→V-3）** | 中周边——仅能检测运动 | 站：跨越舒适→适度阈值（~25°）；坐：跨越舒适→适度阈值（~30°） | 头部为主（>22°） | 颈椎承压增大 + XR 头显重心前移效应叠加——坐姿靠背虽延缓负荷，但头显物理效应抵消部分优势 |

> **与水平方向的对比**：水平方向的 V0 对称边界为 ±5°（由视觉能力主导），而 H0 为 ±10°（由周中央凹范围主导）。差异的原因是：水平方向上人眼在 ±10° 内仍保持有效识别能力（周中央凹覆盖），但垂直方向在超出 ±5° 后，精细阅读能力下降的主观感受更直接（向上/向下偏离自然视线会更快感到"不在最佳位置"）。此外，垂直方向每条边界都受姿态影响而产生站/坐两个版本——这是水平方向不具备的复杂性。

#### 4.2.2 垂直负荷区域定义

以自然静息视线为 0° 基准，向上为正，向下为负。由于垂直方向的姿态依赖性，本节分别给出**站姿**和**坐姿**的区域参数，并提供**通用设计推荐值**（取两者的保守交集）。

---

##### V0 — 自然视线区 (Natural Gaze Band)

| 属性 | 站姿 | 坐姿 |
|---|---|---|
| **角度范围** | -5° ~ +5° | -5° ~ +5° |
| **身体感受** | 完全无感，视线的默认归宿。如同正前方平视时最放松的那条线。 | 同站姿 |

> 该区域为纯眼球运动范围，不受姿态影响。

---

##### V+1 — 舒适上视区 (Comfortable Upper Band)

| 属性 | 站姿 | 坐姿 |
|---|---|---|
| **角度范围** | +5° ~ +15° | +5° ~ +10° |
| **身体感受** | 轻微的向上瞥视，类似于从桌面抬眼看对面人的脸。自然且不费力。 | 类似站姿体验，但到达上界（+10°）时已感到轻微颈后紧张。坐姿时躯干固定使颈椎后伸受限，仰头的"余量"更小。 |

---

##### V-1 — 舒适下视区 (Comfortable Lower Band)

| 属性 | 站姿 | 坐姿 |
|---|---|---|
| **角度范围** | -5° ~ -20° | -5° ~ -25° |
| **身体感受** | 轻微下瞥，类似于阅读时视线自然下沉。符合人类数千年阅读和手工操作的习惯，反而比平视更放松。 | 比站姿更舒适——靠背分担了部分颈部负荷，低头姿态可持续更久。这是坐姿下文本和手部交互内容的最佳区域。 |

---

##### V+2 — 适度上视区 (Moderate Upper Band)

| 属性 | 站姿 | 坐姿 |
|---|---|---|
| **角度范围** | +15° ~ +25° | +10° ~ +20° |
| **身体感受** | 需要轻微仰头，颈后肌群开始工作。类似于站在展厅中抬头看墙上较高处的画作。短暂可接受（数秒至十余秒），但不适合持续阅读。 | 仰头负荷比站姿更明显，更快产生颈后疲劳。上界 +20° 在坐姿下的感受约等于站姿的 +25°。 |

---

##### V-2 — 适度下视区 (Moderate Lower Band)

| 属性 | 站姿 | 坐姿 |
|---|---|---|
| **角度范围** | -20° ~ -35° | -25° ~ -40° |
| **身体感受** | 需要中度低头，类似于站着看手机。颈前侧和肩部开始有轻微负荷，但比同等幅度的仰头更容易忍受。 | 靠背支撑使负荷进一步降低，下界可延伸至 -40°。但 XR 头显重心前移的效应在此角度开始显现。 |

---

##### V+3 — 极限上视区 (Extreme Upper Band)

| 属性 | 站姿 | 坐姿 |
|---|---|---|
| **角度范围** | >+25° | >+20° |
| **身体感受** | 明显仰头，颈部后伸接近极限。类似于仰头看天花板或高楼。长时间保持会导致颈部疲劳甚至头晕。**应避免放置需要持续注视的内容。** | 坐姿下更早进入极限区——+20° 即开始出现显著不适。**坐姿用户的上方内容必须更为克制。** |

---

##### V-3 — 极限下视区 (Extreme Lower Band)

| 属性 | 站姿 | 坐姿 |
|---|---|---|
| **角度范围** | <-35° | <-40° |
| **身体感受** | 过度低头，类似于长时间低头看地面。颈椎承受较大压力。在 XR 中还可能因为头显重心前移而加剧不适。**应避免放置需要持续注视的内容。** | 虽然下界比站姿更低，但进入此区域后头显重心前移效应突出，实际体验仍然不适。**不应因坐姿下界更低而在此区域增加内容。** |

---

##### 垂直区域姿态汇总

| 区域 | 站姿范围 | 坐姿范围 | 通用设计推荐（保守交集） |
|---|---|---|---|
| V0 自然视线区 | -5° ~ +5° | -5° ~ +5° | **-5° ~ +5°** |
| V+1 舒适上视区 | +5° ~ +15° | +5° ~ +10° | **+5° ~ +10°** |
| V-1 舒适下视区 | -5° ~ -20° | -5° ~ -25° | **-5° ~ -20°** |
| V+2 适度上视区 | +15° ~ +25° | +10° ~ +20° | **+10° ~ +20°** |
| V-2 适度下视区 | -20° ~ -35° | -25° ~ -40° | **-20° ~ -35°** |
| V+3 极限上视区 | >+25° | >+20° | **>+20°** |
| V-3 极限下视区 | <-35° | <-40° | **<-35°** |

> **设计建议**：
> - **姿态未知/混合场景**：使用"通用设计推荐"列的保守值，确保坐/站姿下均保持舒适。
> - **姿态感知应用**：若应用可检测用户姿态（如通过头显 IMU 或座椅传感器），可动态切换为对应姿态的区域参数，充分利用坐姿下的下方空间或站姿下的上方空间。
> - **注意**：通用推荐中 V+1 仅有 5° 宽度（+5° ~ +10°），这意味着在姿态未知时，上方的"舒适区"非常有限——再次印证了**向下延伸优于向上**的设计原则。

### 4.3 二维复合区域地图

将水平和垂直区域叠加，形成一个二维区域地图。每个位置的综合舒适等级取水平和垂直的**较低者**（木桶效应）。以下图示使用**通用设计推荐值**（站姿与坐姿的保守交集）。

```
                    V+3 极限上视 (>+20°)
                    ─────────────────────
                    V+2 适度上视 (+10°~+20°)
                    ─────────────────────
                    V+1 舒适上视 (+5°~+10°)
                    ═════════════════════
                    V0  自然视线 (±5°)      ← 中心
                    ═════════════════════
                    V-1 舒适下视 (-5°~-20°)
                    ─────────────────────
                    V-2 适度下视 (-20°~-35°)
                    ─────────────────────
                    V-3 极限下视 (<-35°)

H5←──H4──←──H3──←──H2──←──H1──←H0→──H1──→──H2──→──H3──→──H4──→H5
身体    极限     刻意     自然     核心  焦点  核心     自然     刻意     极限   身体
转向    边缘     转头     转视     舒适        舒适     转视     转头     边缘   转向
```

**黄金区域（Golden Zone）**：H0-H1 × V0-V-1 的交汇区域，即水平 ±25° 且垂直 +5° 至 -20° 的区域。这是整个空间中最舒适、最高效的内容放置区域，对应了"毫不费力"的注视体验。

**次优区域（Silver Zone）**：H2 × V+1/V-2 的延伸区域，即水平 ±45° 且垂直 +10° 至 -35°（通用推荐值）。适合放置需要偶尔查看但不需持续注视的内容。

> **姿态感知时的调整**：若应用可检测姿态，站姿下次优区域上界可放宽至 +15°；坐姿下次优区域下界可延伸至 -40°。

## 5. 内容类型与区域映射

### 5.1 映射总表

| 内容类型 | 推荐水平区域 | 推荐垂直区域 | 映射理由 |
|---|---|---|---|
| **主交互面板** | H0 ~ H1 (0°~±25°) | V0 ~ V-1 (±5° ~ -20°) | 频繁操作需最低视觉/运动代价，下方偏移符合手部自然操作位置 |
| **信息展示** | H0 ~ H1 (0°~±25°) | V0 ~ V+1 (+5° ~ +15°) | 阅读需要中央凹视觉，信息内容倾向于放在视线水平或略上方 |
| **通知/提示** | H1 ~ H2 边缘 (±15°~±35°) | V+1 (+5° ~ +15°) | 需要可被感知但不遮挡主任务；上方偏移利用"跳出感"吸引注意 |
| **导航元素** | H2 ~ H3 (±25°~±70°) | V0 (±5°) | 导航本身分布在空间中，需要适度转头来建立空间认知 |
| **媒体内容** | H0 (0°~±10°) | V0 (±5°) | 持续观看需最大舒适度，居中放置是唯一合理选择 |
| **环境氛围** | H3 ~ H4 (±45°~±100°) | 任意 | 不需要直接注视，存在于周边视觉中即可营造空间感 |

### 5.2 内容类型详细策略

#### 主交互面板（菜单、工具栏、输入表单）

```
推荐区域：H0-H1 × V0-V-1
最大容许区域：H2 × V+1-V-2
```

- 放置在黄金区域内，确保最频繁的操作零颈部负荷
- 垂直偏向下方（V-1），原因：(1) 手部自然下垂/前伸的操作位置偏低 (2) 符合桌面交互心智模型
- 面板宽度应控制在 H0-H1 范围内（±25°），避免要求用户在面板内大幅扫视
- 若面板较大，主操作区居中放在 H0，辅助功能向两侧 H1 展开

#### 信息展示（文本、图片、数据看板）

```
推荐区域：H0-H1 × V0-V+1
最大容许区域：H2 × V+1-V-1
```

- 需要持续阅读的文字内容必须在 H0 范围内（±10°），确保中央凹视觉覆盖
- 数据看板可以向 H1-H2 延展，但核心数据指标应在 H0-H1 内
- 垂直位置略偏上（V0-V+1），符合"信息在上、操作在下"的空间层次直觉

#### 通知/提示（系统消息、警告、状态更新）

```
推荐区域：H1边缘-H2 × V+1
备用区域：H1 × V+1-V+2（紧急通知可更靠近中心）
```

- 放在核心舒适区的边缘，可被周边视觉感知但不直接遮挡主任务
- 垂直偏上（V+1），利用上方空间的"跳出感"——人类对上方出现的新物体更为警觉
- 紧急通知可临时进入 H0-H1 区域，但应提供快速消除机制
- 避免放在 V-1 以下，因为下方的新物体容易被忽视

#### 导航元素（空间锚点、路径指引）

```
推荐区域：H2-H3 × V0
备用区域：根据空间结构可延伸至 H4
```

- 导航元素的特殊性在于它们服务于空间定位，本身就需要分布在较广的角度范围
- 保持在 V0 附近的水平带上，减少垂直方向的搜索成本
- 重要的方向性指引可以出现在 H3 甚至 H4，因为其目的是引导用户转身，转身本身就是预期行为
- 应配合动画/运动线索，利用周边视觉的运动敏感性来引导注意

#### 媒体内容（视频播放、3D 模型展示）

```
推荐区域：H0 × V0
最大容许区域：H1 × V+1-V-1
```

- 媒体内容通常需要持续注视数分钟甚至更长，必须放在最舒适的中心位置
- 视频播放面板的角宽度建议不超过 ±15°（参考 30° 的影院最佳观影角）
- 3D 模型展示可以略宽（H1 范围），因为用户会围绕模型自然扫视
- 若用户主动将媒体拖入 H2+ 区域，应提供"回到中心"的快捷手势

#### 环境/氛围元素（背景装饰、空间分隔）

```
推荐区域：H3-H4 × 任意垂直区域
备用区域：可延伸至 H5（用户身后）
```

- 不需要用户直接注视，在周边视觉中以色彩、光影、缓慢动画的方式存在
- 可以填充刻意转头区及更外侧的空间，赋予虚拟环境"空间完整感"
- 避免在此区域使用快速运动或高对比度变化，以免持续干扰周边视觉
- 可以作为空间分区的隐性线索（例如用不同色调区分不同功能区域）

## 6. 设计规则与策略

### 6.1 通用规则

| # | 规则 | 理由 |
|---|---|---|
| R1 | **高频交互内容限制在 ±25° 水平范围内** | 超出此范围的频繁交互会导致颈部疲劳 |
| R2 | **持续注视内容限制在 ±10° 水平 × +5°/-20° 垂直范围内** | 持续注视需要中央凹视觉和零头部运动 |
| R3 | **垂直布局遵循"信息上、操作下"原则** | 符合重力方向的认知直觉和手部操作的自然位置 |
| R4 | **利用垂直不对称性：向下延伸优于向上** | 向下看比仰头更舒适，且与日常习惯一致 |
| R5 | **通知出现在主任务上方边缘** | 上方出现的变化更易引起注意，且不遮挡操作区域 |
| R6 | **避免在 H4+ 区域放置需要精确交互的内容** | 大幅转头后的手部精确操作会显著退化 |
| R7 | **兼顾两种姿态时，取角度范围的交集** | 确保同一 UI 在坐/站姿下都保持舒适 |
| R8 | **周边区域仅使用低频、大面积、运动型视觉线索** | 周边视觉无法处理文字和精细图形 |

### 6.2 区域间过渡策略

内容不应突然从一个区域跳到另一个区域。以下策略确保区域间的平滑过渡：

- **渐进式信息密度**：从 H0 到 H4，信息密度和交互复杂度应逐渐降低
- **视觉引导**：当内容需要引导用户从中心区看向外围区域时，应使用渐进式视觉线索（颜色渐变、动画轨迹）
- **回归吸引**：外围区域的内容在完成交互后，应提供明确的视觉线索引导用户回到中心区域
- **区域过渡缓冲**：在区域边界处避免放置关键内容，因为边界处的体验评级有渐变性

### 6.3 Head-locked UI 附加建议

虽然 Head-locked UI 始终在视野内，但视野角分区仍然影响其在视野中的放置位置：

| 策略 | 说明 |
|---|---|
| 避开 H0 × V0 | Head-locked 内容不应遮挡中心凝视区，否则干扰主任务 |
| 优选视野边缘 | 持续性 Head-locked 信息（如状态栏）应放在 V+1 或 V-1 的边缘 |
| 透明度与虚化 | 在周边位置使用高透明度，用户主动注视时再逐渐变清晰 |
| 极简信息量 | Head-locked UI 永远在视野中，因此内容必须极度精简以避免视觉疲劳 |

## 7. 角度参数速查表

### 7.1 水平区域参数

| 区域 | 范围 | 舒适等级 | 适合内容 | 最大停留时长 |
|---|---|---|---|---|
| H0 静默焦点区 | 0°~±10° | ★★★★★ | 主任务、媒体、精细阅读 | 无限 |
| H1 核心舒适区 | ±10°~±25° | ★★★★☆ | 主交互、信息展示、辅助工具 | 数分钟 |
| H2 自然转视区 | ±25°~±45° | ★★★☆☆ | 通知、次要信息、导航 | 数十秒 |
| H3 刻意转头区 | ±45°~±70° | ★★☆☆☆ | 导航锚点、环境元素、低频操作 | 数秒 |
| H4 极限边缘区 | ±70°~±100° | ★☆☆☆☆ | 环境氛围、空间方位线索 | 瞬间 |
| H5 身体转向区 | >±100° | ☆☆☆☆☆ | 避免放置内容 | 避免 |

### 7.2 垂直区域参数

| 区域 | 范围（自然视线基准） | 舒适等级 | 适合内容 |
|---|---|---|---|
| V0 自然视线区 | -5°~+5° | ★★★★★ | 核心任务内容 |
| V+1 舒适上视区 | +5°~+15° | ★★★★☆ | 信息展示、通知 |
| V-1 舒适下视区 | -5°~-20° | ★★★★☆ | 交互面板、手部操作 |
| V+2 适度上视区 | +15°~+25° | ★★★☆☆ | 环境信息、沉浸元素 |
| V-2 适度下视区 | -20°~-35° | ★★★☆☆ | 辅助信息、低频操作 |
| V+3 极限上视区 | >+25° | ★☆☆☆☆ | 避免（仅环境氛围） |
| V-3 极限下视区 | <-35° | ★★☆☆☆ | 避免（仅环境氛围） |

### 7.3 姿态偏移修正值

在真实水平线坐标系中，需加上以下偏移量：

| 姿态 | 自然视线偏移 | 应用方式 |
|---|---|---|
| 站姿 | 真实水平线 -10° | 所有垂直区域角度 + (-10°) 得到相对水平线的绝对角度 |
| 坐姿 | 真实水平线 -15° | 所有垂直区域角度 + (-15°) 得到相对水平线的绝对角度 |

**示例**：V-1 区域（-5° ~ -20°）在站姿下的绝对角度为 -15° ~ -30°（相对真实水平线），在坐姿下为 -20° ~ -35°。
