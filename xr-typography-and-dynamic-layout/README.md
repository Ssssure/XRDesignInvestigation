# XR 环境中的字体排版与动态布局适应设计规范

## 基于认知-视觉双重负担最小化的排版与布局策略

---

## 1. 概述

本规范从资深 UI/UX 设计师视角出发，系统梳理 XR 环境中文字排版和布局系统的特殊约束，建立一套**以降低认知负担和视觉疲劳为核心目标**的字体排版参数体系与动态布局适应策略。

XR 环境中的排版面临三个传统屏幕设计不存在的根本挑战：

1. **文字存在于三维空间中**——字号不再是像素数，而是视角尺寸（angular size），随深度距离动态变化
2. **用户头部和身体持续运动**——排版必须在运动中保持可读性，文字渲染受头部跟踪精度和帧率影响
3. **视觉环境高度动态**——亮度、对比度、焦距持续变化，排版参数需要实时适应

本规范不重复其他规范已详述的基础生理数据，而是**直接引用并在排版维度上延伸应用**。

### 1.1 适用范围

| 属性 | 说明 |
|---|---|
| 平台 | XR 头显（VR、AR、MR），通用，不绑定特定硬件 |
| 覆盖维度 | 字体选型、字号系统、排版微观参数（行高/字距/行长）、对比度与色彩排版、信息层级、动态布局适应 |
| 核心目标 | 降低认知负担（信息加工成本）与视觉疲劳（眼肌/调节系统负荷）的双重负担 |
| 内容类型 | 系统 UI 文字、应用内文本、空间标注、通知信息、辅助说明、长文阅读 |

### 1.2 与其他规范的关系

| 配套规范 | 本规范引用的内容 | 引用方式 |
|---|---|---|
| **XR 空间 UI 布局设计规范**（空间布局） | 视野角分区（§2）、头部转动舒适度（§3）、眼-头协同（§4）、三问判定法（§5.1） | 排版参数随空间位置分级调整 |
| **XR 亮度变化与人眼适应策略**（亮度适应） | 三种视觉模式（§2.1）、暗场景色彩策略 D5（§5.6）、对比度保护 D4（§5.5） | 排版参数随亮度环境动态适应 |
| **VAC 深度分区模型**（深度分区） | 深度分区边界、视觉属性策略 | 排版参数随内容深度分级调整 |

### 1.3 核心设计哲学

**排版的终极目标是"不被注意"。** 用户在阅读时应当完全沉浸于信息内容本身，而非与文字的物理形态搏斗。每一个排版参数的选择——字号、行高、对比度、字重——都应指向同一个目标：**让文字的视觉呈现透明化，使认知资源 100% 投入到语义理解中。**

在 XR 中，实现这一目标的难度远大于传统屏幕，因为影响可读性的变量从 2 个（字号 × 对比度）变成了至少 6 个（字号 × 对比度 × 深度 × 角度位置 × 环境亮度 × 用户运动状态）。本规范的核心任务就是为这 6 个变量建立联合适应模型。

## 2. 人因基础：XR 环境中的阅读机制

### 2.1 眼球运动与阅读

阅读不是连续扫描——眼球通过一系列**注视（Fixation）** 和**跳视（Saccade）** 完成信息提取。

| 参数 | 传统屏幕 | XR 环境 | 差异原因 |
|---|---|---|---|
| 单次注视时长 | 200–250 ms | 250–350 ms | XR 中调节-辐辏冲突增加对焦负荷；头显光学系统引入的像差降低文字清晰度 |
| 注视跨度（一次注视能处理的字符数） | 英文 7–9 字符 / 中文 3–4 字 | 英文 5–7 字符 / 中文 2–3 字 | XR 中有效视锐度低于裸眼（受 PPD 限制），注视跨度缩窄 |
| 回视率（需要重新阅读的比例） | 10%–15% | 15%–25% | 文字渲染模糊导致首次编码失败概率上升 |
| 阅读速度 | 250–300 词/分钟 | 150–200 词/分钟 | 上述三因素叠加，XR 阅读速度约降至传统屏幕的 60%–70% |

> **设计含义**：XR 中阅读本身就更慢、更累。排版设计的首要任务是**尽一切可能弥补这一先天劣势**——增大字号、增加行高、减少单行字数、提高对比度——而非照搬传统屏幕的排版参数。

### 2.2 像素每度（PPD）：XR 排版的基础物理约束

**像素每度（Pixels Per Degree, PPD）** 是 XR 排版中最核心的物理参数，它决定了在给定硬件上，文字能被渲染得多清晰。

```
PPD = 水平像素数 / 水平视场角（度）
```

| 设备代际 | 典型 PPD | 等效效果 | 对排版的约束 |
|---|---|---|---|
| 第一代 VR（2016–2018） | 10–14 PPD | 类似于在 30cm 处看 720p 手机 | 小于 1.2° 视角的文字无法辨认 |
| 第二代 VR（2019–2022） | 18–22 PPD | 类似于在 60cm 处看 1080p 显示器 | 小于 0.7° 视角的文字模糊 |
| 第三代 VR/MR（2023–2026） | 25–35 PPD | 类似于在 50cm 处看 2K 显示器 | 0.5° 视角的文字基本可读 |
| 视网膜级别（理论目标） | 60+ PPD | 匹配人眼中央凹分辨力极限 | 无排版限制 |

> **关键阈值**：人眼在中央凹区的分辨力极限约为 60 PPD（1 弧分 = 1/60 度）。当前大多数 XR 设备的 PPD 仅为这一极限的 30%–50%，这意味着**即使是最佳渲染条件下，XR 中的文字清晰度也远低于视网膜屏手机或高分辨率显示器**。排版设计必须主动补偿这一硬件缺陷。

### 2.3 视角尺寸（Angular Size）：取代像素的度量单位

在 XR 中，文字大小不能用像素（px）或点（pt）衡量，因为同样的"像素数"在不同深度、不同头显上呈现的实际大小完全不同。正确的度量单位是**视角尺寸（Angular Size）**，即文字在用户视野中所占的角度。

```
视角尺寸（度）= 2 × arctan(物理尺寸 / (2 × 观看距离))
简化近似：视角尺寸（度）≈ 57.3 × 物理尺寸 / 观看距离
```

| 用语 | 定义 | 换算参考（在 1m 距离） |
|---|---|---|
| **dmm（distance-independent millimeter）** | 在 1 米距离处 1mm 的视角大小（≈ 0.057°） | 1 dmm = 0.057° |
| **视角度（degree）** | 直接以角度为单位 | 1° ≈ 17.5 dmm |
| **弧分（arcmin, ′）** | 角度的 1/60 | 1′ = 0.0167° ≈ 0.29 dmm |

> **为什么必须用视角尺寸？** 一个 1cm 高的字母放在 1 米处，视角约为 0.57°；放在 3 米处，视角仅约 0.19°——前者清晰可读，后者几乎不可见。排版规范如果以像素为基准，在深度变化时立即失效。

### 2.4 认知负荷理论在 XR 排版中的应用

认知负荷理论（Cognitive Load Theory, Sweller 1988）将认知资源消耗分为三类：

| 负荷类型 | 定义 | 在 XR 排版中的具体表现 |
|---|---|---|
| **内在负荷（Intrinsic）** | 信息本身的复杂度 | 文本内容的语义难度——排版无法改变，但可通过信息分块降低单次处理量 |
| **外在负荷（Extraneous）** | 信息呈现方式引起的额外加工 | **排版设计的核心战场**——字号过小需要额外眯眼聚焦、对比度不足需要额外辨认、行高过密需要额外区分行、信息布局混乱需要额外搜索定位 |
| **生成负荷（Germane）** | 将信息整合为心智模型的有益加工 | 良好的排版层级帮助用户建立信息结构，促进理解——这是排版应当最大化的目标 |

> **排版设计的认知目标**：**最小化外在负荷、最大化生成负荷**。每一个排版参数的选择都应接受这一检验——"这个参数值是否增加了用户辨认文字的额外努力？"

### 2.5 视觉疲劳的多源模型

XR 环境中的视觉疲劳不是单一来源，而是多个系统同时承受负荷的累积结果：

| 疲劳来源 | 生理机制 | 排版相关的加剧因素 | 排版相关的缓解策略 |
|---|---|---|---|
| **调节疲劳（Accommodative）** | 睫状肌持续收缩维持对焦 | 小字号迫使用户额外用力聚焦；深度频繁变化的文字布局 | 增大字号降低对焦精度要求；避免文字跨深度层分布 |
| **辐辏疲劳（Vergence）** | 内直肌持续收缩维持双眼汇聚 | 文字放置在 VAC 冲突显著的深度 | 将需要阅读的文字放在 VAC 舒适区（参见 VAC 深度分区模型） |
| **眼外肌疲劳（Extraocular）** | 眼球运动肌反复扫视 | 行长过长增加水平扫视幅度；行高过密增加垂直定位难度 | 控制行长在舒适扫视范围内；增加行高降低行间混淆 |
| **瞳孔疲劳（Pupillary）** | 虹膜肌反复调节瞳孔大小 | 前景/背景亮度反差过大触发频繁瞳孔调节 | 控制文字与背景的亮度比在舒适范围内 |
| **认知疲劳（Cognitive）** | 注意力资源持续消耗 | 信息密度过高、排版层级不清晰 | 控制信息密度、建立清晰的视觉层级 |

> **关键洞察**：这五种疲劳源是**相乘而非相加**的关系。单独一个来源的轻度负荷可以被耐受，但多个来源同时处于中度负荷时，总体疲劳会急剧上升。排版优化的价值在于——即使不能消除所有负荷源，降低其中 2–3 项也能显著改善整体体验。

## 3. 字体选择

### 3.1 XR 环境字体选型原则

XR 环境的低 PPD 和光学像差对字体的笔画结构提出了特殊要求，其严格程度远超传统屏幕。

| 原则 | 要求 | 原理 | 正面示例 | 反面示例 |
|---|---|---|---|---|
| **P1：优先无衬线（Sans-serif）** | 正文和系统 UI 强制无衬线 | 衬线在低 PPD 下退化为模糊噪点，消耗像素预算却不增加辨识度，反而干扰字形识别 | Noto Sans、Inter、SF Pro | Times New Roman、Georgia |
| **P2：大 x-height 比例** | x-height/cap-height ≥ 0.72 | 大 x-height 意味着小写字母主体更高，在相同字号下有效可读面积更大；XR 中像素预算紧张，每一个像素都应用于有效信息 | Inter（0.76）、Roboto（0.74） | Futura（0.68）、Didot（0.64） |
| **P3：开放字怀（Open Counter）** | 字母 a/e/s/g 的内部空间应充分打开 | 低 PPD 下封闭的字怀容易糊成一团，导致字母混淆（a/o/e 难以区分） | Noto Sans（开放字怀设计） | Helvetica（较封闭的字怀） |
| **P4：清晰的字形区分** | 1/l/I、0/O、rn/m 等易混淆字符必须有明确视觉差异 | 低分辨率放大了字形歧义，尤其在代码、数字密集的界面中 | JetBrains Mono（专为辨识设计）、IBM Plex Sans | Arial（1/l/I 难以区分） |
| **P5：均匀笔画粗细** | 粗细对比比率 ≤ 1.5:1 | 细笔画在低 PPD 下可能低于 1 像素渲染阈值而消失，造成笔画断裂 | Inter、Source Sans Pro | Bodoni、Playfair Display |
| **P6：充分的字母间自然间距** | 字体默认 tracking 不应过密 | XR 中相邻字母在低分辨率下容易粘连，需要更多呼吸空间 | 多数 Google Fonts（默认间距合理） | 紧缩排版的标题字体用于正文 |

### 3.2 中文字体特殊要求

中文字符的结构复杂度远高于拉丁字母（平均笔画数 8–12 划 vs. 拉丁 2–3 划），在 XR 低 PPD 环境下面临更严峻的可读性挑战。

| 原则 | 要求 | 原理 |
|---|---|---|
| **C1：优先黑体（Gothic/Sans-serif）** | 系统 UI 和正文强制使用黑体系 | 宋体/明体的横细竖粗结构在低 PPD 下横画消失，仅剩竖画骨架 |
| **C2：笔画匀称，无极细横画** | 避免仿宋、楷体 | 极细横画（< 1 px 渲染宽度）在 XR 中完全不可见 |
| **C3：最小字号高于拉丁文 20%–30%** | 中文正文最小视角 ≥ 拉丁正文的 1.2–1.3 倍 | 中文字符的信息密度更高（笔画更多），需要更大面积才能分辨 |
| **C4：字面率 ≥ 90%** | 字符应充分利用字身框面积 | 字面率低的字体在相同字号下视觉尺寸更小，浪费像素预算 |

**推荐中文字体**：Noto Sans CJK（思源黑体）、PingFang SC（苹方）、HarmonyOS Sans

### 3.3 字重（Font Weight）策略

字重选择在 XR 中比传统屏幕更为关键——笔画粗细直接决定了在低 PPD 下文字是否可见。

| 使用场景 | 推荐字重 | 原理 | 禁止字重 |
|---|---|---|---|
| **系统 UI 正文** | Medium (500) ~ Semibold (600) | Regular (400) 的细笔画在低 PPD 下接近消失阈值；Medium 提供足够的笔画面积 | Light (300) 及以下 |
| **长文阅读** | Regular (400) ~ Medium (500) | 长时间阅读需要降低视觉"重量感"以减少认知疲劳，但不能低于可读阈值 | Thin (100)、Light (300) |
| **标题/层级强调** | Bold (700) ~ ExtraBold (800) | 标题需要在排版层级中形成明确的视觉锚点 | — |
| **暗场景文字** | Semibold (600) ~ Bold (700) | 暗视觉下视锐度降低，需更粗笔画保持可辨性（参见亮度适应规范 §5.6 D5 R4） | Regular (400) 及以下 |
| **辅助说明/次要信息** | Regular (400) ~ Medium (500) | 与主体内容形成层级对比，但不得低于可读阈值 | Light (300) 及以下 |

> **XR 字重的底线规则**：在当前主流 XR 设备（PPD 20–35）上，**Light (300) 及以下字重在任何场景下都不应出现**。即使是最次要的辅助信息，也至少使用 Regular (400)。这一规则在设备 PPD 达到 60+ 之前不应放松。

## 4. 字号系统：多维度动态适应

### 4.1 基础字号模型（基于视角尺寸）

XR 中的字号以**视角度**为基准单位，确保文字在任何深度和设备上呈现一致的视觉大小。

#### 4.1.1 最小可读视角与舒适阅读视角

| 阈值类型 | 拉丁文（cap-height 视角） | 中文（字身高度视角） | 来源与依据 |
|---|---|---|---|
| **绝对下限（可辨认但吃力）** | 0.35° | 0.45° | 在 25 PPD 设备上约 8–9 px 高度，接近单个笔画 1 px 渲染极限 |
| **最小可读（可完成阅读但较慢）** | 0.5° | 0.6° | ISO 9241-303 推荐的最小字符高度；多项 XR 阅读研究的共识下限 |
| **舒适阅读（正常阅读速度）** | 0.7°–1.0° | 0.8°–1.2° | 阅读速度和错误率在此范围趋于平稳；Apple visionOS HIG 推荐正文约 0.9° |
| **最佳阅读（阅读效率最大化）** | 1.0°–1.4° | 1.2°–1.6° | 实验证实此范围阅读速度最高、主观舒适度最佳 |

> **设计启示**：传统屏幕正文字号（16px）在手机上约为 0.5°–0.7° 视角。XR 中要达到同等阅读体验，视角应至少 0.7°，理想为 1.0° 左右——这意味着**XR 正文字号在物理尺寸上应比手机大 40%–100%**。

#### 4.1.2 排版层级字号系统（Type Scale）

以正文字号为基准（1.0×），建立倍率递进的排版层级：

| 层级 | 倍率 | 视角范围（拉丁/中文） | 用途 | 字重建议 |
|---|---|---|---|---|
| **Caption / 说明** | 0.75× | 0.5°–0.6° / 0.6°–0.7° | 时间戳、辅助标注、版权信息 | Regular–Medium |
| **Body / 正文** | 1.0× | 0.7°–1.0° / 0.8°–1.2° | 主体阅读内容、表单标签、按钮文字 | Regular–Medium |
| **Subheading / 小标题** | 1.25× | 0.88°–1.25° / 1.0°–1.5° | 卡片标题、分组标签、列表项标题 | Medium–Semibold |
| **Heading / 标题** | 1.6× | 1.12°–1.6° / 1.3°–1.9° | 页面二级标题、面板标题 | Semibold–Bold |
| **Title / 大标题** | 2.0× | 1.4°–2.0° / 1.6°–2.4° | 页面主标题 | Bold–ExtraBold |
| **Display / 展示** | 2.5×–3.0× | 1.75°–3.0° / 2.0°–3.6° | 启动画面、空状态提示、沉浸式标题 | Bold–ExtraBold |

> **倍率递进比约 1.25–1.3×**，接近主三度音程比例（Major Third Scale），在视觉上形成自然、和谐的层级递进。

### 4.2 深度适应（与 VAC 深度分区模型联动）

文字放置在不同深度时，排版参数需要适应两个独立因素：

1. **物理视角缩放**——距离越远，相同物理尺寸的文字视角越小
2. **VAC 冲突对视锐度的影响**——偏离光学焦平面越远，对焦越模糊，有效视锐度越低

| 深度分区 | 与焦平面的屈光度差 | 对焦清晰度影响 | 字号补偿策略 | 可承载的最高排版层级 |
|---|---|---|---|---|
| **舒适区** | ΔD ≤ 0.3 | 无明显模糊 | 无需补偿，使用基准字号 | 全层级可用（Caption ~ Display） |
| **可用区** | 0.3 < ΔD ≤ 0.6 | 轻微散焦，精细笔画开始柔化 | 字号 +10%~+15%，字重 +1 级 | Body 及以上（避免 Caption） |
| **容忍区** | 0.6 < ΔD ≤ 1.0 | 明显散焦，阅读速度显著下降 | 字号 +20%~+30%，字重 +1~2 级 | Subheading 及以上（避免 Body 长文） |
| **不适区** | ΔD > 1.0 | 严重模糊，持续注视引发不适 | 仅使用 Display 级大字 + 高对比度 | 仅 Title/Display（极短文本） |

> **核心规则**：**需要逐字阅读的正文文字必须放在 VAC 舒适区内**。VAC 可用区仅适合短标签和按钮文字。容忍区及以外不应放置需要阅读的文字，仅允许一瞥即可辨识的大字标题或图标。

### 4.3 角度位置适应（与空间布局规范联动）

文字在空间中的位置（相对于用户注视中心的偏移角度）决定了它落在哪个视野功能分区内，而每个分区的视锐度直接约束了可用的排版参数。

| 视野分区 | 偏移角度 | 视锐度 | 字号适应策略 | 可承载排版内容 |
|---|---|---|---|---|
| **中央凹区** | 0°–5° | 100% | 基准字号，全层级可用 | 正文、精细控件标签、所有可读文字 |
| **周中央凹** | 5°–10° | ≈30% | 字号 ≥ 2.0× 基准（即 Heading 级以上），字重 Semibold+ | 大标题、高辨识度图标标签 |
| **近周边** | 10°–30° | 仅轮廓感知 | 不放置需要阅读的文字 | 仅色块状态标记、高对比度扫视锚点 |
| **中周边** | 30°–60° | 仅运动检测 | 不放置任何文字 | 仅动态视觉提示（动画、闪烁） |
| **远周边** | 60°+ | 仅明暗感知 | 不放置任何文字 | 仅环境光变化、紧急警报明暗 |

> **"文字必须在中央凹区"原则**：由于视锐度在 5° 以外急剧衰减至 30% 以下，**所有需要逐字阅读的文本都必须设计为用户注视后落入中央凹区（0°–5°）才呈现完整内容**。5°–10° 区域的文字仅用于提供"这里有什么"的预览信号（大标题/图标），引导用户注视后再阅读。

### 4.4 亮度适应（与亮度适应规范联动）

环境亮度的变化直接影响用户的有效视锐度和色彩感知能力，排版参数需要相应调整。

| 亮度级别 | 场景平均亮度 | 视觉模式 | 字号调整 | 字重调整 | 对比度调整 | 色彩调整 |
|---|---|---|---|---|---|---|
| **极亮（>150 nits）** | 雪地、白色 UI 为主 | 明视觉高段 | 基准 | 基准 | 基准（≥4.5:1） | 基准 |
| **亮（80–150 nits）** | 标准室内照明 | 明视觉正常段 | 基准 | 基准 | 基准 | 基准 |
| **中等（30–80 nits）** | 黄昏、昏暗灯光 | 明视觉中段 | +5%~+10% | 基准 | +10% 对比度 | 基准，监控红色可见性 |
| **暗（5–30 nits）** | 夜晚户外、暗色影院 | 明视觉低端 → 中间视觉 | +15%~+25% | +1 级（→ Semibold） | ≥5:1 | 红色→琥珀色；偏好蓝绿色系 |
| **极暗（<5 nits）** | 深空、全暗房间 | 中间视觉边界 | +25%~+35% | +1~2 级（→ Bold） | ≥7:1 | 仅用亮度对比 + 蓝绿色；放弃色彩编码 |

> **动态切换规则**：当系统检测到场景平均亮度跨越级别边界时，排版参数的调整应**在 0.5–1.5 秒内平滑过渡完成**，不可瞬间跳变（瞬间字号变化会导致文本重排，增加认知中断；瞬间色彩变化会引起视觉不适）。

### 4.5 字号适应的联合决策模型

当深度、角度、亮度三个维度同时影响排版时，取**最保守（最大字号）的约束**作为最终字号。

```
最终字号 = 基准字号 × max(深度补偿系数, 角度补偿系数, 亮度补偿系数)

其中：
  深度补偿系数 = 1.0（舒适区）/ 1.1–1.15（可用区）/ 1.2–1.3（容忍区）
  角度补偿系数 = 1.0（中央凹 0°–5°）/ 2.0+（周中央凹 5°–10°）/ ∞（>10° 不放文字）
  亮度补偿系数 = 1.0（亮/极亮）/ 1.05–1.1（中等）/ 1.15–1.25（暗）/ 1.25–1.35（极暗）
```

**示例：暗色洞穴场景中 VAC 可用区的信息面板标题**

| 维度 | 条件 | 补偿系数 |
|---|---|---|
| 深度 | VAC 可用区（ΔD=0.4） | 1.12 |
| 角度 | 用户注视后在中央凹区 | 1.0 |
| 亮度 | 暗（15 nits） | 1.2 |
| **最终** | max(1.12, 1.0, 1.2) = **1.2** | 正文基准 0.8° × 1.2 = **0.96°**，取 Heading 级 |

## 5. 排版微观参数

### 5.1 行高（Line Height）

行高是控制阅读节奏和减少"串行"（视线从当前行末跳至下一行时跳错行）的核心参数。

| 内容类型 | 推荐行高（相对字号倍数） | 传统屏幕参考值 | XR 增大幅度 | 原理 |
|---|---|---|---|---|
| **短文本（1–3 行）** | 1.3–1.5× | 1.2–1.4× | +10% | 短文本串行风险低，行高适度即可 |
| **正文段落（4+ 行）** | 1.5–1.8× | 1.4–1.6× | +10%–15% | XR 中回视率高（§2.1），增加行高降低串行概率 |
| **长文阅读（沉浸式阅读）** | 1.6–2.0× | 1.5–1.7× | +10%–20% | 长时间阅读时每一次串行都造成认知中断和时间浪费 |
| **代码/等宽文本** | 1.4–1.6× | 1.3–1.5× | +10% | 代码依赖精确的行对齐，行高过大反而影响代码结构感知 |
| **标题/Display** | 1.1–1.3× | 1.0–1.2× | +10% | 标题通常 1–2 行，紧凑行高强化整体性 |

> **行高的认知作用**：行高不仅仅是"行间空白"——它直接影响**回车扫视（return sweep）** 的准确率。回车扫视是阅读中最容易出错的眼球运动（从行末跳到下一行开头），行高不足时错误率会从 5% 上升至 20% 以上，每次错误都需要额外的认知资源来修正。

### 5.2 字间距（Letter Spacing / Tracking）

| 内容类型 | 推荐调整（相对字体默认值） | 原理 |
|---|---|---|
| **正文** | +2%–5%（略微加宽） | 低 PPD 下字母边缘模糊容易粘连，微增间距提高分离度 |
| **大标题（≥ 2.0× 基准）** | 0%–2%（接近默认或微增） | 大字号下字母自然间距已足够，过度加宽破坏词形整体性 |
| **全大写文字（UPPERCASE）** | +5%–10% | 全大写字母缺少升部降部的形状变化，需要更多间距避免字母粘连 |
| **中文** | +0%–3% | 中文字符为等宽方块，天然间距较好，微调即可 |
| **暗场景** | 额外 +3%–5% | 暗视觉下视锐度降低，需要更大间距补偿分辨力下降 |

> **禁止负 tracking**：在 XR 环境中，**任何场景下都不应使用负字间距（紧缩排版）**。传统平面设计中用于标题的紧缩排版在 XR 低分辨率下会导致字母融合不可辨认。

### 5.3 行长（Line Length / Measure）

行长是阅读效率和舒适度的关键变量。过长的行增加回车扫视距离和错误率；过短的行增加换行频率和认知碎片化。

| 内容类型 | 推荐行长（字符数） | 推荐行长（视角） | 传统屏幕参考 | 原理 |
|---|---|---|---|---|
| **拉丁文正文** | 40–60 字符 | 20°–28° | 50–75 字符 | XR 中注视跨度缩窄（§2.1），缩短行长减少每行回视次数 |
| **中文正文** | 20–30 字 | 18°–25° | 25–35 字 | 中文单字信息密度高，等效内容所需字符数更少 |
| **代码** | 60–80 字符 | 25°–30° | 80–120 字符 | 代码行长受语法结构约束，但 XR 中应避免超长行 |
| **通知/短文本** | 20–40 字符 | 12°–20° | — | 短文本追求一瞥可读，行长应在注视跨度 2–3 倍以内 |

> **行长与空间位置的关系**：行长不应超过零成本核心区的水平宽度（±15° = 30° 总宽度，参见空间布局规范 §7.9）。如果文字面板水平跨度超过 30°，用户阅读时需要头部微转，这在长文阅读场景中会累积为显著的颈部疲劳。

### 5.4 段落间距

| 参数 | 推荐值 | 原理 |
|---|---|---|
| **段落间距** | 行高的 0.75–1.0 倍（即段落间距 = 一行的 75%–100% 高度） | 段落分隔需要比行间距明显更大的空白，使视觉分组清晰；但不应大到破坏内容连贯性 |
| **标题与下方正文间距** | 行高的 0.5–0.75 倍 | 标题应与其归属的内容块紧密关联（格式塔接近律） |
| **标题与上方正文间距** | 行高的 1.0–1.5 倍 | 标题上方需要更大间距以标记新的内容段落开始 |

> **格式塔接近律在 XR 中的放大效应**：在 3D 空间中，人对"接近"和"分离"的感知阈值与 2D 不同。XR 中需要**更明确的间距差异**才能使分组关系清晰——传统屏幕上 8px vs 16px 的差异在 XR 中可能不够明显，需要放大到 1.5–2 倍。

### 5.5 对齐方式

| 对齐方式 | XR 中的适用性 | 原理 |
|---|---|---|
| **左对齐（LTR）/ 右对齐（RTL）** | ✅ **强制推荐** | 回车扫视的起点位置固定，降低定位成本；XR 中每一次定位失败的代价更大 |
| **居中对齐** | ⚠️ 仅限标题和短文本（≤ 3 行） | 居中对齐使每行起始位置不同，增加回车扫视定位难度；长文本居中对齐在 XR 中几乎不可读 |
| **两端对齐** | ❌ **禁止** | 两端对齐产生不均匀的词间距，在 XR 低 PPD 下这些不均匀间距会被放大为明显的"河流效应"（white rivers），严重干扰阅读 |

## 6. 对比度与色彩排版策略

### 6.1 文本对比度要求

文本可读性对对比度的依赖在 XR 中比传统屏幕更为强烈，因为低 PPD 已经削弱了一层可读性，对比度不足会造成双重打击。

| 内容类型 | 最低对比度（WCAG 亮度比） | XR 推荐对比度 | 原理 |
|---|---|---|---|
| **正文文字** | 4.5:1 | ≥ 6:1 | WCAG 4.5:1 是为视网膜屏设计的下限；XR 低 PPD 需要更高对比度补偿 |
| **大标题（≥ 2.0× 基准）** | 3:1 | ≥ 4.5:1 | 大字号在 XR 中仍受 PPD 限制，不应过度降低对比度 |
| **按钮/交互元素文字** | 4.5:1 | ≥ 6:1 | 交互元素需要快速辨认，不容许犹豫 |
| **辅助说明/次要文字** | 3:1 | ≥ 4.5:1 | 次要信息也需要在想看时能看清 |
| **暗场景文字** | 4.5:1 | ≥ 7:1 | 暗视觉下对比度感知能力下降，需要更高对比度补偿（参见亮度适应 §5.5 D4） |
| **AR 叠加文字** | 4.5:1 | ≥ 8:1（含背板对比度） | AR 文字与真实环境竞争，有效对比度受环境光严重削弱 |

### 6.2 文本色彩策略

| 规则 | 说明 | 原理 |
|---|---|---|
| **T1：正文文字使用中性色** | 正文文字应为白/浅灰（深底）或黑/深灰（浅底），避免使用彩色正文 | 彩色文字降低可读性——色彩处理占用额外认知资源，且部分色相在低 PPD 下与背景混淆 |
| **T2：彩色文字仅用于功能标记** | 链接、错误提示、关键词高亮等功能性标记可使用彩色，但必须同时有非色彩区分手段 | 不应仅依赖颜色传达语义（无障碍原则 + 中间视觉下色彩不可靠） |
| **T3：暗场景中的色彩文字偏移** | 暗场景中红色文字→琥珀色（#FFB74D），绿色文字→青绿色（#4DB6AC） | Purkinje 位移使红色在低亮度下失效（参见亮度适应 §2.4） |
| **T4：避免纯白正文大面积使用** | 暗背景上的正文使用 85%–90% 白度（如 #D9D9D9），而非 #FFFFFF | 纯白正文在暗背景上形成过度眩光，导致光晕效应模糊字形边缘 |
| **T5：AR 文字必须有背板** | AR 环境中所有文字都必须有深色半透明背板（≥ 60% 不透明度） | AR 叠加显示无法显示"黑色"，无背板的文字在浅色环境中完全不可读 |

### 6.3 背景与文本的关系

| 背景类型 | 文本可读性策略 | 原理 |
|---|---|---|
| **纯色深色背景** | 浅色文字（85%–90% 白度），对比度 ≥ 6:1 | 最基本的暗底亮字方案 |
| **纯色浅色背景** | 深色文字（15%–20% 白度），对比度 ≥ 6:1 | 明底暗字方案 |
| **半透明背景（blur/frost）** | 文字层加深色底衬（text shadow 或 backdrop scrim），确保最差情况下对比度 ≥ 4.5:1 | 半透明背景的透视内容可能与文字色相近，必须保证极端情况可读 |
| **动态/视频背景** | 文字区域加 80%+ 不透明度深色遮罩 | 动态背景会持续干扰文字识别，必须用高不透明度遮罩隔离 |
| **3D 场景叠加** | 文字加深色背板（≥ 70% 不透明度）+ 细微阴影/描边 | 3D 场景的亮度和色彩在空间中变化，仅靠对比度无法保证一致可读性 |

## 7. 信息层级与密度控制

### 7.1 信息密度原则

XR 中信息密度的上限远低于传统屏幕，这是多个因素叠加的结果：

| 限制因素 | 传统屏幕 | XR 环境 | 对信息密度的影响 |
|---|---|---|---|
| **阅读速度** | 250–300 词/分钟 | 150–200 词/分钟 | 单位时间可处理信息量减少 30%–40% |
| **有效阅读区域** | 整个屏幕 | 中央凹区（0°–5°） | 同时可见且可读的区域大幅缩小 |
| **视觉疲劳速率** | 低（静态屏幕无 VAC） | 高（VAC + 光学失真持续消耗） | 需要降低信息负荷以延长可用时长 |
| **字号下限** | ≈ 12px | ≈ 0.5° 视角（约等于传统 18–20px） | 相同面积能容纳的文字量减少 |

> **经验法则**：XR 中单个面板的信息密度应为同功能传统屏幕界面的 **50%–60%**。这不是简单地把所有文字放大，而是需要**重新设计信息架构**——减少同时呈现的信息量，通过分层、分页、渐进披露来管理总量。

### 7.2 信息密度的量化指标

| 指标 | 定义 | XR 推荐上限 | 原理 |
|---|---|---|---|
| **文字密度** | 每平方视角度内的字符数 | 拉丁文 ≤ 6 字符/度² / 中文 ≤ 3 字/度² | 超过此密度，字符间距被压缩到低 PPD 下的分辨极限 |
| **元素密度** | 中央凹区（0°–5°）内可见的独立 UI 元素数量 | ≤ 7 个 | Miller 法则（7±2）在 XR 中因额外认知负荷应取下限 |
| **层级深度** | 排版层级的最大嵌套层数 | ≤ 4 层 | 超过 4 层排版层级（如 Display > Title > Heading > Subheading > Body）在 XR 中难以仅靠视觉快速区分 |
| **色彩通道数** | 用于传达不同语义的色彩种类数 | ≤ 5 种功能色 | XR 中色彩感知受 PPD 和亮度影响不可靠，过多色彩语义增加学习和辨认负担 |

### 7.3 渐进式信息披露（Progressive Disclosure）

XR 环境是渐进式信息披露的天然场景——三维空间提供了比 2D 屏幕更丰富的信息分层手段。

| 披露策略 | 实现方式 | 适用场景 | 认知负担管理效果 |
|---|---|---|---|
| **注视触发展开** | 用户注视某元素 300–500ms 后展开详情 | 列表项摘要 → 完整内容 | 未注视时仅展示最小信息量，降低环境信息噪声 |
| **深度层级分离** | 摘要在前方（近处），详情在稍后方（远处）；注视后详情面板前移至舒适阅读深度 | 空间标注、卡片详情 | 利用空间深度实现自然的信息层级 |
| **距离衰减** | 远处内容自动简化为标题/图标级，用户靠近后逐步显示完整内容 | 虚拟白板、空间中的信息节点 | 远距离仅提供存在感知，近距离提供完整信息——匹配视锐度的自然衰减 |
| **上下文折叠** | 与当前任务无关的面板自动折叠为最小化状态（仅标题/图标） | 多面板工作区 | 减少外围信息对中央任务的干扰 |

## 8. 动态布局适应系统

### 8.1 适应维度总览

动态布局适应是指系统根据实时感知的用户状态和环境条件，自动调整排版和布局参数。

| 适应维度 | 感知信号 | 调整内容 | 调整速度 |
|---|---|---|---|
| **用户姿态** | 头显 IMU（坐/站/躺）、身体追踪 | 内容垂直偏移、面板倾斜角 | 0.5–1.0 秒缓动 |
| **注视行为** | 眼动追踪数据 | 注视区域字号优化、非注视区域降级 | 即时（< 100ms 延迟） |
| **阅读状态** | 注视模式分析（扫视 vs. 细读 vs. 浏览） | 字号、行高、信息密度 | 2–5 秒渐变 |
| **认知负载** | 瞳孔直径变化、注视持续时间、眨眼频率 | 信息密度、装饰元素可见度 | 5–10 秒渐变 |
| **环境亮度** | 头显亮度传感器 / passthrough 分析 | 对比度、色彩方案、字号 | 1–3 秒渐变 |
| **累积疲劳** | 使用时长、眨眼频率变化趋势、头部运动频率 | 字号渐大、行高渐增、信息密度渐降 | 分钟级渐变 |
| **内容深度** | 内容锚定的 Z 轴位置 | 字号缩放、字重调整 | 随深度变化即时调整 |

### 8.2 基于注视的动态排版（Gaze-Adaptive Typography）

利用眼动追踪实现的排版自适应是 XR 独有的能力，传统屏幕不具备。

#### 8.2.1 注视热区字号优化

| 区域 | 定义 | 排版策略 | 过渡方式 |
|---|---|---|---|
| **注视焦点** | 注视点周围 2°–3° | 全精度渲染，基准字号，最高可读性 | — |
| **近注视区** | 注视点 3°–8° | 维持基准字号，渲染质量可略降（注视式渲染，foveated rendering） | — |
| **预测扫视区** | 基于阅读方向预测的下一注视落点（行内右方 5°–8°） | 维持全精度渲染——为即将到来的注视准备好清晰文字 | — |
| **非注视区** | >10° 偏移 | 可降低渲染精度；如有文字，保持基准字号但允许抗锯齿级别降低 | — |

> **关键约束**：注视热区的排版优化**不应改变文字的布局位置或大小**——用户不应感知到文字在"动"。优化仅限于渲染精度和清晰度的调整，且必须在扫视到达目标前完成（扫视持续时间约 30–80ms，渲染调整必须在此时间内完成）。

#### 8.2.2 阅读状态检测与适应

系统可以通过眼动模式推断用户的阅读状态，并据此调整布局：

| 阅读状态 | 眼动特征 | 系统响应 |
|---|---|---|
| **精读（Deep Reading）** | 注视时长 250ms+，规律的水平扫视模式，极少跳跃 | 维持当前排版；关闭外围通知（除紧急级别）；降低环境动态效果 |
| **浏览（Scanning）** | 注视时长 150–200ms，大幅度跳视，跳过整行或整段 | 可考虑**加粗首句/关键词**辅助浏览；面板标题可见度增强 |
| **搜索（Searching）** | 不规律的大幅度扫视，频繁方向变化 | 提供搜索/目录快捷入口；增大标题/锚点的视觉权重 |
| **走神/疲劳** | 注视点漂移，固定注视时间异常延长（>500ms），眨眼频率上升 | 轻触提示休息；降低信息密度；增大字号 5%–10% |

### 8.3 基于认知负载的适应

瞳孔直径的变化（与亮度无关的微变化）是认知负载的可靠生理指标（task-evoked pupillary response, TEPR）。

| 认知负载水平 | 生理信号 | 布局适应策略 |
|---|---|---|
| **低负载** | 瞳孔稳定，眨眼频率正常（12–15 次/分钟） | 正常布局，可呈现完整信息 |
| **中等负载** | 瞳孔微扩张（+0.2–0.5mm），注视时长增加 | 非关键装饰元素降低可见度（降至 30% 不透明度）；减少动画干扰 |
| **高负载** | 瞳孔显著扩张（+0.5mm+），眨眼频率降低（< 10 次/分钟） | 主动简化界面——折叠次要面板，仅保留当前任务核心内容；字号 +5%，行高 +10% |
| **过载信号** | 瞳孔波动不定，频繁长眨眼（> 300ms），注视游移 | 建议休息提示；如正在执行复杂操作，提供分步引导简化任务 |

> **实施注意**：认知负载适应必须**极度保守和缓慢**。错误的负载判断（假阳性）会导致界面突然变化，反而增加认知负担。所有基于认知负载的适应：（1）变化渐进且可逆；（2）仅在信号持续 >5 秒时触发；（3）用户可手动覆盖（一键恢复完整界面）。

### 8.4 基于累积疲劳的时间维度适应

长时间使用 XR 后，视觉疲劳会累积。排版系统应随使用时长做渐进补偿。

| 使用时长 | 预期疲劳程度 | 排版渐进调整 |
|---|---|---|
| **0–30 分钟** | 低 | 基准参数，无调整 |
| **30–60 分钟** | 轻度 | 字号 +3%，行高 +3%（用户几乎不可感知） |
| **60–90 分钟** | 中度 | 字号 +5%–8%，行高 +5%–8%，建议休息提示 |
| **90–120 分钟** | 显著 | 字号 +10%，行高 +10%，信息密度自动降低 10%（非关键元素折叠） |
| **>120 分钟** | 高度 | 强烈建议休息提示（非强制但持续可见）；若继续使用，排版补偿保持 +10% 上限 |

> **补偿上限**：排版参数的疲劳补偿最大不超过 +10%。超过此幅度会明显改变布局和信息密度，用户会意识到"界面变了"，这本身成为认知干扰。超出补偿能力的疲劳应通过**建议休息**来解决，而非无限放大排版参数。

### 8.5 姿态适应

用户姿态（坐/站/躺）影响自然视线方向和舒适注视范围，进而影响内容面板的最佳位置和倾斜角度。

| 姿态 | 自然视线方向 | 内容面板偏移 | 面板倾斜角 | 行长调整 |
|---|---|---|---|---|
| **站姿** | 低于水平线约 10° | 面板中心在视线下方 5°–8° | 微向上倾斜 5°–10°（类似阅读时书本微倾） | 基准 |
| **坐姿** | 低于水平线约 15° | 面板中心在视线下方 5°–8°（绝对位置更低） | 微向上倾斜 10°–15° | 基准，坐姿水平舒适范围略小（±30° vs. ±35°） |
| **倚靠/半躺** | 低于水平线约 25°–30° | 面板自动上移至视线正前方偏下 5° | 向上倾斜 20°–30° | 可略放宽行长（用户更放松，水平舒适范围扩大） |
| **仰卧** | 正上方 | 面板直接置于正上方，面向用户 | 与用户视线垂直 | 基准，注意此姿态下头显重量压迫面部更严重，建议降低阅读时长建议 |

> **姿态检测与过渡**：姿态变化时面板位置的调整应使用 **0.5–1.0 秒的缓动动画**（ease-out），且文字内容在过渡期间不应重排。先完成位置迁移，若新位置导致原有布局不适（如面板角度变化后行长需调整），再在 1–2 秒内平滑调整排版参数。

### 8.6 环境上下文适应

| 环境变量 | 感知方式 | 排版适应 |
|---|---|---|
| **环境亮度突变** | 亮度传感器 / passthrough 帧分析 | 按 §4.4 亮度适应表调整字号、字重、对比度；过渡时间 1–3 秒 |
| **环境噪声（AR）** | 麦克风分析 | 嘈杂环境中用户注意力分散，字号 +5%–10%，关键信息加粗 |
| **运动状态** | IMU 加速度数据 | 步行/运动中字号 +15%–20%，减少信息密度至 30%–40%；仅显示最关键信息 |
| **多人协作** | 空间感知/手势追踪 | 共享面板字号需兼顾最远参与者的舒适视角；个人面板保持个性化 |

## 9. 过渡与动画策略

### 9.1 文本变化的过渡原则

文本内容或排版参数的变化在 XR 中比传统屏幕更容易引起认知中断，因为 3D 空间中的视觉变化比 2D 更引人注目。

| 变化类型 | 推荐过渡方式 | 持续时间 | 禁止做法 |
|---|---|---|---|
| **字号变化** | 平滑缩放动画（scale） | 300–500ms ease-out | 瞬间跳变（导致文本重排闪烁） |
| **字重变化** | 平滑插值（可变字体）或交叉淡入淡出 | 300–500ms | 瞬间字重切换（视觉"闪烁"） |
| **对比度/颜色变化** | 平滑色彩插值 | 500–1500ms | 瞬间色彩跳变（尤其暗↔亮） |
| **文本内容替换** | 交叉淡入淡出（旧内容淡出 + 新内容淡入） | 200–400ms | 瞬间替换（认知不连续） |
| **布局位置变化** | 平滑位移动画 | 300–600ms ease-out | 瞬间跳跃（空间迷失） |
| **面板出现/消失** | 淡入/淡出 + 微缩放（从 95% 缩放到 100%） | 200–400ms | 瞬间弹出（惊吓效应） |

### 9.2 动画对阅读的干扰管理

| 规则 | 说明 | 原理 |
|---|---|---|
| **R1：阅读中的面板不动画** | 用户正在阅读的文本面板禁止任何位移、缩放、旋转动画 | 阅读是依赖精确注视定位的认知活动，面板运动直接破坏注视-文字对应关系 |
| **R2：外围动画降级** | 当用户处于精读状态时（§8.2.2），非紧急的外围动画（通知、装饰效果）应自动降至 30% 不透明度或暂停 | 周边视觉对运动高度敏感，外围运动会不自觉地吸引注意力，打断阅读 |
| **R3：文字不跟随头部** | Head-locked（固定在头部坐标系中的）文字在头部转动时会产生相对背景的运动，应避免用于可读文字 | 运动中的文字无法稳定注视，阅读体验极差且容易引发晕动症 |
| **R4：使用 body-locked 或 world-locked 文字** | 可阅读文字应锚定在世界坐标或身体坐标中，仅在用户主动触发时跟随 | 文字在空间中"静止"是阅读的前提条件 |

## 10. 常见设计错误汇总

### 10.1 字号与字体错误

| 错误做法 | 生理/认知原因 | 修正方案 |
|---|---|---|
| 使用传统屏幕的 16px 字号直接转换到 XR | 16px 在 XR 中视角可能仅 0.3°–0.4°，远低于可读阈值 | 以视角度为基准重新设定字号，正文 ≥ 0.7° |
| 使用衬线字体作为 XR 正文 | 衬线在低 PPD 下退化为模糊噪点 | 系统 UI 和正文强制无衬线 |
| 使用 Light (300) 字重 | 细笔画在 XR 分辨率下接近消失 | 最低使用 Regular (400)，系统 UI 推荐 Medium (500)+ |
| 中文使用宋体/仿宋 | 横细竖粗结构在低 PPD 下横画不可见 | 使用黑体系（Noto Sans CJK、苹方） |
| 全大写长文本 | 全大写消除词形轮廓变化，降低阅读速度 10%–15% | 仅标签/按钮使用全大写，正文用正常大小写 |

### 10.2 排版参数错误

| 错误做法 | 生理/认知原因 | 修正方案 |
|---|---|---|
| 行高使用传统屏幕值（1.2×–1.4×） | XR 中回视率高，行间距不足时串行率飙升 | 正文行高 ≥ 1.5×，长文 ≥ 1.6× |
| 行长超过 80 个拉丁字符 | 回车扫视距离过长导致频繁串行 | 拉丁文正文 40–60 字符，中文 20–30 字 |
| 使用两端对齐 | 不均匀词间距在低 PPD 下形成"白河"效应 | 强制左对齐 |
| 使用负字间距（紧缩排版） | 低 PPD 下字母融合不可辨 | 禁止负 tracking，正文微增 +2%–5% |
| 段落间距与行间距差异不明显 | 段落边界模糊，信息分组失效 | 段落间距 ≥ 行高的 0.75 倍 |

### 10.3 对比度与色彩错误

| 错误做法 | 生理/认知原因 | 修正方案 |
|---|---|---|
| XR 中使用 WCAG 最低标准（4.5:1）作为目标 | WCAG 4.5:1 是为高 PPI 屏幕设定的下限，XR 低 PPD 需要更高补偿 | XR 正文对比度目标 ≥ 6:1 |
| 暗场景中使用纯白文字 #FFFFFF | 暗背景上的纯白产生过度眩光，光晕模糊字形边缘 | 使用 85%–90% 白度（如 #D9D9D9） |
| 暗场景中使用红色标记错误/警告 | Purkinje 位移使红色在低亮度下显著变暗 | 红色→琥珀色（#FFB74D） |
| AR 文字没有背板 | AR 叠加显示无法呈现黑色，浅色环境中文字不可见 | 深色半透明背板（≥ 60% 不透明度） |
| 对比度调整与亮度调整不同步 | 降低整体亮度时对比度被连带压缩 | 亮度调节时必须保护对比度（参见亮度适应 §5.5 D4） |

### 10.4 信息密度错误

| 错误做法 | 生理/认知原因 | 修正方案 |
|---|---|---|
| 将传统屏幕 UI 直接移植到 XR | XR 阅读速度仅传统屏幕 60%–70%，信息容量过载 | 重新设计信息架构，密度降至传统屏幕的 50%–60% |
| 中央凹区堆叠 10+ 个交互元素 | 超出短时记忆容量（7±2），工作记忆过载 | 中央凹区内 ≤ 7 个独立元素 |
| 一次性展示所有信息 | 信息过载导致选择困难和认知瘫痪 | 渐进式信息披露——按需展开 |
| 多个面板同时有文字动画/更新 | 多重运动干扰注意力，认知资源被碎片化 | 同一时刻仅一个面板有内容更新 |

### 10.5 动态适应错误

| 错误做法 | 生理/认知原因 | 修正方案 |
|---|---|---|
| 字号变化瞬间跳变 | 文本重排闪烁造成认知中断 | 300–500ms 平滑过渡 |
| Head-locked 可读文字 | 头部运动时文字相对世界运动，无法稳定阅读 | 使用 body-locked 或 world-locked |
| 用户阅读时面板移动 | 注视点与文字位置对应关系被破坏 | 阅读中冻结面板动画 |
| 基于认知负载的适应过于激进 | 界面频繁变化本身增加认知负担 | 适应变化渐进（>5 秒才触发）、缓慢、可逆 |
| 疲劳补偿无上限 | 过度放大排版参数改变布局，用户困惑"界面怎么变了" | 补偿上限 +10%，超出则建议休息 |
